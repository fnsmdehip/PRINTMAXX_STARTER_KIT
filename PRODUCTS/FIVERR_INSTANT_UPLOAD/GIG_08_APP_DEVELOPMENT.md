## FAQs

**Q: What's the difference between a PWA and a native app?**
```
A PWA runs in the browser but looks and feels like a native app. It can be installed on home screens, work offline, and send push notifications. The main differences: no App Store listing (your users access it via a link), no 30% Apple/Google commission on payments, and it works on ALL devices with one codebase instead of building separately for iOS and Android.
```

**Q: Can I submit a PWA to the App Store?**
```
You can wrap a PWA in a native shell using tools like PWABuilder or Capacitor and submit it to both stores. This is an add-on if needed. The PWA works great on its own without store listing. If store presence is important, I'll include the wrapper setup for an additional fee.
```

**Q: Do I own the code?**
```
100%. You get the full source code, deployed live, with documentation. You can modify it, hand it to another developer, host it wherever you want. No vendor lock-in. No monthly fees to me.
```

**Q: What about hosting costs?**
```
Most PWAs I build run on Vercel's free tier ($0/month) for up to 100K visitors. Standard apps with a database use Firebase or Supabase free tiers. Premium apps with high traffic may need paid hosting ($5-20/month). I'll tell you the expected cost before building.
```

**Q: Can you add features later?**
```
Yes. The source code is modular and well-documented. You can order additional features as new gigs, or any developer familiar with React/Next.js can extend it. Premium includes 30 days of support for feature adjustments after delivery.
```
