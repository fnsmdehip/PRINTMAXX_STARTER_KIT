questions

what's the difference between a PWA and a native app?
a PWA runs in the browser but looks and feels native. installable, works offline, push notifications. no App Store listing needed. works on ALL devices with one codebase.
do i own the code?
100%. full source code, deployed live, with documentation. modify it, hand it to another developer, host it wherever you want. no vendor lock-in.
what about hosting costs?
most apps run on Vercel's free tier ($0/month). apps with databases use Firebase/Supabase free tiers. Premium high-traffic apps: $5-20/month.
can i submit it to the App Store?
yes, with a native wrapper (PWABuilder or Capacitor). that's an add-on if needed. the PWA works great standalone.
can you add features later?
yes. code is modular and documented. order additional features as new gigs, or any React/Next.js developer can extend it.
