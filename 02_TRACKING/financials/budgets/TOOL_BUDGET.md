# Tool Budget by Phase

## Philosophy

- Free tiers first, upgrade only when hitting limits
- One tool per job (avoid redundancy)
- ROI must be clear before spending
- Audit quarterly, cancel what you don't use

---

### Tool Selection Criteria

Before adding any tool, answer:
1. What problem does this solve?
2. How much time will it save weekly?
3. Is there a free alternative?
4. What's the true monthly cost including time to learn?
5. Can I cancel easily if it doesn't work?

## Quarterly Tool Audit

Every 3 months, review:

1. **Usage check:** Did I use this tool weekly?
2. **ROI check:** Is it saving/making money?
3. **Overlap check:** Do I have duplicate tools?
4. **Upgrade check:** Am I hitting limits?
5. **Downgrade check:** Am I paying for unused features?

## Tool Mistakes to Avoid

1. **Shiny object syndrome** - Stick with working tools
2. **Annual billing too early** - Monthly until proven value
3. **Enterprise tiers** - You don't need them yet
4. **Multiple tools for same job** - One winner, cut the rest
5. **Complexity over simplicity** - Basic often better
6. **Paying for team features alone** - Wait for actual team
7. **Lifetime deals trap** - Many never get used
