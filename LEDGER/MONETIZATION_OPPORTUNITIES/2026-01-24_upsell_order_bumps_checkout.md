# Upsell Flow: Order Bumps (Pre-Purchase)

## Source
- E-commerce conversion benchmarks 2026
- Order bump vs upsell comparison research

## Tactic
Add checkbox offer on checkout page (before purchase) for related low-friction add-on.

## Specific Numbers
- **Good conversion rate: 2.5-3%** with unique strategies
- **Displayed on checkout page** before purchase completes
- **Checkbox acceptance** (not full product selection)
- **Smaller revenue boost than post-purchase upsells** but stackable

## How It Works
1. Customer proceeds to checkout
2. See checkbox offer below cart: "Add [product] for only $X"
3. Single checkbox to add to order
4. No leaving checkout page
5. Included in same transaction

## Order Bump vs Post-Purchase Upsell
| Feature | Order Bump | Post-Purchase Upsell |
|---------|------------|----------------------|
| Timing | Before purchase | After purchase |
| Conversion | 2.5-3% | 4-15% |
| Friction | Very low (checkbox) | None (one-click) |
| Offer Type | Small add-ons | Related/upgrade products |
| Best For | Impulse adds | Considered upgrades |

## Ideal Order Bump Offers
- Low-priced add-ons ($5-20)
- Consumables related to main product
- Extended shipping/warranty
- Digital add-ons to physical products
- Small upgrades (express shipping, gift wrap)

## Implementation Requirements
- Checkout page access (Shopify, WooCommerce)
- Order bump plugin or custom code
- A/B testing capability

## Tools Needed
**E-commerce:**
- WooCommerce: UpsellWP, WP Swings Upsell
- Shopify: Native checkout extensions
- FunnelKit for WooCommerce

**Info Products:**
- ThriveCart order bumps
- Gumroad (limited order bump support)

## Conversion Optimization
- **Keep it simple**: Single checkbox, no complex selection
- **Low price**: Under 30% of main product price
- **Clear value**: "Protect your purchase" or "Complete the set"
- **Visual**: Product image + benefit statement

## Why Conversion is Lower Than Post-Purchase
- Customer hasn't committed yet (no commitment bias)
- May add friction to checkout (cart abandonment risk)
- Competes with main purchase decision

## When to Use Order Bumps
**Use when:**
- Low-priced add-ons available
- Main product has obvious companion items
- Want to test multiple offers (order bump + post-purchase)

**Skip when:**
- No relevant low-priced add-ons
- Checkout friction is already high
- Better to save offer for post-purchase

## Applicable Money Methods
- INFO_PRODUCTS (MM002) - Add workbook/templates to course purchase
- AGENCY_SERVICES (MM005) - Add consultation call to package
- APP_FACTORY (MM001) - Less applicable (native checkout)

## Implementation Priority
MEDIUM - Lower conversion than post-purchase, but stackable for additional 2-3% lift

## Best Practices
- Test one order bump at a time (don't show 3+ options)
- A/B test price points ($9, $14, $19)
- Match offer to main product (don't upsell unrelated)
- Use scarcity sparingly ("Limited time" works if true)

## Example Offers
**Info Product:**
- Course ($99) + Order Bump: Workbook PDF ($14)
- Ebook ($29) + Order Bump: Video summary ($9)

**Physical Product:**
- Supplement ($49) + Order Bump: Travel size ($12)
- Camera ($299) + Order Bump: Memory card ($19)

**SaaS:**
- Annual plan ($199) + Order Bump: Onboarding call ($49)

## Anti-Patterns to Avoid
- Order bump more expensive than main product
- Multiple order bumps (creates decision fatigue)
- Unrelated offers (kills conversion)
- Hidden in fine print (must be visible)
