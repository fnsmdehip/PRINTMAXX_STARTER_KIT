# Pricing Strategy: Quarterly A/B Testing

## Source
- B2B SaaS Conversion Benchmarks 2026
- Indie Hackers pricing research

## Tactic
Run quarterly A/B tests on pricing tiers and anchoring to optimize conversion rates.

## Specific Numbers
- **Up to 25% improvement** in conversion rates from pricing A/B tests
- **60% of high-growth SaaS** conduct quarterly competitive pricing reviews
- **Visual design and value framing** more impactful than actual price changes
- **Pricing pages with 4+ options** reduce conversion by 30%

## How It Works
1. Set up A/B test infrastructure (Stripe Pricing Tables, custom)
2. Test variables quarterly:
   - Pricing tier structure
   - Anchor pricing (highest tier positioning)
   - Value framing language
   - Visual presentation
3. Measure conversion rate impact
4. Deploy winning variant

## Implementation Requirements
- A/B testing tool (Optimizely, VWO, or Stripe Experiments)
- Sufficient traffic for statistical significance
- Conversion tracking setup

## Tools Needed
- Stripe Pricing Tables (built-in A/B testing)
- Google Optimize (free)
- Optimizely or VWO (paid)

## Key Insights
- Most indie hackers leave **40-60% of potential revenue** on the table
- Visual design > price changes for conversion impact
- Keep options ≤3 tiers for maximum conversion
- Never deploy major pricing changes without A/B test first

## Applicable Money Methods
- APP_FACTORY (MM001) - Test subscription tier pricing
- INFO_PRODUCTS (MM002) - Test course pricing tiers
- SAAS (MM004) - Test monthly vs annual pricing
- AGENCY_SERVICES (MM005) - Test package pricing

## Implementation Priority
HIGHEST - 25% conversion lift proven, low implementation cost, applicable to all revenue models

## Testing Checklist
- [ ] Test 2-3 pricing tiers (not 4+)
- [ ] Test anchor pricing (annual in center as "Best value")
- [ ] Test value framing ("Save 40%" vs "$X/month")
- [ ] Test visual hierarchy (middle tier highlighted)
- [ ] Run for minimum 2 weeks or 100 conversions per variant
