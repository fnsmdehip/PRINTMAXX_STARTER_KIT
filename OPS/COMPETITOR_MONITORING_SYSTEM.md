## Monitoring Workflows

### Daily (5 minutes)

1. Check competitor social posts
2. Note any viral content
3. Log interesting tactics to ALPHA_STAGING.csv

### Weekly (30 minutes)

**Monday:**
1. Review competitor app updates
2. Check App Store/Play Store pages
3. Note any pricing or feature changes

**Friday:**
1. Review competitor content performance
2. Identify top-performing posts
3. Note formats/hooks to test

### Monthly (2 hours)

1. Full competitor audit:
   - Website changes
   - Pricing updates
   - New products/features
   - Team changes

2. Traffic/rank analysis:
   - SimilarWeb or SEMrush
   - App store rankings
   - Social follower growth

3. Strategic review:
   - What's working for them?
   - What opportunities exist?
   - What should we test?

## Competitive Intelligence Log

### Format

```markdown
## [Date] - [Competitor]

### Observation
[What you noticed]

### Analysis
[Why this matters]

### Action Items
- [ ] [What to do about it]
- [ ] [Test or implement]
```

### Example Entry

```markdown
## 2026-01-23 - Forest App

### Observation
Forest added "Social Forest" feature - users can plant trees together.

### Analysis
Adding social/community features to solo apps. Increases retention
and word-of-mouth. Could apply to PrayerLock (prayer partners).

### Action Items
- [ ] Research community features for focus apps
- [ ] Add "Prayer Partner" feature to PrayerLock backlog
- [ ] Monitor Forest reviews for feature reception
```

## Competitive Response Framework

### When Competitor Launches Feature

1. **Assess:** Is this relevant to our users?
2. **Prioritize:** Should we copy, improve, or ignore?
3. **Timeline:** If building, how quickly?
4. **Differentiate:** How do we do it better?

### When Competitor Changes Pricing

1. **Analyze:** Are they going up or down? Why?
2. **Impact:** How does this affect our positioning?
3. **Response:** Adjust messaging, not necessarily price
4. **Communicate:** If relevant, tell customers

### When Competitor Goes Viral

1. **Study:** What specifically drove virality?
2. **Extract:** Replicable elements vs. one-time luck
3. **Apply:** Test similar approaches
4. **Document:** Add to content playbook
