## 2. Feature Prioritization Framework

### Impact vs Effort Matrix

**Quadrant Definitions:**

| Quadrant | Characteristics | Action |
|----------|-----------------|--------|
| Quick Wins | High impact, low effort | Do immediately |
| Big Bets | High impact, high effort | Plan and schedule |
| Fill-Ins | Low impact, low effort | Do when blocked |
| Money Pits | Low impact, high effort | Avoid entirely |

### User Value Scoring (1-5)

| Score | Criteria |
|-------|----------|
| 5 | Feature requested by >20% of users, solves major pain point |
| 4 | Frequently requested, clear user benefit |
| 3 | Some demand, moderate benefit |
| 2 | Occasional request, nice-to-have |
| 1 | No user demand, internal idea only |

**User Value Signals:**
- App Store review mentions
- Support ticket frequency
- Survey responses
- Social media requests
- Competitor feature parity

### Business Value Scoring (1-5)

| Score | Criteria |
|-------|----------|
| 5 | Direct revenue increase >20%, major competitive advantage |
| 4 | Clear revenue impact, strong differentiation |
| 3 | Moderate revenue/retention impact |
| 2 | Minor business benefit |
| 1 | No measurable business impact |

### Technical Complexity Assessment (1-5)

| Score | Criteria |
|-------|----------|
| 1 | <1 day, single developer, no dependencies |
| 2 | 1-3 days, frontend only, minimal testing |
| 3 | 1-2 weeks, backend changes, moderate testing |
| 4 | 2-4 weeks, new infrastructure, significant testing |
| 5 | 1+ month, major architecture changes, extensive testing |

### Priority Calculation

**Formula:**
```
Priority Score = (User Value + Business Value) / Technical Complexity
```

**Priority Tiers:**
- Score >4.0 = P0 (Do now)
- Score 2.5-4.0 = P1 (Plan next sprint)
- Score 1.5-2.5 = P2 (Backlog)
- Score <1.5 = P3 (Don't do)

### Feature Evaluation Template

| Feature | User Value | Business Value | Complexity | Score | Notes |
|---------|------------|----------------|------------|-------|-------|
| Example | 4 | 5 | 2 | 4.5 | High priority |

## 3. Content Channel Roadmap

### Launch Phase (Month 1-3)

**Week 1-2: Foundation**
- [ ] Create accounts on all target platforms
- [ ] Complete profile optimization (bio, links, profile pic)
- [ ] Warmup accounts (follow relevant accounts, engage)
- [ ] Set up content calendar
- [ ] Batch first 30 posts

**Week 3-4: Content Velocity**
- [ ] Post 3x/day minimum
- [ ] Test 5 different content formats
- [ ] Track performance per format
- [ ] Identify top 2 performing formats
- [ ] Double down on winners

**Month 2: Consistency**
- [ ] Maintain 3x/day posting
- [ ] Respond to all comments
- [ ] Start DM engagement strategy
- [ ] Collaborate with 2 similar accounts
- [ ] Hit 1,000 followers milestone

**Month 3: Momentum**
- [ ] Scale to 5x/day if performing

## 4. Info Product Roadmap

### Research Phase (Week 1-2)

**Market Research:**
- [ ] Identify 10 competitor products
- [ ] Analyze pricing ($9-$997 range)
- [ ] Read 50+ reviews of competitors
- [ ] List common complaints/gaps
- [ ] Survey audience (if available)

**Topic Validation:**
- [ ] Search volume check (Ahrefs, Google Trends)
- [ ] Reddit/forum demand signals
- [ ] Pre-sell test (landing page + email capture)
- [ ] Direct outreach: "Would you buy X?"
- [ ] Minimum 50 email signups before building

**Research Phase Exit Criteria:**
- [ ] Clear gap in market identified
- [ ] 50+ pre-launch email signups
- [ ] Competitor pricing mapped
- [ ] Unique angle defined

---

### Creation Phase (Week 3-6)

**Week 3: Outline**
- [ ] Complete product outline
- [ ] Define modules/chapters
- [ ] List all deliverables
- [ ] Create asset checklist (templates, worksheets)

**Week 4-5: Production**
- [ ] Write/record core content
- [ ] Create supporting materials
- [ ] Build delivery platform (Gumroad, Teachable)
- [ ] Design sales page copy

**Week 6: Polish**
- [ ] Beta test with 5-10 users

**Creation Phase Exit Criteria:**
- [ ] Product complete and hosted

## Roadmap Review Cadence

**Weekly:** Check metrics against targets, adjust tactics
**Monthly:** Review milestone progress, update priorities
**Quarterly:** Strategic review, roadmap adjustment
**Annually:** Full retrospective, next year planning

## Templates Usage Guide

1. Copy relevant template section
2. Customize milestones for your specific product
3. Fill in actual dates and owners
4. Track progress weekly
5. Adjust based on real data (not assumptions)

**Remember:** Roadmaps are guides, not commitments. Adjust based on:
- User feedback (most important)
- Revenue data (second most important)
- Competitive moves
- Resource constraints
- Market changes
