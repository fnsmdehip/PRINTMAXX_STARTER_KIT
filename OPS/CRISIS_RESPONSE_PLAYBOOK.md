# Crisis Response Playbook

Protocols for handling PR crises, platform issues, app store problems, legal matters, and technical emergencies.

---

## Crisis Severity Levels

### Level 1: Minor (Handle within 24h)
- Negative comment/review
- Minor feature bug
- Single customer complaint
- Content underperforming

### Level 2: Moderate (Handle within 4h)
- Multiple negative reviews simultaneously
- Partial service disruption
- Payment processing issues
- Social media callout from small account

### Level 3: Severe (Handle immediately)
- App store rejection/removal
- Account suspension
- Data breach or security issue
- Legal threat received
- Viral negative content
- Complete service outage

---

## 1. App Store Crises

### App Rejected

**Immediate actions (within 1 hour):**
1. Read full rejection reason in App Store Connect
2. Screenshot all communications
3. Don't panic-reply

**Response process:**
1. Identify exact guideline violated
2. Research if others have faced same issue
3. Prepare appeal with:
   - Acknowledgment of issue
   - Specific changes made
   - Evidence of compliance
4. Submit appeal via Resolution Center

**Appeal template:**
```
Dear App Review Team,

Thank you for reviewing [App Name]. We understand the concern regarding [specific guideline].

We have made the following changes:
1. [Change 1]
2. [Change 2]
3. [Change 3]

We believe these changes address the [guideline number] requirements because [explanation].

We respectfully request a re-review of our application.

Thank you for your time.
[Your name]
```

**If appeal denied:**
- Request phone call with App Review (available for developers)
- Modify approach and resubmit
- Consider alternative features

### App Removed

**Immediate actions:**
1. Check email for removal reason
2. Assess if this affects other apps on same account
3. Stop all marketing spend immediately

**Recovery options:**
1. Address violations and appeal
2. Create new developer account (last resort, risks)
3. Modify app significantly and rebrand

**Prevention:**
- Maintain multiple developer accounts
- Don't use same company name across accounts
- Read guidelines before every major update

## 2. Social Media Crises

### Account Suspended

**Platform-specific recovery:**

**Instagram:**
1. Check email for specific violation
2. Submit appeal through app or help.instagram.com
3. Wait 24-48h, then submit again if no response
4. Use Facebook Business support if available

**TikTok:**
1. Check Creator Portal for violation details
2. Appeal through app notification
3. Email: feedback@tiktok.com
4. Wait 7 days between appeals

**X/Twitter:**
1. Check email for suspension reason
2. Appeal at help.twitter.com/forms/account-access
3. Provide context and evidence
4. Be persistent (can take weeks)

**YouTube:**
1. Check Studio for strikes
2. Appeal each strike individually
3. Complete required training if offered
4. Request reinstatement after appeal

### Negative Viral Content

**Assessment (first 30 minutes):**
1. Is it accurate or false?
2. Who posted it? (Reach, credibility)
3. How fast is it spreading?
4. What's the sentiment in replies?

**Response decision tree:**

```
Is the criticism accurate?
├── YES: Acknowledge, apologize, fix
│   └── Public response within 2 hours
└── NO: Assess if correction is worth it
    ├── Small account (<10k): Monitor, don't amplify
    └── Large account (>10k): Calm correction with evidence
```

**Response templates:**

**When you're wrong:**
```
You're right. We messed up on [specific issue].

Here's what we're doing:
1. [Immediate fix]
2. [Process change to prevent recurrence]
3. [Make-good for affected customers]

We appreciate you holding us accountable.
```

**When it's false:**
```
We want to address some inaccurate information:

[Claim]: "[What they said]"
[Fact]: "[What's actually true]"

Happy to provide documentation/evidence to anyone who wants it.
```

**When it's subjective:**
```
We hear you. [Product] isn't for everyone.

We designed it for [specific use case]. If that's not your need, we get it.

For those it does fit, here's what they've found: [positive examples]
```

### Harassment/Doxxing

**Immediate actions:**
1. Screenshot everything (with timestamps)
2. Do NOT engage publicly
3. Report to platform
4. If threats involved, contact local police
5. Consider temporary account deactivation

**Protection measures:**
- Remove personal address from public records
- Set up Google Alerts for your name
- Enable two-factor on all accounts
- Use PO Box for business

## 3. Technical Crises

### Service Outage

**First 15 minutes:**
1. Confirm outage scope (partial vs complete)
2. Identify cause if possible
3. Post status update

**Communication template:**
```
[TIMESTAMP] We're aware of issues with [specific feature/service].

Status: Investigating
Impact: [Who's affected, what's not working]
Next update: [Time]

We'll update this thread as we learn more.
```

**Resolution announcement:**
```
[TIMESTAMP] [Service] is now fully operational.

What happened: [Brief, non-technical explanation]
Duration: [Time down]
Affected users: [Number/percentage]

We've implemented [change] to prevent recurrence.

Thank you for your patience.
```

### Data Breach

**Legal requirement: Notify affected users within 72 hours in most jurisdictions.**

**Immediate actions:**
1. Contain the breach (disconnect systems if needed)
2. Document everything
3. Contact legal counsel
4. Begin forensic investigation
5. Draft notification

**User notification template:**
```
Subject: Important Security Notice - Action Required

Dear [User],

We're writing to inform you of a security incident affecting your account.

What happened:
On [date], we discovered [brief, factual description].

What information was involved:
[List specific data types - email, password hash, etc.]

What we're doing:
1. [Action 1]
2. [Action 2]
3. [Action 3]

What you can do:
1. Change your password immediately
2. [Other specific actions]
3. Monitor your accounts for unusual activity

We take your privacy seriously and apologize for this incident.

Questions? Contact us at [email/phone].

[Signature]
```

### Payment Processing Failure

**Immediate actions:**
1. Identify scope (all payments vs specific provider)
2. Enable backup payment processor if available
3. Pause any automated billing
4. Communicate with affected customers

**Customer communication:**
```
Hi [Name],

There was an issue processing your recent payment.

Don't worry - your account is still active. We'll retry the payment in 24 hours.

If you'd like to update your payment method: [link]

Questions? Just reply to this email.

Thanks,
[Name]
```

## 4. Legal Crises

### Cease and Desist Received

**Immediate actions:**
1. Do NOT respond immediately
2. Save all documentation
3. Contact attorney
4. Assess validity of claims

**Assessment questions:**
- Is this from a legitimate law firm?
- What specific claim are they making?
- Do they have a valid legal basis?
- What are they asking you to do?
- What's the deadline?

**Options:**
1. Comply (if valid and not worth fighting)
2. Negotiate (partial compliance)
3. Contest (if invalid claim)
4. Ignore (risky, only for obvious nonsense)

### DMCA Takedown

**If you receive one:**
1. Review the specific content claimed
2. Assess if fair use applies
3. Either remove content or file counter-notice

**Counter-notice (if you believe it's fair use):**
```
[Your name and contact info]

I declare under penalty of perjury that I have a good faith belief that the material was removed or disabled as a result of mistake or misidentification of the material.

Content removed: [URL/description]
Reason for counter-notice: [Explanation of fair use or ownership]

I consent to jurisdiction of [Federal District Court for your address].

[Signature]
[Date]
```

### Trademark Issues

**Someone using your trademark:**
1. Document the infringement
2. Send polite cease and desist
3. Escalate to platform (trademark complaint forms)
4. Legal action if needed

**Accused of trademark infringement:**
1. Assess if claim has merit
2. Options: rebrand, fight, or negotiate coexistence
3. Consult attorney before responding

## 5. Customer Crises

### Viral Customer Complaint

**Response within 2 hours:**
1. Acknowledge publicly
2. Move to private conversation
3. Resolve issue generously
4. Follow up publicly with resolution

**Public response template:**
```
We're sorry you had this experience. That's not the [Brand] standard.

I've DMed you to make this right. We'll update here once resolved.

— [Name], [Title]
```

**After resolution:**
```
Update: We've resolved [Customer]'s issue.

They received [resolution]. We've also [systemic fix] to prevent this for others.

Thanks for your patience and for holding us accountable.
```

### Refund Demand (Legitimate)

Process immediately and generously. Cost of refund < cost of negative review.

**Response:**
```
Hi [Name],

I've processed your full refund. You should see it within 3-5 business days.

I'm sorry [product/service] didn't meet your expectations. If you're comfortable sharing, I'd love to know what we could improve.

Either way, thank you for giving us a try.

[Name]
```

### Refund Demand (Scam/Abuse)

**Red flags:**
- Requesting refund after fully using service
- Pattern of refund requests
- Threatening negative review unless refunded

**Response:**
```
Hi [Name],

I've reviewed your account and see that [specific usage that contradicts refund claim].

Our refund policy states [policy]. Based on your usage, a refund isn't available.

If there's a specific issue with the product, I'm happy to help resolve it.

[Name]
```

## 6. Communication Templates

### Holding Statement (When You Need Time)

```
We're aware of [situation] and are looking into it.

We take this seriously and will share more information as soon as we have it.

We expect to update you by [time].
```

### Full Public Apology

```
We made a mistake with [specific issue]. We're sorry.

What happened:
[Honest, non-defensive explanation]

Who was affected:
[Scope]

What we're doing:
1. [Immediate fix]
2. [Compensation/make-good]
3. [Systemic change]

We know trust is earned through actions, not words. We'll do better.

[Signature]
```

### Internal Crisis Brief

```
CRISIS BRIEF
Date/Time: [When discovered]
Severity: [1/2/3]

2. [Action with owner and timestamp]

1. [Action with owner and deadline]
2. [Action with owner and deadline]

STAKEHOLDERS NOTIFIED:
[List]
```

## 8. Post-Crisis Actions

### Immediate (Within 24h)

- [ ] Document timeline of events
- [ ] Screenshot all communications
- [ ] Calculate impact (users, revenue, time)
- [ ] Notify relevant stakeholders

### Short-term (Within 1 week)

- [ ] Conduct post-mortem
- [ ] Identify root cause
- [ ] Implement preventive measures
- [ ] Update documentation
- [ ] Train team on changes

### Long-term (Within 1 month)

- [ ] Follow up with affected users
- [ ] Monitor for recurrence
- [ ] Update crisis playbook
- [ ] Review insurance coverage
- [ ] Consider public case study (if appropriate)

### Post-Mortem Template

```
POST-MORTEM: [Crisis Name]
Date: [Date of incident]
Severity: [1/2/3]
Duration: [Start to resolution]

SUMMARY:
[One paragraph overview]

[Timestamp]: [Event]

- Reputation impact: [Assessment]

WHAT WENT WELL:
1. [Positive]
2. [Positive]

1. [Negative]
2. [Negative]

ACTION ITEMS:

|--------|-------|----------|--------|
| | | | |

[Specific measures to prevent recurrence]
```
