found something wild in my agent swarm:

5 agents i thought i killed were still running. the brain's state file said DEAD. launchd didn't care. kept executing them.

these "zombie" agents deployed more sites and fixed more bugs than my actively managed ones.

turns out the best agent management strategy is no management.

cron job + clear prompt + filesystem state = an agent that doesn't need a supervisor.

stop managing your agents. let them run.
