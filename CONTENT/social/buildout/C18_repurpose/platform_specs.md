## Platform-by-Platform Deep Spec

### TikTok

**Video:**
- Resolution: 1080x1920 (minimum 720x1280)
- Aspect ratio: 9:16 (16:9 allowed, won't perform as well)
- Frame rate: 24-60 fps (30 or 60 recommended)
- Format: MP4 or MOV
- Codec: H.264 or H.265
- Max duration: 10 minutes (but 15-60 sec performs best in 2025)
- Max file size: 287.6 MB

**Audio:**
- Required for most content
- Codec: AAC
- Sample rate: 44,100 Hz
- Channels: Stereo preferred
- Volume: normalize to -14 LUFS

**Caption:**
- Max: 2,200 characters
- Hashtags count toward character limit
- First 100 characters visible before "more"
- Hook must be in first 2 lines

**Thumbnail:**
- 1080x1920 (same as video)
- JPG or PNG
- Auto-generated if not provided

**Best practices:**
- Upload natively via TikTok app (algorithm favors native uploads)
- No watermarks from other platforms — TikTok suppresses them
- Posted content gets a 15-minute window to pick up engagement before algorithm classifies it
- Captions with keywords improve search discoverability

### Instagram Reels

**Video:**
- Resolution: 1080x1920 recommended
- Minimum: 500x888
- Aspect ratio: 9:16 required for Reels
- Frame rate: 23-60 fps
- Format: MP4 or MOV
- Codec: H.264
- Max duration: 90 seconds (15-30 sec performs best)
- Max file size: 4 GB

**Audio:**
- Required
- Normalized audio performs better with IG algorithm

**Caption:**
- Max: 2,200 characters
- First 125 characters show before "more"
- Line breaks supported
- Hashtags: put in first comment to keep caption clean, or batch at bottom

**Cover/Thumbnail:**
- 1080x1920 JPG or PNG
- Will appear in profile grid cropped to 1:1

**Hashtag strategy:**
- 5-8 targeted hashtags outperform 30 generic ones (as of 2025)
- Mix: 1 large (1M+), 2 medium (100K-1M), 2-3 niche (under 100K)
- Don't use banned hashtags (shadowban risk)

**Saves signal > likes to Reels algorithm**
- Always include a CTA to save ("save this for later")

### YouTube Shorts

**Video:**
- Resolution: 1080x1920
- Aspect ratio: 9:16 required (16:9 excluded from Shorts shelf)
- Frame rate: 24-60 fps
- Format: MP4
- Max duration: 60 seconds
- No minimum file size

**Audio:**
- Optional but recommended
- Copyright-cleared audio required for monetization

**Title:**
- Max 100 characters
- Include #Shorts in title or description for Shorts shelf eligibility
- Keyword-first titles help search

**Description:**
- Max 5,000 characters
- Include #Shorts
- No clickable links (links only in first comment or channel description)

**Thumbnail:**
- Auto-generated from video frames
- Custom thumbnail supported (1280x720 for Shorts)

**Upload note:**
- Upload via YouTube Studio or mobile
- Do NOT use third-party upload tools (suppresses reach)
- After upload, YouTube takes 24-48 hours to classify as Short

### Facebook Reels

**Video:**
- Resolution: 1080x1920
- Aspect ratio: 9:16
- Frame rate: 24-60 fps
- Format: MP4 or MOV
- Max duration: 90 seconds
- Max file size: 4 GB

**Audio:** Required

**Caption:**
- Max: 63,206 characters (essentially unlimited)
- Hashtags: 0-3 perform best (FB algorithm doesn't boost for hashtags like TikTok)

**Easiest distribution method:**
- Publish IG Reel → Meta Business Suite → crosspost to Facebook → 1 click
- Takes 2 minutes, no reformatting needed

**Best practices:**
- Facebook audience skews older (25-45) vs TikTok (18-30)
- Longer captions work better on FB (more storytelling)
- Facebook Groups crosspost drives extra reach

### YouTube Long-Form

**Video:**
- Resolution: 1920x1080 (4K: 3840x2160)
- Aspect ratio: 16:9 standard (1:1 and 4:3 supported)
- Frame rate: 24, 30, 48, 60 fps
- Format: MP4 recommended (MOV, AVI, MKV supported)
- Codec: H.264 (H.265 for 4K)
- Max duration: 12 hours (but 8-15 min sweet spot for algorithm)
- Max file size: 128 GB

**Audio:**
- Stereo AAC, 320kbps
- Normalize to -14 LUFS

**Title:**
- Max 100 characters
- Front-load primary keyword
- Use numbers and outcome words
- Example: "7 Side Hustles That Actually Pay in 2025 (No Experience)"

**Description:**
- Max 5,000 characters
- First 200 characters appear in search
- Include timestamps, affiliate links, subscribe CTA, social links

**Tags:**
- 10-15 tags (YouTube tag limit: 500 characters total)
- Use exact match of title + 3-4 variants + broad topic tags

**Thumbnail:**
- 1280x720 minimum (16:9)
- Max 2 MB file size
- JPG, GIF, PNG
- Rule: bright color, bold text under 5 words, contrasting face or graphic

### X / Twitter

**Video:**
- Resolution: 1280x720 (16:9) or 1080x1920 (9:16)
- Aspect ratio: 16:9, 1:1, or 9:16 all supported
- Frame rate: Up to 60 fps
- Format: MP4 or MOV
- Codec: H.264
- Max duration: 2 minutes 20 seconds
- Max file size: 512 MB

**Caption/Tweet:**
- Max: 280 characters (standard), 4,000 (Twitter Blue / X Premium subscribers)
- Links count as 23 characters regardless of length
- No more than 1-2 hashtags (algorithm suppresses heavy hashtag use)

**Image:**
- JPG, PNG, GIF, WEBP
- Max 5 MB for photos, 15 MB for GIFs
- Up to 4 images per tweet

**Thread format:**
- Each tweet: 280 chars max
- Thread numbered tweets: "1/" "2/" etc. are optional but common
- Don't use "Thread:" — just start with the hook

### LinkedIn

**Video:**
- Resolution: 1920x1080 (16:9) or 1080x1920 (9:16 in app)
- Frame rate: 10-60 fps
- Format: MP4, ASF, AVI, FLV, MOV, MPEG-1, MPEG-4, MKV, WebM
- Max duration: 10 minutes
- Max file size: 5 GB
- Min file size: 75 KB

**Post text:**
- Max: 3,000 characters
- First 140 characters visible before "see more"
- Hook must be in first 2 lines
- DO NOT include links in post body — LinkedIn suppresses posts with external links
- Post link in first comment immediately after publishing

**Article (LinkedIn native):**
- No character limit
- Headline: max 150 characters
- Best for long-form thought leadership content

**Hashtags:**
- 3-5 perform best
- Add at end of post, not inline
- Niche-specific hashtags drive more relevant impressions

### Pinterest

**Video Pin:**
- Resolution: 1080x1920 (2:3 also supported)
- Min resolution: 240x240
- Frame rate: 25 or 30 fps
- Format: MP4 or MOV
- Max duration: 15 minutes (under 60 sec performs best)
- Max file size: 2 GB

**Static Pin:**
- Resolution: 1000x1500 (2:3) — standard Pin
- Resolution: 1000x1000 (1:1) — square Pin
- Format: JPG or PNG
- Max file size: 20 MB

**Description:**
- Max: 500 characters
- First 50-60 characters appear in feed
- Include target keyword in first sentence
- No hashtags in description (Pinterest algo ignores them)

**Title:**
- Max: 100 characters
- Front-load keyword

**Board:**
- Board name: keyword-rich, clear topic
- Board description: 500 characters, keyword-dense

## Audio-Only Platforms

### Podcast (via RSS — Buzzsprout, Anchor, Podbean)

**Audio file:**
- Format: MP3 (universal) or M4A
- Bitrate: 128 kbps mono (voice-only) or 192 kbps stereo (music/effects)
- Sample rate: 44,100 Hz
- Normalized to: -16 LUFS (podcast standard)
- Max file size: depends on host (Buzzsprout free: 3 hours/month total)

**RSS distribution:**
- Submit RSS once to: Apple Podcasts, Spotify, Google Podcasts, Amazon Music, iHeartRadio, Stitcher, TuneIn
- Buzzsprout auto-submits to most

**Episode metadata:**
- Title: max 255 chars (keep under 60 for display)
- Description/Show notes: 4,000 chars recommended
- Tags/Categories: select from Apple's fixed category list

## Format Conversion Cheat Sheet (ffmpeg)

```bash
# 16:9 → 9:16 (crop center for vertical)
ffmpeg -i input.mp4 -vf "crop=ih*9/16:ih,scale=1080:1920" output_vertical.mp4

# Trim clip (start=90sec, duration=45sec)
ffmpeg -i input.mp4 -ss 00:01:30 -t 00:00:45 -c copy clip.mp4

# Convert to mp4 H.264 (universal compatibility)
ffmpeg -i input.mov -c:v libx264 -c:a aac -strict -2 output.mp4

# Normalize audio to -14 LUFS
ffmpeg -i input.mp4 -filter:a loudnorm=I=-14:TP=-1.5:LRA=11 output_normalized.mp4

# Add burned-in captions from SRT file
ffmpeg -i input.mp4 -vf subtitles=captions.srt output_captioned.mp4

# Scale to 1080p
ffmpeg -i input.mp4 -vf scale=1920:1080 -c:a copy output_1080p.mp4

# Extract audio only (for podcast)
ffmpeg -i input.mp4 -vn -acodec mp3 -ab 192k output_audio.mp3

# Add black letterbox bars to make 9:16 without cropping
ffmpeg -i input_16x9.mp4 -vf "scale=1080:-1,pad=1080:1920:(ow-iw)/2:(oh-ih)/2:black" output_vertical_padded.mp4
```
