Core differences I've identified:
- Google AIO selects for direct question-answering, not keyword density

Built a starter kit covering:
1. Content audit framework (what to change vs. what to delete)

This is early research, not a definitive playbook. AIO selection criteria are not fully reverse-engineered yet.

## POST 3 — Ask HN: What's the actual technical barrier to offline-first PWAs getting traction?
**Title:** Ask HN: Why do offline-first PWAs consistently underperform equivalent native apps in user retention?

**Body:**
I've built several offline-first PWAs that are functionally equivalent to native apps — they install to home screen, work without network, use local storage, push notifications, etc.

Consistent observation: even when the feature set is identical, native apps retain users at roughly 2-3x the rate of PWAs, particularly in the first 30 days.

Possible explanations I've considered:
- Discoverability (App Store surfaces native apps, PWAs have no equivalent)
- Trust signals (App Store review process creates implicit trust)
- Installation friction (Add to Home Screen is obscure for most users)
- Performance perception (even equivalent PWAs are often perceived as slower)
- Push notification opt-in rates (PWA notification prompts convert at lower rates than native)

Curious if anyone has data on which of these dominate. I'm trying to decide whether to convert 76 existing PWAs to native wrappers (Capacitor) or invest in better PWA discovery/installation UX.
