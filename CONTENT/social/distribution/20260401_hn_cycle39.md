## POST 1 — Show HN: TruthScope (pre-launch, post when App Store submission is in)

**Title:** Show HN: TruthScope — Lie detector using real PPG, voice stress analysis, and facial landmark tracking

**Body:**
Most lie detector apps on the App Store are random number generators with biometric-looking UIs. I verified this by downloading each one and watching the network requests — zero sensor calls, just timers and Math.random().

TruthScope uses three real physiological channels:

**1. PPG (photoplethysmography)**
The phone camera + flash measures subtle color changes in your fingertip caused by blood flow at each heartbeat. The same principle your Apple Watch uses for heart rate. Sampling at 30fps, we extract heart rate and beat-to-beat variability (HRV proxy) — both correlate with autonomic nervous system activation under stress.

**2. Voice stress analysis**
Short-time Fourier transform on live audio input, looking for 8-14 Hz tremors in the fundamental frequency. These microtremors are associated with laryngeal muscle tension under stress. Not lie detection per se — stress detection correlated with deception attempts. We're honest about this distinction in the UI.

**3. Facial micro-expressions**
MediaPipe Face Mesh at 30fps for 468-point landmark tracking. The 7 Ekman universal micro-expressions (fear, disgust, surprise, contempt, sadness, happiness, anger) are detected via landmark displacement patterns at sub-250ms. The UI doesn't claim this identifies lies — it identifies involuntary emotional leakage.

**Multi-modal fusion**
Three independent signals feed a fusion layer. Single-channel noise (bad lighting, background noise, inconsistent finger pressure) gets down-weighted. Only when multiple channels agree does the confidence score move significantly.

**What we don't claim:**
- Not polygraph accuracy (polygraphs themselves have ~80% accuracy and are beatable)
- Not court-admissible
- Not medical
- Entertainment and social use — but with real signal

**Tech stack:** Expo React Native, react-native-vision-camera (real camera frames — not expo-camera), react-native-webrtc for audio FFT, MediaPipe via WebView for face mesh. Stripe Payment Links for premium tier.

Happy to go deep on any of the signal processing. The PPG implementation in particular involved a lot of trial and error with lighting normalization.

## POST 3 — Show HN: cnsnt — Offline consent form builder with AES-256 encryption

**Title:** Show HN: cnsnt — Local-first consent form builder (AES-256-GCM, no server, Web Crypto API)

**Body:**
I needed consent forms for freelance work and couldn't find a tool that:
1. Didn't store data on their servers
2. Used real encryption, not just HTTPS
3. Worked offline
4. Had mobile-native UX (most consent tools are desktop-only form builders)

So I built cnsnt.

**Technical overview:**

All crypto uses the Web Crypto API (browser-native, no third-party crypto libraries):
- Key derivation: PBKDF2, SHA-256, 100,000 iterations
- Encryption: AES-256-GCM with random 96-bit IV per record
- Integrity: HMAC-SHA-256 on each stored record
- Export format: JSON with encrypted fields + integrity hash

Zero server. No accounts. No telemetry. Data lives in localStorage (web) or AsyncStorage (native).

**Consent templates included:**
- Photography (model release, minor release)
- Creative services (general freelance scope)
- Medical/health (general wellness, not clinical)
- Relationship/personal (documented mutual agreements)
- Event/performance
- Research participant
- Plus 5 custom templates

**Three surfaces:**
- Web PWA (installs offline): cnsnt-web.surge.sh
- macOS desktop (Tauri, 3.4MB DMG): cnsnt-downloads.surge.sh
- iOS/Android native: in App Store review

The Web Crypto implementation was interesting — GCM mode means you get authenticated encryption (AE), which handles both confidentiality and integrity in one primitive. The PBKDF2 iteration count is tunable via settings if you want to trade performance for brute-force resistance.

Source isn't open yet but I'll open the Web Crypto implementation if there's interest — it's clean and the error handling for tampered records is worth sharing.
