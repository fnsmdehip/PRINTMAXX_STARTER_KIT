## Email 2: The Stack (Day 2)

**Subject:** $0 hosting for 140 websites. here's the exact stack.
**Preview text:** surge.sh + vanilla JS + PWAs. the whole setup in 5 minutes.

---

people keep asking how i host 140+ websites for free. here's the entire stack.

**hosting:** surge.sh (free tier, no limits)
**framework:** none. vanilla HTML/CSS/JS.
**apps:** PWAs (progressive web apps, install from browser, work offline)
**deploy command:** `npx surge ./build site-name.surge.sh`

that's it. no webpack. no next.js. no docker. no CI/CD pipeline. no github actions.

**the math that matters:**
- vercel pro: $20/mo per project. 140 projects = $2,800/mo.
- surge free: $0/mo. 140 projects = $0/mo.
- savings: $33,600/yr.

i spend that saved money on nothing because i also spend $0 on everything else.

**why vanilla JS instead of React:**
- react: 40KB just for the framework
- my apps: 55KB total (framework + logic + styles + everything)
- react needs a build step. vanilla JS needs nothing.
- react apps break when the CDN goes down. vanilla JS works offline.

for complex apps (dashboards, real-time data), use react. for habit trackers and simple tools, vanilla JS wins every time.

**try it yourself:**
1. make an index.html file
2. run: `npx surge . my-test-site.surge.sh`
3. your site is live in 3 seconds

reply if you try it. i want to see what you build.
