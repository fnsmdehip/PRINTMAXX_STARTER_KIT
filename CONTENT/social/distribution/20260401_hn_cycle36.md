## POST 1 — Show HN: WalkToUnlock (PRIORITY — queued from cycle 35)

**Title:** Show HN: WalkToUnlock — a PWA that locks your phone until you've walked 500 steps

**Body:**
WalkToUnlock locks your home screen (via a full-screen PWA overlay) until you hit a configurable step count. Default is 500 steps, adjustable.

The mechanic: you open the app, it reads steps from the Health API (iOS) or Step Counter API (Android), and won't show the "unlocked" screen until you've hit your target for the day. It's not technically a lock — you can still swipe out of it — but the friction plus the gamification does something useful for motivation.

No account. No sync. Offline-first. All localStorage. Free.

https://walktounlock-web.surge.sh

It started as a dumb experiment. I use it. It works better than I expected.

Tech: vanilla JS PWA, Web APIs for step counting (falls back to manual entry on desktop).

## POST 2 — Show HN: cnsnt desktop — offline consent form builder with AES-256-GCM (Tauri / macOS)

**Title:** Show HN: cnsnt — offline consent form builder, AES-256 encrypted, Tauri desktop app, 3.4MB

**Body:**
cnsnt is a consent form and document manager that keeps everything local. No server, no account, no cloud unless you opt in.

The architecture:
- Forms stored encrypted (AES-256-GCM, PBKDF2 key derivation with 100K iterations)
- HMAC integrity checks on stored files (detects tampering)
- Backup is just an encrypted file you copy wherever you want
- 11 templates: general consent, photo release, NDA, media release, freelance contracts, medical consent, etc.

Desktop version is Tauri — Rust backend, React frontend, compiles to a 3.4MB DMG. The web version is the same codebase at cnsnt-web.surge.sh.

Downloads page: https://cnsnt-downloads.surge.sh

Built this because the alternatives either require a cloud account or don't encrypt locally. Photographers, therapists, and freelancers collecting sensitive consent forms should own their own data.

Feedback welcome on the crypto implementation specifically — I used standard Web Crypto API primitives but happy to have it reviewed.

**Notes:** Technical HN audience will appreciate the crypto implementation detail. Inviting review signals confidence and gets engagement.
