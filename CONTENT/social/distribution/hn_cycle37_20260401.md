**Title:** Show HN: TruthScope — iPhone physiological stress analyzer (contact PPG + HRV, not a "lie detector")

**Body:**
Built this after doing a deep dive into what iPhone sensors can actually measure that's relevant to deception detection research.

The short technical version: contact PPG via rear camera + flashlight gives you heart rate with >99.6% correlation to ECG gold standard under controlled conditions. From the pulse signal you extract RMSSD, SDNN, LF/HF ratio — the same cardiovascular channels polygraphs measure.

I intentionally didn't ship this as a "lie detector" because that would be overselling. What it does: measures physiological arousal with medical-grade hardware. Deception correlates with arousal. So does anxiety, exercise, coffee, and embarrassment. The honest framing is "stress response analyzer."

Signal processing pipeline:
- 30fps rear camera video capture, torch on
- Red channel extraction per frame (deepest tissue penetration)
- 2nd-order Butterworth bandpass: 0.7-4.0 Hz (42-240 BPM)
- Peak detection via adaptive threshold
- HRV: RMSSD, SDNN, pNN50, LF/HF via FFT

Currently iPhone only (expo-camera + react-native-vision-camera frame processors).

**Title:** Ask HN: What does your 2026 AI tool stack actually look like for solo product development?

**Body:**
I've been tracking this carefully across a large solo dev project. My current stack that actually gets daily use:

- Claude Code: primary code gen + agentic workflows (subscription, not API)
- Gemini Flash: media analysis and classification tasks
- Playwright MCP: browser automation (eliminates need for most web scraping infrastructure)
- EAS Build: React Native iOS builds
- surge.sh: static deployments

What I dropped after trying:
- Cursor: useful but Claude Code outperforms for my agentic use case
- LangChain: abstraction overhead not worth it vs. raw API calls
- Various "AI workflow" platforms: all add latency and cost vs. claude -p in bash

The pattern I'm seeing: subscription-based access to the most capable models beats pay-per-token for high-volume development. The economics flip entirely when you're running 33 agents doing 6AM batch processing daily.

What are other solo devs actually running?
