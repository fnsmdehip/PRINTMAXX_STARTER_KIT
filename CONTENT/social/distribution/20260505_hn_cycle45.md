## SHOW HN 1: TruthScope — Biometric analysis app using real sensor data (no faked readings)

**Title:** Show HN: TruthScope – lie detection via real PPG/voice/CV sensors, not Math.random()
**URL:** https://truthscope.surge.sh

**Submission Text:**

I noticed that every lie detection app I could find uses fake sensor data behind realistic-looking UIs. They generate random percentages with no actual sensor processing.

I built TruthScope to use real sensor pipelines:

- Heart rate: react-native-vision-camera reading actual camera frames → rPPG (photoplethysmography)
- Voice stress: pitch detection via F0 frequency analysis, not volume metering
- Micro-expressions: computer vision per-frame facial analysis

For the browser demo (truthscope.surge.sh), the finger PPG actually reads your heart rate from the camera.

Technical notes:
- Framework: React Native with Expo SDK 54 / native build (not Expo Go — incompatible with Reanimated 4.x)
- Vision: react-native-vision-camera for real frame data
- Voice: react-native-pitch-detector for actual pitch values
- We publish accuracy ranges from published research (Verkruysse et al. 2008 for PPG, etc.) rather than marketing claims
- If a sensor is unavailable, the app shows "SENSOR UNAVAILABLE" instead of a fake reading

The "lie detector" framing is obviously entertainment. The honest positioning is that we measure physiological stress markers — what police interrogators call indicators, not proof.

The code is 17 source files, 8,315 lines. Built in one session with Claude Code.

Happy to discuss the sensor accuracy literature or the technical implementation.

## ASK HN: What's the state of phone-based PPG accuracy in 2026?

**Title:** Ask HN: How accurate is phone camera rPPG (heart rate from camera) actually?
**URL:** (no URL — discussion post)

**Submission Text:**

Building a biometric sensing app and trying to get honest numbers on remote photoplethysmography accuracy.

The foundational paper (Verkruysse et al. 2008) reports good results in controlled conditions. But real-world accuracy drops significantly with:
- Skin tone variation
- Motion artifacts
- Ambient light variation
- Camera hardware differences

The papers I've found cite ranges of 60-85% accuracy for heart rate detection, and much lower for HRV or stress detection.

Has anyone actually benchmarked react-native-vision-camera based rPPG against a reference device? Or found a better approach for stress/arousal detection on mobile hardware?

The alternative is Apple Watch HealthKit data (HR, HRV, SpO2 on Series 6+ / Ultra 2) which is significantly more accurate. But that requires the user to own a Watch.

Specifically interested in: what's the most honest way to communicate accuracy to users so they understand what they're actually measuring?
