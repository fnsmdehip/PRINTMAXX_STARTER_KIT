4/ filesystem as memory: agents don't share context windows. they share files. agent A writes to ALPHA_STAGING.csv. agent B reads ALPHA_STAGING.csv. the filesystem IS the shared memory. no message passing. no coordination protocol. just files.

5/ quality control: every agent output gets audited. my audit hit rate: 70% fully delivered, 15% partial, 15% missing. the 30% that fails is predictable: deployment (needs credentials), judgment calls (needs human), and multi-step reasoning (agents lose track).

6/ the overnight pattern: static prompt file > reads state from disk > does ONE task > writes state to disk > exits. next iteration: fresh context > reads updated state > does next task. memory lives in the filesystem, not the context window.

7/ the real skill: it's not prompt engineering. it's task decomposition. knowing what can run in parallel, what's blocking what, and when to intervene vs when to let the agent figure it out. project management for silicon.
