quick start

# install
pip install sovrun
# or from source:
git clone https://github.com/fnsmdehip/sovrun.git
cd sovrun && pip install -e .

# set project root (all paths relative to this)
export SOVRUN_ROOT=/path/to/your/project
mkdir -p data output state logs

# import your conversation history
sovrun-import --provider claude --dir ~/.claude/projects

# build voice model
sovrun-voice --extract

# build cognition model (correction chains + meta-rules)
sovrun-engine --build-model

# mine patterns (what frustrates you, what works)
sovrun-miner --mine

# set up MCP connectors
sovrun-mcp --setup slack github

# close all loops
sovrun-loops --cycle

# audit the system
sovrun-audit --audit

memory import

import your conversation history from any LLM provider. sovrun normalizes all formats into its internal JSONL format for voice extraction and correction chain analysis.

ChatGPT

# export from ChatGPT settings -> Data controls -> Export data
# you'll get a zip file with conversations.json
sovrun-import --provider chatgpt --file ~/Downloads/chatgpt-export.zip

Claude

# point at your Claude projects directory
sovrun-import --provider claude --dir ~/.claude/projects

# or a specific transcript file
sovrun-import --provider claude --file transcript.jsonl

Gemini

# export from Google Takeout -> Gemini
sovrun-import --provider gemini --file ~/Downloads/gemini-export.json

after import, run sovrun-voice --extract to build your voice model from the combined history.

cognitive layer (7 modules)

voice_extractor

reads your prompt history (JSONL), identifies vocabulary, tone, banned words, sentence patterns. outputs a voice model JSON you inject into any agent prompt.

sovrun-voice --extract     # build model from prompts
sovrun-voice --inject      # get injection string for agent prompts
sovrun-voice --status      # model stats
sovrun-voice --diff        # compare current vs previous model

cognitive_engine

identifies correction chains in your history. builds meta-rules that would have prevented each correction sequence. from production: 168 chains, 6.9 average corrections per chain.

sovrun-engine --build-model     # full cognition model
sovrun-engine --lookup "TASK"    # find similar past tasks
sovrun-engine --rules            # show extracted meta-rules
sovrun-engine --chain-analysis   # correction chain stats

pattern_miner

finds correction markers ("no not that"), escalation triggers ("deeper", "surprise me"), and satisfaction signals ("perfect", "ship it"). mines into transferable rules.

sovrun-miner --mine          # extract all patterns
sovrun-miner --similar "Q"   # find similar past prompts
sovrun-miner --corrections   # correction sequences only
sovrun-miner --effective     # satisfaction triggers only

user_sim_refiner

autonomous critique loop. takes your extracted cognitive architecture and simulates what you would criticize about a project. outputs critique for another agent to act on.

# in Python:
from sovrun.core.user_sim_refiner import UserSimRefiner
refiner = UserSimRefiner()
critique = refiner.simulate_critique("path/to/project")

self_audit

checks your project for AI slop vocabulary in config files, context bloat wasting tokens, missing protocols, orphan scripts not wired into any automation.

sovrun-audit --audit    # full audit
sovrun-audit --report   # latest findings

bias_null (template)

5-point pre-output filter. run before any major output. catches: legacy smuggling, preemptive appeasement, lived-gap check, popular-default trap, training-bias correction.

ships as a template at templates/bias-null.md. inject into agent system prompts.

decision_engine

closed-loop: scan data sources, score opportunities, take action (within safety limits), log every decision with reasoning to CSV. supports rule-based and LLM-delegated judgment.

sovrun-loops --cycle      # includes decision engine
# or standalone:
python3 -m sovrun.core.decision_engine --cycle
python3 -m sovrun.core.decision_engine --dry-run

orchestration layer (4 modules)

orchestration

compose agents into DAGs using three primitives: SequentialAgent, ParallelAgent, LoopAgent. non-dependent phases run in parallel. typed state management.

sovrun-orchestrate --visualize   # render DAG as HTML
sovrun-orchestrate --status      # execution status

# Python:
from sovrun.core.orchestration import DAGOrchestrator, SequentialAgent, ParallelAgent
dag = DAGOrchestrator()
dag.add_phase("research", ParallelAgent([agent_a, agent_b]))
dag.add_phase("synthesize", SequentialAgent([agent_c]), depends_on=["research"])
dag.run()

handoff

typed agent-to-agent delegation. agent A invokes agent B mid-task, passes context, receives result, continues. scoped guardrails per handoff.

sovrun-handoff --from agent_a --to agent_b --task "analyze data"

# Python:
from sovrun.core.handoff import Handoff
result = Handoff.delegate(
    source="researcher",
    target="writer",
    context={"findings": data},
    guardrails=["no_external_calls"]
)

loop_closer

closes 4 open loops: decision execution, feedback tracking, pipeline advancement, soul drift scoring (0-10 per output against SOUL.md directives).

sovrun-loops --cycle      # run all 4 loops
sovrun-loops --drift      # soul drift scoring only
sovrun-loops --status     # loop health
sovrun-loops --dry-run    # preview without executing

workflow_bridge

agents create n8n workflows programmatically. define triggers, nodes, connections in Python. export as n8n-compatible JSON for import.

sovrun-workflow --create "slack message to google sheets row"
sovrun-workflow --list       # list created workflows
sovrun-workflow --export id  # export as n8n JSON

memory layer (4 modules)

procedural_memory

auto-generates skill docs from successful agent runs. FTS5-indexed (SQLite full-text search). before each run, queries for relevant past solutions.

sovrun-memory --consolidate   # consolidate new skills
sovrun-memory --search "task"  # search past solutions
sovrun-memory --stats          # skill count and categories

# Python:
from sovrun.core.procedural_memory import ProceduralMemory
mem = ProceduralMemory(db_path="skills.db")
results = mem.search("deploy to vercel")
mem.close()

memory_import

see memory import section above.

conversation_logger

extracts user and assistant messages from Claude Code session transcripts into searchable JSONL. incremental processing (tracks byte offsets).

# CLI:
python3 -m sovrun.core.conversation_logger --extract
python3 -m sovrun.core.conversation_logger --search "deployment"
python3 -m sovrun.core.conversation_logger --stats
python3 -m sovrun.core.conversation_logger --recent 20

session_briefing

"what happened since last session" from git diffs, state files, task trackers. no LLM calls. pure file reading. under 30 seconds.

sovrun-briefing --save    # save briefing to file
sovrun-briefing --json    # JSON output

infrastructure layer (3 modules)

resilience

shared module for production-grade agents: retry with exponential backoff + jitter, file locking (fcntl on unix), circuit breaker (CLOSED/OPEN/HALF_OPEN), input sanitization, loop detection.

from sovrun.core.resilience import retry, locked_file, CircuitBreaker
from sovrun.core.resilience import sanitize_for_prompt, TrajectoryLogger, LoopDetector

durable

deterministic replay on crash. each step logged atomically. crash at step 14 replays from step 14, not step 1. zero wasted API calls on recovery.

sovrun-durable --status    # show checkpoint state
sovrun-durable --replay    # replay from last checkpoint

# Python:
from sovrun.core.durable import DurableExecution
with DurableExecution("my_pipeline") as dur:
    result_1 = dur.step("fetch_data", fetch_function)
    result_2 = dur.step("process", process_function, result_1)
    # crash here? next run replays from step 3, not step 1

tracing

HTML timeline of LLM calls, tool executions, handoffs, failures. per-agent token attribution and cost tracking.

sovrun-trace --visualize   # generate HTML timeline
sovrun-trace --cost         # token cost breakdown by agent

connectors (105 MCP servers)

105 pre-configured MCP server definitions across 18 categories: communication, data, social, dev, commerce, AI, storage, CRM, marketing, scraping, search, monitoring, productivity, finance, media, analytics, support, infrastructure.

# list all available connectors
sovrun-mcp --list

# set up a connector (adds to claude_desktop_config.json)
sovrun-mcp --setup slack
# prompts for required env vars: SLACK_BOT_TOKEN, SLACK_TEAM_ID

# set up multiple at once
sovrun-mcp --setup slack github notion postgres stripe

# search connectors by category
sovrun-mcp --category communication
sovrun-mcp --category data

each connector definition includes: MCP server command, required environment variables, capabilities list, and setup instructions. the registry is at connectors/registry.json.

data format

sovrun reads JSONL files. each line is a JSON object with at minimum:

{"ts": "2026-03-15T14:30:00", "prompt": "your prompt text here"}

optional fields: role (user/assistant), session_id, content_length. you can use content or text instead of prompt.

templates

sovrun ships with 4 templates you customize for your system:

templatepurpose
SOUL.mdagent identity, behavioral directives, anti-patterns. the soul drift scorer reads this.
bias-null.md5-point pre-output filter. inject into agent system prompts.
voice-config.jsonexample voice model structure. voice_extractor generates the real one.
CLAUDE.mdtemplate for wiring sovrun into your project's system instructions.
