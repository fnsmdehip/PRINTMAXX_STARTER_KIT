## What

Build a minimal Bluesky post scheduler + analytics dashboard. Bluesky has ~35M users as of May 2026 and ZERO polished paid scheduling tools. The gap is 18-24 months behind where X tooling was in 2020. First mover window is now.

## Why

- Bluesky AT Protocol has a clean public API with free write access
- No auth walls -- any developer can build without applying for API access
- Existing scheduling tools (Buffer, Later, Hootsuite) have NOT prioritized Bluesky
- The creator segment on Bluesky (journalists, indie developers, researchers) has budget
- Buffer charges $18/mo for multi-platform scheduling -- Bluesky-native tool can undercut at $9/mo
- Tech is a weekend build: SQLite queue + cron + AT Protocol Python SDK

## How

**Step 1 (Day 1):** Build core scheduler in Python
- `atproto` Python library (pip install atproto) -- already wraps Bluesky's AT Protocol
- SQLite table: `scheduled_posts(id, handle, app_password, post_text, schedule_time, sent_at)`
- Cron worker: every minute, check for due posts, call `client.send_post()`
- No frontend yet -- launch as a "Twitter DM your post" style service first

**Step 2 (Day 2):** Minimal web UI
- Next.js single-page: login with Bluesky handle + app password, write post, pick time
- Stripe Payment Link: $9/mo for 50 scheduled posts, $19/mo for 500
- Deploy on Railway (free tier handles initial load)

**Step 3 (Launch, Day 3):** Pre-sell before building analytics
- Post on Bluesky itself to the `#indiedev` and `#solopreneur` feeds
- Post in r/Bluesky and r/SideProject
- No ads needed -- organic reach within Bluesky community is high for native tools

**Analytics add-on (Week 2):**
- Pull post engagement via AT Protocol `getPostThread` -- impressions, replies, reposts
- Weekly digest email via Resend (free tier: 3K emails/mo)
