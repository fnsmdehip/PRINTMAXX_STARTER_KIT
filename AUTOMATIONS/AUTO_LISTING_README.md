# Auto-List Products on Marketplaces

Browser automation script that reads ready-to-list product data from `PRODUCTS/` and fills marketplace listing forms via Playwright.

**Semi-automated:** Fills forms automatically, pauses for you to verify and click publish.

## Supported Platforms

| Platform | Source File | Products |
|----------|-----------|----------|
| Gumroad | `PRODUCTS/GUMROAD_READY_LISTINGS.md` | 10 digital products |
| Etsy | `PRODUCTS/ETSY_LISTINGS_20.md` | 20 POD/digital listings |
| Redbubble | `PRODUCTS/REDBUBBLE_LISTINGS.md` | 20 POD designs |
| Fiverr | `OPS/FIVERR_LAUNCH_PACKAGE.md` | Service gigs |
| eBay | `output/ecom_autopilot/queues/ebay_queue.csv` | Dynamic arb winners |

## Quick Start

```bash
# 1. List what would be posted (no browser, just parse)
python3 AUTOMATIONS/auto_list_products.py --platform gumroad --list

# 2. Dry run (shows what would happen, no browser launched)
python3 AUTOMATIONS/auto_list_products.py --platform etsy --dry-run

# 3. Live listing (opens browser, fills forms, you verify + publish)
python3 AUTOMATIONS/auto_list_products.py --platform gumroad

# 4. Batch control (start from listing 3, do 5 at a time)
python3 AUTOMATIONS/auto_list_products.py --platform redbubble --start 3 --count 5

# 5. eBay winner queue (generated by ecom_autopilot)
python3 AUTOMATIONS/ecom_autopilot.py --tick
python3 AUTOMATIONS/auto_list_products.py --platform ebay --list
```

## How It Works

1. Parses product data from the markdown source file (names, prices, descriptions, tags)
2. Opens Chromium browser via Playwright
3. Navigates to the platform's listing page
4. Pauses so you can log in manually
5. For each product: fills form fields, then pauses for you to review and click publish
6. Tracks progress in `AUTOMATIONS/logs/listing_progress.json` (skips already-listed)
7. Logs every action to `AUTOMATIONS/logs/listing_log.csv`

## Flags

| Flag | Description |
|------|-------------|
| `--platform` | Required. One of: gumroad, etsy, redbubble, fiverr, ebay |
| `--list` | Parse and display products, then exit. No browser. |
| `--dry-run` | Show what would happen without launching a browser |
| `--start N` | Start from listing number N (1-indexed, default 1) |
| `--count N` | Process N listings (default 0 = all) |
| `--headless` | Run browser headless (not recommended, you need to verify) |

## Requirements

- Python 3.9+
- Playwright: `pip install playwright && playwright install chromium`
- Product markdown files in `PRODUCTS/` (already exist)

## Progress Tracking

The script saves progress to `AUTOMATIONS/logs/listing_progress.json`. If you stop mid-batch and restart, it skips products already listed. To re-list something, remove it from the JSON file.

Every action is logged to `AUTOMATIONS/logs/listing_log.csv` with timestamp, platform, product name, action, and status.

## Error Recovery

- Each listing gets 2 retries on failure
- Human-like random delays (2-5s) between actions to avoid detection
- If a form field selector fails, it tries 4-5 alternative selectors
- Progress saves after each successful listing (resume-safe)

## Workflow

```
1. Create accounts first (see OPS/TWITTER_ACCOUNT_QUICK_SHEETS/)
2. Run --list to verify parsing looks correct
3. Run --dry-run to preview the batch
4. Run live mode, log in when prompted
5. Review each form fill, click publish manually
6. Check logs/listing_log.csv for results
```
