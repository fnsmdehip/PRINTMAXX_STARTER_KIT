The Core Insight

Claude Code's context window is 200K tokens. That's enough to hold an entire codebase, make decisions, and execute. But it's temporary. When the session ends, everything is gone.

The breakthrough: filesystem = memory, context = disposable scratchpad.

Every agent reads state from files, does ONE task, writes results back, and exits. The next iteration starts fresh with full context. This means:

  • No context window crashes (the #1 killer of long-running AI agents)
  • Perfect crash recovery (state is always on disk)
  • Unlimited runtime (each iteration is independent)
  • Git-backed memory (every state change is versioned)

System Architecture

The 4-Layer Stack

Layer 1: Execution (launchd + cron)
macOS launchd plists and crontab entries trigger agents on schedule. Each agent is a Python script that runs, does work, and exits. No long-running processes to crash.
CORE
Layer 2: Orchestration (CEO Agent + Swarm Brain)
The CEO agent runs a 16-phase cycle every 6 hours. The swarm brain (every 4h) reads all agent outputs and makes strategic decisions. Together they coordinate 33 agents without conflicts.
OPUS
Layer 3: Intelligence (Router + Alpha Index)
SQLite FTS5 index over 15,000+ alpha entries. Intelligence router aggregates alpha, strategy docs, swarm reports, and method CSVs into a single brief per venture+task. Every agent queries before acting.
META
Layer 4: Memory (Filesystem + Git)
All state stored in JSON/JSONL files. Git commits every state change. Subconscious memory system extracts key learnings at session end. Memories persist across sessions via injection at startup.
PERSISTENT

Key Architectural Decisions

Why filesystem over database? Claude Code can read/write files natively. No ORM, no connection pooling, no schema migrations. Files are human-readable, git-versioned, and work offline. SQLite only used for full-text search where file scanning is too slow (15K+ entries).

Inter-Agent Communication

Agents communicate through a shared message bus (message_bus.jsonl). Append-only JSONL format ensures no write conflicts. Each message has:

{
  "from": "gap_hunter",
  "to": "asset_deployer",
  "type": "DEPLOY_REQUEST",
  "payload": {"asset": "landing-page-v2", "priority": "HIGH"},

Checkpoint-Resume Pattern

# CEO agent saves progress after each phase
def save_checkpoint(phase_num, state):
    checkpoint = {
        "phase": phase_num,
        "timestamp": datetime.now().isoformat(),
        "state": state,
        "completed_phases": list(range(1, phase_num + 1))
    }
    Path("ceo_state.json").write_text(json.dumps(checkpoint))

# On startup, resume from last checkpoint
def load_checkpoint():
    state = json.loads(Path("ceo_state.json").read_text())
    return state["phase"]  # Resume from here

Ralph Loops: Infinite Agent Execution

The core pattern that makes everything work. Named after the concept of recursive autonomous loops with persistent filesystem memory.

The Pattern

while true; do
  cat PROMPT.md | claude --dangerously-skip-permissions --print
done

That's it. That's the core. But the magic is in how PROMPT.md is structured:

PROMPT.md Anatomy

# Role
You are the [VENTURE] agent. Working directory: /path/to/project

# State (read every iteration)
Read `.ralph/progress.md` for what you did last.
Read `.ralph/state.json` for current metrics.

# Task
Do exactly ONE of these (pick highest priority):
1. [First priority task]
2. [Second priority task]
3. [Third priority task]

# Rules
- Do ONE task per iteration, then EXIT
- Write results to `.ralph/progress.md`
- Update `.ralph/state.json` with new metrics
- If blocked, write blocker to `.ralph/blockers.md` and move to next task
- NEVER loop within a single iteration
- Total runtime per iteration: max 10 minutes

# Context
[Paste relevant state files, recent outputs, etc.]
Why this works: Each iteration gets a fresh 200K context window. No degradation over time. The filesystem IS the memory. Git IS the version control. The loop IS the scheduler. Zero infrastructure beyond Claude Code and bash.

Ralph Loop Best Practices

PracticeWhy
ONE task per iterationPrevents context window bloat. Clean execution.
Max 10 min per iterationPrevents runaway loops. Keeps costs predictable.
State in JSON filesMachine-readable. Easy to parse. Git-friendly.
Progress in markdownHuman-readable audit trail. Debug-friendly.
Blockers filePrevents infinite retry on impossible tasks.
Exit codes0 = success, 1 = error, 2 = blocked, 3 = nothing to do.

Template 1: Basic Agent Script

#!/usr/bin/env python3
"""Template: Basic autonomous agent with resilience."""
import json
from pathlib import Path
from datetime import datetime

STATE_FILE = Path(__file__).parent / "agent_state.json"
LOG_FILE = Path(__file__).parent / "agent.log"

def load_state():
    if STATE_FILE.exists():
        return json.loads(STATE_FILE.read_text())
    return {"cycles": 0, "last_run": None, "results": []}

def save_state(state):
    state["last_run"] = datetime.now().isoformat()
    state["cycles"] += 1
    STATE_FILE.write_text(json.dumps(state, indent=2))

def log(msg):
    with open(LOG_FILE, "a") as f:
        f.write(f"[{datetime.now().isoformat()}] {msg}\n")

def main():
    state = load_state()
    log(f"Cycle {state['cycles'] + 1} starting")

    # YOUR AGENT LOGIC HERE
    # 1. Read inputs (files, APIs, scraped data)
    # 2. Process / analyze / decide
    # 3. Take action (write files, deploy, send)
    # 4. Log results

    result = {"action": "example", "status": "success"}
    state["results"].append(result)
    save_state(state)
    log(f"Cycle complete: {result}")

if __name__ == "__main__":
    main()

Template 6: Message Bus Communication

import json
from datetime import datetime
from pathlib import Path

BUS_FILE = Path("message_bus.jsonl")

def send_message(from_agent, to_agent, msg_type, payload):
    """Append message to the shared bus."""
    message = {
        "from": from_agent,
        "to": to_agent,
        "type": msg_type,
        "payload": payload,
        "timestamp": datetime.now().isoformat(),
        "read": False
    }
    with open(BUS_FILE, "a") as f:
        f.write(json.dumps(message) + "\n")

def read_messages(agent_name, msg_type=None):
    """Read unread messages for this agent."""
    if not BUS_FILE.exists():
        return []
    messages = []
    for line in BUS_FILE.read_text().strip().split("\n"):
        if not line:
            continue
        msg = json.loads(line)
        if msg["to"] == agent_name and not msg["read"]:
            if msg_type is None or msg["type"] == msg_type:
                messages.append(msg)
    return messages
