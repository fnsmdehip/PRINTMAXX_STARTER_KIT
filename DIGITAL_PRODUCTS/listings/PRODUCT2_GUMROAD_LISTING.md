**the freemium trap**

- the math: 100K downloads at 2% conversion vs 100K downloads at 15% trial-to-paid
- real revenue comparison across 5 app categories
- when freemium actually makes sense (hint: almost never for indie devs)

**6-screen onboarding flow**
- screen 1: personalization question (creates investment)
- screen 2: goal selection (triggers commitment)
- screen 3: progress preview (shows the future state)
- screen 4: social proof (real numbers, not "thousands of users")
- screen 5: feature showcase (3 features max, benefit-first)
- screen 6: paywall with animated elements
- per-niche scripts for fitness, faith, productivity, finance

**animated paywall spec**
- static paywalls: 4.2% conversion
- animated paywalls: 12.1% conversion (2.9x lift)
- exact animation specs (Lottie, timing, easing curves)
- pricing layout: annual highlighted, monthly as anchor, weekly as decoy
- strike-through pricing: 74% of top apps use this. now you know the spec.

**RevenueCat A/B testing**
- test 1 (week 1-2): trial length (3-day vs 7-day)
- test 2 (week 3-4): pricing ($4.99/wk vs $9.99/mo vs $49.99/yr)
- test 3 (week 5-6): paywall layout (feature list vs social proof)
- test 4 (week 7-8): onboarding length (4 screens vs 6 screens)
- statistical significance calculator included

**warm-hard escalation model**
- day 0-3: full access, track engagement
- day 3: soft paywall (premium badge on 2 features)
- day 7: medium paywall (core features locked)
- day 14: hard paywall (full lock, conversion peak)
- recovery sequence for churned users

**code examples**
- TypeScript paywall component (React Native)
- RevenueCat integration
- A/B test flag setup
- analytics event tracking
- onboarding flow state machine
