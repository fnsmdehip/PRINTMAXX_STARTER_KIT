## Framework

### Phase 0: Audience Building (6 months, prerequisite)
**Build 5-8K X followers first.**

Content cadence: 1 tweet/day + 1 thread/week
Niche: Your target market (developers, founders, indie hackers)
Style: Tactical how-tos, build-in-public, contrarian takes

### Phase 1: Rapid Build (48hrs)
**Use vibe coding + Claude + Gemini Flash**

Example stack:
- Frontend: Next.js + Tailwind (2 hours)
- Backend: Supabase or Firebase (1 hour)
- Core feature: Single use case, ultra-focused (8 hours)
- Polish: Minimal but functional (2 hours)
- Deploy: Vercel (30 minutes)

**Critical:** Ship ugly but working. Don't polish.

---

### Phase 2: X Launch (Day 3)

**Post Template:**
```
Building [tool name].

[Problem it solves in 1 sentence].

Reply 'EARLY' for 50% off presale.

[Demo video or screenshot]
```

**Comment hook strategy (ALPHA049):**
- Comments = algorithm boost (10x reach vs likes)
- Every reply = notification to replier's audience
- "Reply X" creates engagement loop

**Follow-up in comments:**
```
Presale is $49 (50% off $99 launch price).

Stripe link: [link]

Shipping in 2 weeks. Full refund if not happy.
```

---

### Phase 3: Validation Gate (Day 3-7)

**Decision Matrix:**

| Presales | Action |
|----------|--------|
| 50+ | FINISH PRODUCT (validated) |
| 20-49 | CONDITIONAL (ask why, iterate) |
| 10-19 | PIVOT (wrong audience or positioning) |
| <10 | KILL (move to next idea) |

**Kill fast.** Most ideas fail. That's fine. You spent 3 days, not 3 months.

---

### Phase 4: Build to Finish (Week 2-3, if validated)

**Only if 50+ presales.**

Prioritize:
1. Core feature completion
2. Payment integration (Stripe)
3. Onboarding flow
4. Email for presale customers: "It's ready"

Ship in 10-14 days. No longer.
