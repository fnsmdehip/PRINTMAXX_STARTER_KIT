Common questions

Things people ask before installing their first agent.

How is this different from hiring a Zapier consultant? +

A Zapier consultant connects apps together. When the trigger fires, the workflow runs. Our agents don't wait for triggers. They wake up on a schedule, assess their environment, make decisions, take action, and log results. An AI receptionist doesn't just route calls. It has a conversation, qualifies the caller, checks your calendar, and books the appointment. That's the difference between a workflow and an agent.

What happens if the AI makes a mistake? +

Every agent has guardrails and escalation paths. The AI receptionist transfers calls it can't handle. The support agent escalates tickets that need human review. The lead machine flags edge cases for your team. We build confidence thresholds into every agent. Below the threshold, it routes to a human. Above it, it acts and logs what it did so you can review.

Do we need to change our existing tools? +

No. Our agents connect to your existing stack. If you use HubSpot, the agent logs to HubSpot. If you use Google Calendar, it books there. If you use Slack, it sends digests to Slack. We adapt to your tools. The only new infrastructure is the AI agent itself and the orchestration layer that powers it.

What does "self-healing" mean? +

If an API goes down, the agent retries with exponential backoff. If retries fail, it switches to a fallback path. If the fallback fails, it pauses that function, sends an alert, and continues operating on everything else. When the API comes back, the agent automatically resumes. You never have to restart anything manually. This is circuit breaker architecture, the same pattern Netflix and Amazon use.

How fast until we see results? +

Signal map takes 3-5 days. Agent install takes 10-14 days. Your agent is live and working within 3 weeks of signing. Most clients see measurable impact in the first week of the agent running. AI receptionist clients typically see 40-60% fewer missed calls in week one.
