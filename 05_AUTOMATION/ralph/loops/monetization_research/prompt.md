# Monetization Research Loop

## Context
You are scanning for new monetization tactics, pricing strategies, and payment innovations. Focus on what converts and maximizes revenue.

## Task
1. Check state.md for the next uncompleted category
2. Research that category using web search and available sources
3. Write findings to disk at the specified output location
4. Mark the category as complete in state.md
5. Output `<promise>COMPLETE</promise>` when all categories done

## Sources to Scan
- Stripe/RevenueCat changelogs
- @levelsio - Revenue optimization
- @dannypostmaa - Honest revenue data
- Gumroad trending products
- r/SaaS pricing discussions
- IndieHackers revenue milestones

## Research Categories
1. **Payment methods** - New payment options, crypto, regional methods
2. **Pricing strategies** - A/B test results, price anchoring, bundles
3. **Subscription models** - Trial lengths, annual discounts, usage-based
4. **Upsell flows** - Proven upsell sequences, order bumps
5. **Affiliate tactics** - High-converting programs, commission structures
6. **Info product pricing** - What price points convert by category

## Output Format
For each opportunity found, create a file at:
`LEDGER/MONETIZATION_OPPORTUNITIES/[date]_[tactic_type]_[name].md`

Include:
- Monetization tactic
- Specific numbers (conversion rates, revenue lift)
- Implementation requirements
- Tools needed
- Example/case study
- Applicable money methods

## Quality Filter
APPROVE if:
- Has specific conversion or revenue data
- From proven source
- Applicable to our stack

DIG DEEPER if:
- Claims high conversion without proof
- Industry-specific (may not transfer)
