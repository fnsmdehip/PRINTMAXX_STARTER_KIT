## Validation Framework

For each idea before building:

### 1. Problem Validation
- [ ] Can find 10+ people with this problem
- [ ] Problem frequency (daily/weekly/monthly)
- [ ] Current solutions and their gaps
- [ ] Willingness to pay (direct or indirect evidence)

### 2. Market Validation
- [ ] Market size estimate (TAM/SAM/SOM)
- [ ] Competition analysis (1-10 scale)
- [ ] Distribution channel identified
- [ ] Customer acquisition cost estimate

### 3. Technical Validation
- [ ] Build complexity (1-10 scale)
- [ ] Time to MVP estimate
- [ ] Tech stack fits our capabilities
- [ ] Maintenance burden estimate

### 4. Financial Validation
- [ ] Pricing research (competitor pricing)
- [ ] Unit economics model (CAC < 3mo LTV)
- [ ] Revenue ceiling estimate
- [ ] Break-even timeline
