## Part 4: Implementation Guide

### iOS rules for external links

Apple allows external purchase links in certain conditions as of 2024. Follow these rules:

**What's allowed:**
- Links to external websites for affiliate purchases
- Opening Safari for product pages
- Email with affiliate recommendations
- Push notifications with product suggestions (if user opted in)

**What's NOT allowed:**
- Bypassing App Store for in-app purchases
- Directing to competing payment systems for app features
- Deceptive linking (must be clear user is leaving app)

### Code implementation

**Opening affiliate links (Swift/iOS)**
```swift
// SafariServices for in-app browser (better UX)
import SafariServices

func openAffiliateLink(_ urlString: String) {
    guard let url = URL(string: urlString) else { return }

    // Track click before opening
    Analytics.log(event: "affiliate_click", parameters: [
        "url": urlString,
        "product_category": "supplements"
    ])

    // Open in Safari View Controller
    let safariVC = SFSafariViewController(url: url)
    present(safariVC, animated: true)
}

// Alternative: Open in external Safari
func openExternalAffiliateLink(_ urlString: String) {
    guard let url = URL(string: urlString) else { return }
    UIApplication.shared.open(url)
}
```

**React Native implementation**
```javascript
import { Linking } from 'react-native';
import analytics from '@react-native-firebase/analytics';

const openAffiliateLink = async (url, productCategory) => {
  // Track the click
  await analytics().logEvent('affiliate_click', {
    url: url,
    product_category: productCategory,
  });

  // Check if URL can be opened
  const supported = await Linking.canOpenURL(url);
  if (supported) {
    await Linking.openURL(url);
  }
};
```

### Deep linking setup

Most affiliate programs provide unique tracking IDs. Structure your links:

```
Base format:
https://affiliate-network.com/track?ref=YOURCODE&product=PRODUCTID

Amazon format:
https://amazon.com/dp/ASIN?tag=your-tracking-id

iHerb format:
https://iherb.com/product?rcode=YOURCODE
```

**Store affiliate links in config (not hardcoded):**
```javascript
// affiliate-config.js
export const AFFILIATE_LINKS = {
```

## Part 5: FTC Disclosure Requirements

### In-app disclosure (required)

**Near every affiliate link:**
```
"This is an affiliate link. We may earn a commission at no extra cost to you."
```

**Short version (space-constrained):**
```
"Affiliate link"
```

**Resources section header:**
```
"Disclosure: Some links below are affiliate links. We earn a commission if you purchase, at no extra cost to you. We only recommend products we believe in."
```

### Email disclosure (required)

**Email footer:**
```
"Disclosure: This email contains affiliate links. If you purchase through our links, we may earn a commission at no additional cost to you."
```

### App Store compliance

Add to app description or terms:
```
"This app may contain links to third-party products and services. We may receive compensation for purchases made through these links."
```

### What NOT to do

- Never hide affiliate status
- Never claim products are "free" when you earn commission
- Never make unsubstantiated health claims
- Never imply endorsement from brands without permission
- Never use fake reviews or testimonials
