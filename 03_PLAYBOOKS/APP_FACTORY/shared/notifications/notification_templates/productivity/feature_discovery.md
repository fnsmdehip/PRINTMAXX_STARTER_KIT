# Feature discovery notifications

Productivity apps: StudyLock, PromptVault

Educate users about features they haven't used yet.

---

## Timing rules

1. Only after user has used app for 3+ days
2. Max 1 feature discovery notification per week
3. Only features relevant to user's behavior
4. Never during active sessions
5. Send at 10 AM or 2 PM (low-activity times)

---

## StudyLock features

**Study groups**
- Title: "Study with others"
- Body: "Join a study group for accountability."
- Trigger: User studied alone 7+ days

**Subject tracking**
- Title: "Track subjects"
- Body: "See time spent on each subject."
- Trigger: User hasn't set any subjects

**Focus sounds**
- Title: "Focus sounds"
- Body: "Try ambient noise during sessions."
- Trigger: User hasn't tried sounds

**Pomodoro mode**
- Title: "Try Pomodoro"
- Body: "25 min on, 5 min off. Built-in breaks."
- Trigger: User does long sessions (60+ min)

**Widgets**
- Title: "Home screen widget"
- Body: "See your stats at a glance."
- Trigger: After 14 days of use

## PromptVault features

**Prompt testing**

## Discovery sequence

Week 1: Core features only (let user settle in)
Week 2: Feature #1 (most valuable)
Week 3: No discovery notification
Week 4: Feature #2
Week 5+: Only if user hasn't discovered naturally

---

## A/B test ideas

1. Question vs statement titles
2. Feature name vs benefit in body
3. Morning vs afternoon timing
4. With screenshot vs without
5. Deep link to feature vs general app open
