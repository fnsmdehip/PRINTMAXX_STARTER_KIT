Ready-to-test call-to-action copy, design, and placement variations.

## CTA Copy Tests

### Test 1: Action Verb Comparison

**Hypothesis:** Specific action verbs outperform generic ones.

| Variant | CTA Copy | Psychology |
|---------|----------|------------|
| A | Get Started | Generic, low commitment |
| B | Start Free Trial | Specific, emphasizes free |
| C | Create My Account | Ownership language |
| D | Try It Free | Action + benefit |

---

### Test 2: Outcome vs Action

**Hypothesis:** Outcome-focused CTAs convert better than action-focused.

**Variant A: Action-Focused**
```
[Sign Up Now]
```

**Variant B: Outcome-Focused**
```
[Start Getting Matches]
[Begin My Transformation]
[Write My First Post]
```

**Variant C: Benefit-Focused**
```
[Save 5 Hours This Week]
[Get More Dates]
[Lose 10 Pounds]
```

---

### Test 3: Personalization

**Hypothesis:** First-person ("my") outperforms second-person ("your").

| Variant | CTA Copy | Framing |
|---------|----------|---------|
| A | Start Your Trial | Second person |
| B | Start My Trial | First person |
| C | Start Free Trial | Neutral |

## CTA Design Tests

## CTA Placement Tests

### Test 10: Position on Screen

**Hypothesis:** Sticky CTA increases conversion without hurting UX.

**Variant A: Inline Only**
```
CTA appears at natural decision points
Scrolls with content
```

**Variant B: Sticky Bottom**
```
Fixed CTA bar at bottom of screen
Always visible
```

**Variant C: Appear on Scroll**
```
CTA hidden initially
Appears after 50% scroll
Sticky once visible
```

**Variant D: Multiple Inline**
```
CTA after each section
No sticky element
```

---

### Test 11: Above vs Below Fold

**Hypothesis:** Above-fold CTA captures immediate intent.

**Variant A: Above Fold Only**
```
CTA in hero section
Nothing below fold
```

**Variant B: Below Fold Only**
```
Content first
CTA after value explanation
```

**Variant C: Both**
```
CTA above fold
CTA below fold (after content)
```

---

No alternatives

**Variant B: Primary + Secondary**
```
[Get Started] (primary)
[Learn More] (secondary/text link)
```

**Variant C: Multiple Options**
```
[Start Free Trial]
[See Pricing]
[Watch Demo]
```

## Implementation Templates

### Button Component Structure

```jsx
// Example React Native component
<TouchableOpacity
  style={[
    styles.button,
    variant === 'primary' && styles.primaryButton,
    variant === 'secondary' && styles.secondaryButton,
    size === 'large' && styles.largeButton,
    fullWidth && styles.fullWidthButton,
  ]}
  onPress={onPress}
  activeOpacity={0.8}
>
  {leftIcon && <Icon name={leftIcon} />}
  <Text style={styles.buttonText}>{text}</Text>
  {rightIcon && <Icon name={rightIcon} />}
</TouchableOpacity>
```

## Measurement Framework

### Primary Metrics
- **Tap rate:** CTA views to taps
- **Conversion rate:** Taps to completed action

### Secondary Metrics
- Time to tap
- Scroll depth before tap
- Bounce rate
- Return visitor tap rate

### Segment Analysis
- New vs returning visitors
- Traffic source
- Device type
- Time of day

## Quick Reference: CTA Best Practices

### High-Converting CTA Formulas

1. **Verb + Benefit:** "Get Instant Access"
2. **Verb + Outcome:** "Start Getting Matches"
3. **First Person + Action:** "Start My Free Trial"
4. **Action + Risk Reversal:** "Try Free - Cancel Anytime"
5. **Urgency + Action:** "Claim Your Spot Now"

### Common Mistakes

- Generic CTAs ("Submit", "Click Here")
- Multiple competing CTAs
- CTA doesn't match headline promise
- Low contrast button color
- CTA below fold without above-fold option
- No supporting trust elements

### Test Priority Order

1. CTA copy (highest impact, easiest to test)
2. CTA placement (sticky vs inline)
3. Supporting text
4. Button color
5. Button size/animation
