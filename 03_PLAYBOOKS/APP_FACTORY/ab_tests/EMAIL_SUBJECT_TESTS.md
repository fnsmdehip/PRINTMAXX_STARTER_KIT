# Email Subject Line A/B Tests

Ready-to-test email subject line formulas and variations.

---

## Subject Line Formula Tests

### Test 1: Question vs Statement

**Hypothesis:** Questions create curiosity and increase opens.

| Variant | Subject Line | Formula |
|---------|--------------|---------|
| A | How I made $3k last month from a side project | Statement (result) |
| B | Want to know how I made $3k from a side project? | Question |
| C | Can you make $3k/month from a side project? | Question (you-focused) |

---

### Test 2: Curiosity Gap vs Direct Benefit

**Hypothesis:** Curiosity drives opens, but direct benefit drives quality opens.

| Variant | Subject Line | Approach |
|---------|--------------|----------|
| A | The tool I use every single day | Curiosity gap |
| B | This tool saves me 5 hours per week | Direct benefit |
| C | Why I switched to this tool (and never looked back) | Curiosity + hint |

---

### Test 3: Numbers vs No Numbers

**Hypothesis:** Specific numbers increase credibility and opens.

| Variant | Subject Line | Approach |
|---------|--------------|----------|
| A | How to grow your audience | No numbers |
| B | How to grow your audience to 10,000 followers | Specific number |
| C | How I grew from 0 to 10,000 followers in 90 days | Multiple numbers |

---

### Test 4: Personalization Depth

**Hypothesis:** More personalization increases opens but may feel creepy.

| Variant | Subject Line | Personalization |
|---------|--------------|-----------------|
| A | Quick question | None |
| B | Quick question, {FirstName} | Name only |
| C | {FirstName}, quick question about {Company} | Name + company |
| D | Saw your {RecentPost} on {Platform} | Behavioral |

---

### Test 5: Length Test

**Hypothesis:** Shorter subjects perform better on mobile.

| Variant | Subject Line | Length |
|---------|--------------|--------|
| A | Quick tip | 9 chars |
| B | Quick tip for your content strategy | 35 chars |
| C | Here's a quick tip that helped me 10x my content output | 56 chars |

---

### Test 6: Emoji Usage

**Hypothesis:** Emojis increase opens but may trigger spam filters.

| Variant | Subject Line | Emoji |
|---------|--------------|-------|
| A | Free tool inside | None |
| B | Free tool inside 🎁 | End emoji |
| C | 🎁 Free tool inside | Start emoji |
| D | 🎁 Free tool inside 🎁 | Multiple |

## Subject Line Formulas

### High-Performing Templates

**1. The How-To**
```
How to [achieve result]
How I [achieved result]
How [famous person] [achieved result]
```

**2. The List**
```
[Number] ways to [achieve result]
[Number] tools for [outcome]
[Number] mistakes killing your [area]
```

**3. The Question**
```
What if you could [desired outcome]?
Want to [desired outcome]?
Struggling with [pain point]?
```

**4. The Curiosity Gap**
```
The [thing] nobody talks about
Why [conventional wisdom] is wrong
The real reason [problem exists]
```

**5. The Social Proof**
```
How [person/company] [achieved result]
What [group] knows about [topic]
The [tool/method] [impressive people] use
```

**6. The Urgency**
```
Last chance: [offer]
[X hours] left to [action]
Ends tonight: [benefit]
```

**7. The Personal**
```
{Name}, quick question
For {Name} only
{Name}, I noticed something
```

**8. The Story**
```
I almost gave up, then...
The worst mistake I made
How I went from [A] to [B]
```

## Implementation Guide

### Email Platform A/B Testing

**ConvertKit:**
```
1. Create broadcast
2. Enable A/B testing
3. Enter variants (up to 4)
4. Set test percentage (typically 20-30%)
5. Set winner criteria (open rate, click rate)
6. Set test duration (4-24 hours)
7. Winner auto-sends to remainder
```

**Mailchimp:**
```
1. Create campaign
2. Choose A/B test
3. Select variable (subject line)
4. Enter variants
5. Set test size and duration
6. Choose winning metric
7. Schedule send
```

**Beehiiv:**
```
1. Create post
2. Enable subject line test
3. Add variants
4. Test sends to 30% of list
5. Winner sends to 70%
```

### Sample Size Requirements

| List Size | Test Size | Per Variant | Reliable Results? |
|-----------|-----------|-------------|-------------------|
| 1,000 | 300 | 150 | Marginal |
| 5,000 | 1,500 | 750 | Good |
| 10,000 | 3,000 | 1,500 | Strong |
| 50,000 | 10,000 | 5,000 | Very strong |

**Rule of thumb:** Need 100+ opens per variant for meaningful data.

---

### Metrics to Track

**Primary:**
- Open rate (unique opens / delivered)

**Secondary:**
- Click rate (unique clicks / delivered)
- Click-to-open rate (clicks / opens)
- Unsubscribe rate
- Reply rate (for cold email)

## Subject Line Audit Checklist

Before testing, check each variant:

- [ ] Under 50 characters for mobile preview
- [ ] No spam trigger words (free, guarantee, act now)
- [ ] Matches email content (no clickbait)
- [ ] Personalization tokens work
- [ ] Preview text complements subject
- [ ] No ALL CAPS (except intentional emphasis)
- [ ] Emojis render correctly across platforms

## Quick Wins

### Immediate Tests to Run

1. **Add name personalization** - Usually 10-20% open lift
2. **Shorten long subjects** - Test under 40 chars
3. **Add curiosity element** - "The reason why..." formulas
4. **Test number specificity** - "3 tools" vs "several tools"

### Things That Rarely Work

- ALL CAPS subjects
- Excessive punctuation (!!!)
- Clickbait that doesn't match content
- "Newsletter Issue #47" format
- Generic "Monthly Update" subjects
