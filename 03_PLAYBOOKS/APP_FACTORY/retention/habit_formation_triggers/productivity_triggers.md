# Productivity app habit formation triggers

Optimal times and contexts to reach productivity/focus app users.

Productivity apps have unique triggers because users want to work, not be distracted.

---

## Core principle

Productivity apps face a paradox: notifications interrupt focus.

Solution: Trigger at transition moments, not during work.

---

## Primary trigger windows

### Morning planning (7am-9am)

**Why it works:**
- Day hasn't started yet
- Setting intentions works
- Cognitive energy is highest
- Pre-work routine building

**Trigger timing:**
```
Push notification: 7:30am local time
Pre-work planning

Copy variants:
- "Plan your day in 5 minutes. What's your top priority?"
- "Before the chaos starts. Set today's focus goal."
- "Morning planning. 3 things to accomplish today."
```

**Best practices:**
- Very quick interaction (<2 min)
- Focus on planning, not doing
- Set up the day's focus blocks
- Don't require opening app (quick reply)

---

### First work session (9am-10am)

**Why it works:**
- Traditional work start time
- Fresh cognitive state
- Before meetings/interruptions
- Productive momentum building

**Trigger timing:**
```
Push notification: 9:00am (adjustable)
Start of focused work

Copy variants:
- "Focus time. First 25-minute session ready."
- "Deep work starts now. Your timer is waiting."
- "Morning focus block. Eliminate distractions."
```

**Best practices:**
- Enable Do Not Disturb suggestion
- Offer distraction blocking
- 25-90 minute session options
- Clear start ritual

---

### Post-lunch refocus (1pm-2pm)

**Why it works:**
- Afternoon slump is real
- Need external motivation
- Reset after break
- Second productivity window

**Trigger timing:**
```
Push notification: 1:15pm local time
Afternoon reset

Copy variants:
- "Afternoon focus. Combat the slump with one session."
- "Post-lunch productivity. Your brain is ready."
- "Second wind time. Focus session waiting."
```

**Best practices:**
- Acknowledge the slump
- Shorter session option (15-25 min)
- Energy-appropriate tasks
- Background music/focus sounds

## Secondary trigger windows

### End of day review (5pm-6pm)

**Why it works:**
- Closure on the day
- Celebration of progress
- Planning for tomorrow
- Reduces anxiety about unfinished work

**Trigger timing:**
```
Push notification: 5:30pm local time
End-of-day wrap-up

Copy variants:
- "Day's end. Review what you accomplished."
- "Wrap-up time. Log your progress and plan tomorrow."
- "Close the loop. 3-minute end-of-day review."
```

**Best practices:**
- Quick reflection (<3 min)
- Show accomplishments
- Queue tomorrow's priorities
- Enable "done for the day" feeling

---

### Evening planning (8pm-9pm)

**Why it works:**
- Tomorrow prep reduces morning stress
- Calm reflection time
- Subconscious works overnight
- Morning decision fatigue reduced

**Trigger timing:**
```
Push notification: 8:30pm (optional)
Tomorrow prep

Copy variants:
- "Tomorrow's focus. What's the one thing?"
- "Evening planning. Set up tomorrow's success."
- "Before bed: plan one priority for tomorrow."
```

**Best practices:**
- Single priority focus
- Not a full planning session
- Optional/opt-in only
- Calming, not stressful

---

### Weekend deep work (Saturday/Sunday morning)

**Why it works:**
- No meetings or interruptions
- Extended focus possible
- Side project time
- Learning and growth time

**Trigger timing:**
```
Push notification: 9:00am Saturday/Sunday
Weekend productivity

Copy variants:
- "Weekend focus time. Long session available."
- "No meetings today. Perfect for deep work."
- "Saturday morning productivity. 2-hour focus block?"
```

**Best practices:**
- Longer session options (90-120 min)
- Different from weekday (less urgent)
- Side project encouragement
- Learning/reading modes

## Productivity pattern triggers

### Streak maintenance

```
Active streak:
"Day [X] of focused work. Keep the momentum."
"[X]-day productivity streak. Your session is ready."

Streak at risk (evening):
"One focus session saves your streak. 25 minutes?"

After missed day:
"Yesterday was busy. Today's a fresh start. Begin?"
```

### Best time detection

**Learn user's patterns:**
```
"You're most productive around 10am. Session starting?"
"Your focus peaks mid-morning. Prime time right now."
"Based on your history: now is your best focus window."
```

### Procrastination detection

**Signals:** App opened but session not started, task viewed but not worked on

```
"Ready to start? Just tap to begin focus mode."
"The hardest part is starting. We'll count down from 3."
"Break it into 10 minutes. That's it. Start small."
```

## Anti-interruption strategy

### When NOT to send notifications

- During active focus sessions (obviously)
- Within 30 minutes of session completion
- During calendar events
- During detected sleep hours
- More than 3x per day (fatigue prevention)

### Smart timing rules

```
Rules:
1. Never interrupt active focus
2. Respect system Do Not Disturb
3. Learn optimal times from behavior
4. Ask permission for new trigger times
5. Allow "quiet hours" configuration
```
