# Wholesale and licensing sequence

For bulk licensing, white-label deals, and enterprise partnerships.

---

## B2B licensing outreach

### Subject line variants

- A: Bulk licensing for {{company_name}}
- B: {{app_name}} for your {{members/employees/clients}}
- C: Volume discount opportunity

### Email 1: The pitch

```
{{first_name}},

I run {{app_name}}, a {{one_liner}}.

I noticed {{company_name}} serves {{their_audience}}. There's a potential fit for a bulk licensing deal.

The offer:
- Volume pricing for your {{members/employees/customers}}
- Admin dashboard to manage users
- Custom branding available
- Dedicated support

Example pricing:
- 50-100 seats: $3/user/month
- 100-500 seats: $2.50/user/month
- 500+ seats: Custom pricing

Is this something {{company_name}} would consider?

{{your_name}}
```

---

### Email 2: Use case focus (Day 4)

**Subject:** Re: Bulk licensing

```
{{first_name}},

Following up on the licensing idea.

Here's how companies like {{company_name}} typically use {{app_name}}:

**Use case 1: Employee/member benefit**
Offer it as a perk. You provide value, we handle everything.

**Use case 2: Program integration**
Include it in an existing program (wellness, training, onboarding).

**Use case 3: Client offering**
Resell to your clients as part of your services.

Current partners include:
- {{partner_type_1}}: {{how_they_use}}
- {{partner_type_2}}: {{how_they_use}}

Which use case fits {{company_name}}?

{{your_name}}
```

---

### Email 3: The close (Day 9)

**Subject:** Quick question

```
{{first_name}},

Last note on the licensing opportunity.

Three options:
1. **Let's talk** - I'll send my calendar
2. **Not now** - I'll follow up in a few months
3. **Not interested** - I'll stop emailing

Any response helps.

{{your_name}}
```

## Pricing structure

### Volume licensing

| Seats | Price per user/month |
|-------|---------------------|
| 10-49 | $4.00 |
| 50-99 | $3.50 |
| 100-249 | $3.00 |
| 250-499 | $2.50 |
| 500+ | Custom |

### White-label packages

**Starter:**
- Setup: $2,500
- Monthly: $500
- Includes: Basic branding, standard features

**Professional:**
- Setup: $5,000
- Monthly: $1,000
- Includes: Full branding, custom features, priority support

**Enterprise:**
- Setup: Custom
- Monthly: Custom
- Includes: Everything + dedicated support, SLA, custom development

## Response handling

### If they say "Yes, interested"

```
{{first_name}},

Great. Let's set up a call to discuss specifics.

Before we talk, it would help to know:
1. How many users are you thinking?
2. What's your timeline?
3. Do you need white-label or standard licensing?
4. What's your budget range?

Here's my calendar: {{calendar_link}}

Looking forward to it.

{{your_name}}
```

---

### If they say "What's included?"

```
{{first_name}},

Here's the full breakdown:

**Included in licensing:**
- User accounts (web + mobile apps)
- Admin dashboard for user management
- Usage analytics and reports
- Basic customization (logo, welcome message)
- Email support (24-48 hour response)
- Quarterly product updates

**Add-on options:**
- Custom branding: +${{amount}}/month
- SSO integration: +${{amount}}/month
- API access: +${{amount}}/month
- Dedicated support: +${{amount}}/month

**Not included:**
- Custom feature development (quoted separately)
- On-premise hosting (cloud only)
- Training (available for additional fee)

Questions?

{{your_name}}
```

---

### If they say "Too expensive"

```
{{first_name}},

I hear you. Budget is real.

A few options:

**Option 1: Start smaller**
Pilot with a smaller group (50 users instead of 200). Lower commitment, lower cost.

**Option 2: Annual prepay discount**
Pay annually upfront, get 2 months free. Brings per-user cost down.

**Option 3: Revenue share alternative**
Instead of flat fee, we take a % of what you charge users. No upfront cost.

Which would work better for {{company_name}}?

{{your_name}}
```

---

### If they say "We build our own tools"

```
{{first_name}},

Makes sense. Building in-house gives you control.

Quick question: how much would it cost to build and maintain something like this internally?

Rough estimate for a {{app_type}} app:
- Development: $50-100K+
- Annual maintenance: $20-40K+
- Time to launch: 6-12 months

Versus licensing from us:
- Cost: ${{annual_cost}}/year
- Time to launch: 2 weeks
- We handle all maintenance and updates

If the economics ever make sense to buy instead of build, I'm here.

{{your_name}}
```

## Contract basics

### Terms to include

- Number of licensed seats
- Term length (annual recommended)
- Payment terms (net 30)
- Renewal terms
- Usage restrictions
- Support SLA
- Termination clause

### Red flags to watch for

- Wanting perpetual license for one-time fee
- Unreasonable SLA demands
- Scope creep on custom features
- Payment terms beyond net 60
- Exclusivity requests without premium
