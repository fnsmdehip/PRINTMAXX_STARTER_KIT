## Usage Tips

### Prompt Optimization

**For better results:**

1. **Be specific about dimensions** - Always include pixel sizes
2. **Specify "transparent background"** for mascots you'll composite
3. **Reference "clean vector style"** for crisp, scalable results
4. **Include color hex codes** for brand consistency
5. **Add "on transparent background"** when you need cutouts

### Batch Generation Strategy

**For icon variations:**
```
Generate 5 variations of this app icon design with these differences:
1. Original as described
2. Darker background variant
3. Lighter, more minimal version
4. Higher contrast version
5. Inverted color scheme

[PASTE ORIGINAL PROMPT HERE]
```

**For mascot pose sheets:**
```
Create a character sheet with 6 poses of this mascot character:
1. Front view, neutral
2. Side view, walking
3. Happy/celebrating
4. Thinking/confused
5. Tired/resting
6. Waving hello

Character description: [PASTE CHARACTER DESCRIPTION]
Maintain consistent style, proportions, and colors across all poses.
```

### Style Consistency

**Reference image usage:**
When generating multiple assets, first create your "hero" asset (main app icon or primary mascot pose), then reference it:

```
Using [PREVIOUS IMAGE] as style reference, create [NEW ASSET DESCRIPTION]. Match the exact color palette, line weight, and illustration style.
```

### File Naming Convention

```
[app]_[asset-type]_[variant]_[size].png

Examples:
prayerlock_icon_primary_1024.png
prayerlock_icon_dark_1024.png
prayerlock_mascot_praying_front.png
prayerlock_mascot_happy_front.png
prayerlock_screenshot_feature1_ios.png
walktounlock_ad_install_1080x1350.png
```

### Export Checklist

**For app icons:**
- [ ] 1024x1024 (App Store)
- [ ] 512x512 (Play Store)
- [ ] 180x180 (iOS home screen 3x)
- [ ] 120x120 (iOS home screen 2x)
- [ ] 60x60 (preview testing)

**For screenshots:**
- [ ] 1242x2688 (iPhone 15 Pro Max)
- [ ] 1290x2796 (iPhone 14 Pro Max)
- [ ] 1284x2778 (iPhone 13 Pro Max)

**For mascots:**
- [ ] High-res PNG with transparency
- [ ] SVG if possible for scaling
- [ ] Individual poses as separate files
