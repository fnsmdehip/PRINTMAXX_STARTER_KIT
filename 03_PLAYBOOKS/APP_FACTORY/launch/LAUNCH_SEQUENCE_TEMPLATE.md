# Launch Sequence Template

Complete timeline from 30 days pre-launch through post-launch week.

---

## T-30 days: Pre-launch prep

### Technical foundation

- [ ] App core features complete and tested
- [ ] RevenueCat/payment integration working
- [ ] Analytics SDK integrated (Mixpanel, Amplitude, or PostHog)
- [ ] Crash reporting enabled (Sentry, Crashlytics)
- [ ] Basic onboarding flow functional
- [ ] Privacy policy and terms of service drafted

### Infrastructure setup

- [ ] Domain purchased for landing page
- [ ] Email service configured (ConvertKit, Buttondown, or Instantly)
- [ ] Social accounts created and warming up
- [ ] Support email configured (support@yourdomain.com)
- [ ] Basic landing page live (even if WIP)

### Content planning

- [ ] Brand voice document finalized
- [ ] Screenshot list defined (what scenes to capture)
- [ ] App preview video storyboard
- [ ] Social content calendar drafted
- [ ] Blog post ideas documented

### Team prep

- [ ] Launch day roles assigned (who does what)
- [ ] Communication channels ready (Slack, Discord, Telegram)
- [ ] Emergency contact list
- [ ] Monitoring responsibilities defined

---

## T-14 days: Beta testing

### TestFlight/Internal testing

- [ ] Build uploaded to TestFlight
- [ ] Internal testers invited (5-10 people minimum)
- [ ] Feedback collection system ready (Typeform, Notion, or email)
- [ ] Bug tracking active (Linear, GitHub Issues)
- [ ] Daily test builds if fixing issues

### Beta tester recruitment

- [ ] Reach out to 50+ potential beta testers
- [ ] Sources: X followers, email list, Reddit, Discord communities
- [ ] Offer: free lifetime access, early adopter badge, name in credits
- [ ] Target: 20-50 active beta testers

### Beta feedback loop

- [ ] Daily check-in on feedback
- [ ] Priority bug fixes only (no feature creep)
- [ ] Document most requested features for v1.1
- [ ] Identify power users for testimonials

### Marketing prep

- [ ] Landing page 80% complete
- [ ] Email capture working
- [ ] Social accounts posting 1-2x daily
- [ ] Affiliate program structure defined
- [ ] Press/media list started

---

## T-7 days: Marketing ramp-up

### App Store submission

- [ ] Final TestFlight build uploaded
- [ ] App Store screenshots ready (6.5", 5.5", iPad if applicable)
- [ ] App preview video uploaded (if using)
- [ ] App name, subtitle, keywords finalized
- [ ] Full description written
- [ ] Submit for review (allow 2-3 days)

### Google Play submission

- [ ] Feature graphic ready (1024x500)
- [ ] Screenshots uploaded
- [ ] Short and full descriptions written
- [ ] Data safety form completed
- [ ] Content rating questionnaire done
- [ ] Submit for review

### Landing page finalization

- [ ] Hero section with clear value prop
- [ ] Screenshots/demo video embedded
- [ ] Social proof section (testimonials from beta)
- [ ] FAQ section
- [ ] App Store and Google Play badges (or "Coming Soon")
- [ ] Email capture with launch notification signup
- [ ] Privacy policy and terms linked

### Content stockpile

- [ ] 30 TikTok/Reels videos queued
- [ ] 50 X/Twitter posts written
- [ ] 10 threads/carousels ready
- [ ] Launch announcement draft for all platforms
- [ ] Email launch sequence drafted (5 emails)
- [ ] Blog launch post written

### Outreach prep

- [ ] Media/press email list ready (20-50 contacts)
- [ ] Podcast pitch template written
- [ ] Influencer/affiliate outreach list (50+ contacts)
- [ ] Partner DMs drafted
- [ ] Friends/family launch support list

---

## T-3 days: Final checks

### App Store status

- [ ] iOS app approved or in review
- [ ] Android app approved or in review
- [ ] Set release date (if using scheduled release)
- [ ] Verify all app store assets display correctly
- [ ] Test purchase flow one more time

### Technical verification

- [ ] Test app on 3+ real devices
- [ ] Verify analytics events firing
- [ ] Verify crash reporting working
- [ ] Test all IAP/subscription tiers
- [ ] Verify deep links working
- [ ] Test push notifications

### Marketing systems check

- [ ] Email sequences loaded and tested
- [ ] Social post scheduler loaded
- [ ] Landing page live and fast-loading
- [ ] UTM tracking configured
- [ ] Conversion tracking pixels verified
- [ ] Affiliate tracking links tested

### Communication prep

- [ ] Notify beta testers of launch date
- [ ] Notify affiliates/partners
- [ ] Brief anyone helping on launch day
- [ ] Prepare launch day communication template

### Social hype

- [ ] Post "launching tomorrow" across all platforms
- [ ] Share behind-the-scenes content
- [ ] Go live/Stories showing final prep
- [ ] Engage heavily in comments all day
- [ ] DM warm contacts about tomorrow

### Email list

- [ ] Send "launching tomorrow" email
- [ ] Subject: "[App Name] launches tomorrow"
- [ ] Include specific launch time
- [ ] Build anticipation without full reveal

### Team sync

- [ ] Final launch day walkthrough
- [ ] Confirm everyone knows their role
- [ ] Test all communication channels
- [ ] Set alarms for launch time
- [ ] Get sleep

### Final prep

- [ ] Confirm app approved and ready
- [ ] Queue launch day social posts
- [ ] Queue launch email blast
- [ ] Prepare monitoring dashboards
- [ ] Review launch day playbook

## Launch day: Hour-by-hour plan

See `launch_day_playbook.md` for detailed hour-by-hour schedule.

### Quick summary

**Hour 0 (Launch):**
- Release app on stores
- Send launch email
- Post launch announcement on all platforms
- Notify affiliates/partners

**Hours 1-4:**
- Engage with every comment/reply
- DM warm contacts
- Monitor for crashes/bugs
- Start small paid ad test

**Hours 4-8:**
- Continue engagement
- Share first user feedback/testimonials
- Post additional content pieces
- Track metrics hourly

**Hours 8-12:**
- Respond to all reviews
- Share milestones ("100 downloads!")
- Continue content posting
- Outreach follow-ups

**End of day:**
- Document learnings
- Screenshot metrics
- Plan day 2 content
- Celebrate (briefly)

## T+1 to T+7: Post-launch

### Day 1 (T+1)

- [ ] Review Day 0 metrics
- [ ] Respond to all new reviews
- [ ] Post follow-up content ("Day 1 update")
- [ ] Send day 1 follow-up email
- [ ] Fix any critical bugs reported
- [ ] Continue engagement in comments

### Day 2 (T+2)

- [ ] Post user testimonial/social proof
- [ ] Share behind-the-scenes numbers (if good)
- [ ] Follow up with press/podcast pitches
- [ ] Analyze ad performance, adjust
- [ ] Continue daily content posting

### Day 3 (T+3)

- [ ] Send social proof email (day_3_social_proof.md)
- [ ] Post educational content related to app
- [ ] Reach out to users for reviews/testimonials
- [ ] Start ASO optimization based on early data
- [ ] Review retention metrics

### Day 4-5 (T+4-5)

- [ ] Maintain content posting schedule
- [ ] Respond to all user feedback
- [ ] Prepare any urgent bug fix release
- [ ] Analyze which channels driving installs
- [ ] Double down on what's working

### Day 6 (T+6)

- [ ] Create "1 week since launch" content
- [ ] Share learnings/numbers publicly
- [ ] Thank early adopters
- [ ] Send preview of upcoming features

### Day 7 (T+7)

- [ ] Send "last chance" email (day_7_last_chance.md)
- [ ] Post launch week recap
- [ ] Create full launch retrospective doc
- [ ] Plan Week 2+ content calendar
- [ ] Set ongoing posting schedule
- [ ] Define v1.1 roadmap based on feedback

## Post-launch ongoing

### Daily tasks (first month)

- [ ] Check crash reports and reviews
- [ ] Respond to all user messages
- [ ] Post at least 1 piece of content
- [ ] Monitor key metrics

### Weekly tasks

- [ ] Full metrics review (downloads, revenue, retention)
- [ ] Content performance analysis
- [ ] Ad spend optimization
- [ ] User interview (1-2 per week)
- [ ] Competitor check
- [ ] Update roadmap based on feedback

### Monthly tasks

- [ ] Full funnel analysis
- [ ] Pricing review
- [ ] Feature prioritization
- [ ] Retention cohort analysis
- [ ] Plan next major update
- [ ] Affiliate program review

---

## Key metrics to track

### Launch day

- Downloads (total, by source)
- Landing page visits
- Email open/click rates
- Social engagement
- Crashes/errors
- First reviews

### First week

- Day 1, 3, 7 retention
- Trial start rate
- Conversion rate (trial to paid)
- Revenue
- Reviews (count and rating)
- Cost per install (if running ads)

### First month

- Monthly retention
- MRR/ARR
- LTV (estimated)
- CAC by channel
- Feature usage
- Churn rate

## Launch templates

**Launch announcement (X/Twitter):**
```
[App Name] is live.

After [X weeks/months] of building, it's finally here.

[One-line description of what it does]

[Key benefit 1]
[Key benefit 2]
[Key benefit 3]

Link in bio. First 100 users get [bonus].
```

**Launch email subject lines:**
- "[App Name] is live (finally)"
- "It's here: [App Name]"
- "You asked for it. We built it."
- "Download [App Name] now"

**Review ask template:**
```
Hey! Thanks for trying [App Name].

Quick favor: If you're finding it useful, a 5-star review helps a ton.

Takes 30 seconds: [direct review link]

If something's not working, reply to this email instead. Happy to help.
```
