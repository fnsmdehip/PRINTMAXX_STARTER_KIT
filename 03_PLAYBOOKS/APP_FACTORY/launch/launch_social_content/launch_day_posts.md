# Launch Day Posts

10 posts per platform for launch day. Copy, paste, customize.

## X/Twitter posts (10)

### Post 1: Main announcement (Pin this)

```
[App Name] is live.

After [X] months of building, it's finally here.

[One line: what it does]

- [Benefit 1]
- [Benefit 2]
- [Benefit 3]

Download free: [link]

First 100 users get [bonus].
```

### Post 2: The problem (1-2 hours after launch)

```
Every day, [target audience] waste [X] hours on [problem].

I got tired of it.

So I built [App Name].

Now that same task takes [Y] minutes.

Try it: [link]
```

### Post 3: Behind the scenes (3-4 hours after launch)

```
Building [App Name] in numbers:

- [X] months of development
- [Y] beta testers
- [Z] bug fixes
- [N] cups of coffee

Today: launch day

The hardest part? Hitting "publish."

If you're building something, ship it. The fear never goes away.
```

### Post 4: First milestone (when you hit 50/100 downloads)

```
[X] downloads in [Y] hours.

Wasn't sure anyone would care about [problem space].

Thank you to everyone who tried [App Name] today.

Still have launch pricing for [Z] more hours: [link]
```

### Post 5: Feature highlight (5-6 hours after launch)

```
Most asked question about [App Name]:

"Does it do [feature]?"

Yes.

Here's how it works:
[1-2 sentence explanation]

[Screenshot or GIF if possible]

Try it: [link]
```

### Post 6: Social proof (when you get first positive feedback)

```
First review is in:

"[Quote from user]"

This is why I built [App Name].

[Target audience]: [link to download]
```

### Post 7: Use case (7-8 hours after launch)

```
3 ways people are using [App Name] today:

1. [Use case 1]
2. [Use case 2]
3. [Use case 3]

What would you use it for?

[link]
```

### Post 8: Engagement question (8-9 hours after launch)

```
Quick question for [target audience]:

What's your biggest frustration with [problem space]?

Building the next [App Name] update and want to know what matters most.

Reply or quote tweet.
```

### Post 9: Thank you (10-11 hours after launch)

```
Launch day is almost over.

Thank you to:
- Everyone who downloaded
- Everyone who shared
- Everyone who sent feedback

You made today worth it.

If you haven't tried [App Name] yet: [link]

Tomorrow we keep building.
```

### Post 10: End of day recap (12+ hours after launch)

```
[App Name] launch day results:

- [X] downloads
- [Y] reviews
- [Z] messages

Best moment: [specific positive thing]

Worst moment: [honest struggle, makes you relatable]

Day 1 complete. Day 2 starts now.
```

## TikTok posts (10)

Hook: "I just launched my app"

End: "Link in bio"

Hook: "What people are saying about [App Name]"

## Hashtag strategy

### Instagram

Use 5-10 relevant hashtags:
- #[niche] (e.g., #productivity, #fitness, #faith)
- #app #mobileapp #appstore
- #launch #startup #solopreneur
- #[specific problem] (e.g., #habittracking, #mealprep)

### TikTok

Use 3-5 hashtags:
- #app #newapp
- #[niche]
- #fyp #foryou (controversial but sometimes helps)
- #smallbusiness #solopreneur

### X/Twitter

Generally no hashtags, or 1-2 max:
- #buildinpublic
- #indiehacker
