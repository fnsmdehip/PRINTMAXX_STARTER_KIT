# Launch Day Playbook

Hour-by-hour tasks for a successful launch. Print this out.

---

## Pre-launch (Day before)

### Evening prep (T-12 hours)

- [ ] Confirm app approved on iOS (status: "Ready for Sale")
- [ ] Confirm app approved on Android (status: "Published" or scheduled)
- [ ] Queue all social posts for tomorrow
- [ ] Queue launch email blast
- [ ] Test all links one more time
- [ ] Set up monitoring dashboards on second screen
- [ ] Prepare coffee/snacks
- [ ] Set alarms for launch time
- [ ] Go to bed early

---

## Launch morning

### T-2 hours (before launch)

**Wake up and prep:**
- [ ] Check email for any issues
- [ ] Verify app still approved
- [ ] Open all monitoring dashboards:
  - [ ] RevenueCat (revenue)
  - [ ] App Store Connect / Google Play Console (downloads)
  - [ ] Analytics dashboard (users)
  - [ ] Crash reporting (errors)
  - [ ] Social mentions (Tweetdeck, Brand24, or manual)
- [ ] Verify scheduled posts are ready
- [ ] Verify scheduled email is ready
- [ ] Final TestFlight download and quick test

### T-1 hour

**Team sync:**
- [ ] Confirm everyone online and ready
- [ ] Review everyone's tasks
- [ ] Test communication channel (Slack/Discord)
- [ ] Confirm who handles what:
  - [ ] Social engagement: ________
  - [ ] Email support: ________
  - [ ] Bug monitoring: ________
  - [ ] Analytics: ________

### T-15 minutes

**Final checks:**
- [ ] Landing page loading fast
- [ ] App Store page showing correctly
- [ ] Google Play page showing correctly
- [ ] Review queue: App Store reviews (be ready)
- [ ] Prepare launch tweet/post for manual send
- [ ] Deep breath

---

## Launch hour (T-0)

### Minute 0: GO

- [ ] **iOS:** Set release date to "now" or release manual hold
- [ ] **Android:** Confirm published or release manual hold
- [ ] **Email:** Send launch blast (or confirm scheduled send)
- [ ] **Social:** Post launch announcement on all platforms:
  - [ ] X/Twitter
  - [ ] Instagram (post + story)
  - [ ] TikTok
  - [ ] LinkedIn (if B2B)
  - [ ] Product Hunt (if applicable, see product_hunt_launch.md)

### Minutes 0-15

**Verify everything working:**
- [ ] App showing as available on App Store
- [ ] App showing as available on Google Play
- [ ] Download app from store (not TestFlight)
- [ ] Make test purchase (real money)
- [ ] Verify analytics events appearing
- [ ] Check for any immediate crashes

### Minutes 15-30

**Start engagement:**
- [ ] Like/reply to every comment on launch posts
- [ ] DM top supporters thanking them
- [ ] Share launch post to Stories
- [ ] Check email for any support requests
- [ ] Monitor crash reports

### Minutes 30-60

**Expand reach:**
- [ ] Post in relevant communities:
  - [ ] Reddit (if allowed, use personal account, add value)
  - [ ] Discord servers
  - [ ] Slack communities
  - [ ] Facebook groups
- [ ] DM warm contacts personally:
  - [ ] Friends who said they'd support
  - [ ] Beta testers
  - [ ] Affiliates/partners
  - [ ] Other founders

---

## Hours 1-4

### Continuous tasks

**Every 30 minutes:**
- [ ] Check social notifications, reply to all
- [ ] Check reviews, respond to any negative ones immediately
- [ ] Check crash reports
- [ ] Check support email

**Hour 1:**
- [ ] Screenshot current stats for documentation
- [ ] Share milestone if hit ("50 downloads in first hour!")
- [ ] Continue DM outreach
- [ ] Thank anyone who shares/posts

**Hour 2:**
- [ ] Post additional piece of content (not just reposts)
- [ ] Share any positive feedback received
- [ ] Check ad campaigns (if running small test)
- [ ] Update team on status

**Hour 3:**
- [ ] Post behind-the-scenes content
- [ ] Share screenshot of dashboard (if numbers are good)
- [ ] Follow up on any unanswered DMs
- [ ] Check Google Alerts / social mentions

**Hour 4:**
- [ ] Full metrics check
- [ ] Post engagement content (poll, question)
- [ ] Start preparing "end of launch day" content
- [ ] Team sync call (15 min)

---

## Hours 4-8

### Maintain momentum

**Hour 5:**
- [ ] Post user testimonial (if received)
- [ ] Share interesting usage stat
- [ ] Follow up with press/podcast contacts
- [ ] Continue engagement

**Hour 6:**
- [ ] Check international downloads (time zone coverage)
- [ ] Post educational content related to app
- [ ] Respond to all pending DMs
- [ ] Review ad performance, adjust budget

**Hour 7:**
- [ ] Post "still going!" update
- [ ] Share any milestones hit
- [ ] DM anyone who engaged heavily (potential power users)
- [ ] Check for any new reviews

**Hour 8:**
- [ ] Full metrics snapshot
- [ ] Document any bugs reported
- [ ] Plan evening content
- [ ] Take a break (30 min, you earned it)

---

## Hours 8-12

### Evening push

**Hour 9:**
- [ ] Post evening content for different time zones
- [ ] Follow up with any outstanding support
- [ ] Check international app store rankings
- [ ] Thank team for the push

**Hour 10:**
- [ ] Post "X hours since launch" update with numbers
- [ ] Engage with late-night audience
- [ ] Respond to any new reviews
- [ ] Check overnight support queue

**Hour 11:**
- [ ] Final social posts for the day
- [ ] Draft tomorrow's content
- [ ] Document day 1 learnings
- [ ] Prepare day 1 follow-up email

**Hour 12 (End of launch day):**
- [ ] Full metrics screenshot
- [ ] Send internal team recap
- [ ] Schedule tomorrow's first post
- [ ] Celebrate (you shipped!)
- [ ] Sleep

## Social posting schedule

### Launch day post timing

| Time | Platform | Content type |
|------|----------|--------------|
| 0:00 | All | Launch announcement |
| 0:05 | IG/TikTok | Stories/short video |
| 1:00 | X | Engagement post |
| 2:00 | All | Share first feedback |
| 4:00 | X | Milestone update |
| 6:00 | IG/TikTok | Feature demo |
| 8:00 | X | Behind-the-scenes |
| 10:00 | All | Thank you post |
| 12:00 | X | Day recap with stats |

### Post templates

**Launch announcement:**
```
[App Name] is live.

[Time period] of building. Today it ships.

[What it does in one line]

3 reasons to try it:
- [Benefit 1]
- [Benefit 2]
- [Benefit 3]

[Call to action + link]
```

**Milestone update:**
```
[Number] downloads in [time].

Wasn't sure anyone would care about [problem space].

Thank you to everyone who downloaded.

Still free/on launch special: [link]
```

**Thank you post:**
```
Overwhelmed by the response today.

[X] downloads
[Y] messages
[Z] reviews

You made launch day worth it.

Tomorrow: [what's next]
```

## PR outreach timing

### Morning (if launching at 9 AM)

- T-0: Email journalists on your list
- T+1h: Follow up on X/Twitter with personalized DMs
- T+2h: Submit to tech blogs that accept tips

### Avoid

- Don't spam the same person twice on launch day
- Don't pitch before 9 AM or after 6 PM local time
- Don't send mass BCC emails

## Monitoring dashboard

### What to have open

**Screen 1 (primary):**
- RevenueCat dashboard (revenue, subscribers)
- App Store Connect / Google Play Console (downloads)

**Screen 2 (secondary):**
- Analytics dashboard (Mixpanel/Amplitude)
- Crash reporting (Sentry/Crashlytics)

**Phone:**
- X/Twitter notifications
- Instagram notifications
- Email inbox

### Alert thresholds

| Metric | Alert if | Action |
|--------|----------|--------|
| Crash rate | > 1% | Investigate immediately |
| Error rate | Spike | Check error logs |
| Revenue | $0 after 4h | Verify payment flow |
| Downloads | 0 after 1h | Check store listing visibility |
| Negative review | Any | Respond within 2h |
