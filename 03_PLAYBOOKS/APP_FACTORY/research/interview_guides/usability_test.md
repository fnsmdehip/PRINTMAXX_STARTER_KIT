# Usability Test Guide

Testing if features work the way users expect.

**Duration:** 20-30 minutes
**Participants:** 5 users (finds 80% of issues)
**Goal:** Identify friction points and confusion

---

## Before the test

### Prep checklist
- [ ] Prototype or live feature ready
- [ ] Clear testing device (phone/desktop as appropriate)
- [ ] Tasks written out
- [ ] Recording setup working (screen + audio)
- [ ] Note-taking template ready
- [ ] Backup plan if tech fails

### Define tasks

Write 3-5 specific tasks. Good tasks:
- Have a clear success criteria
- Match real user goals
- Don't give away the answer

**Good task:** "You want to log your morning devotional. Show me how you'd do that."

**Bad task:** "Click the plus button in the top right to add a devotional entry."

---

## Test script

### Opening (2 min)

> Thanks for helping test [product/feature]. I'm going to ask you to try a few things while thinking out loud. There are no right or wrong answers. I'm testing the design, not you.
>
> Please tell me what you're thinking as you go. If something confuses you, that's useful information. Don't worry about hurting my feelings.
>
> Any questions before we start?

---

### Warm-up (2 min)

> Before we dive in, tell me briefly about your experience with [relevant category of app].

**Purpose:** Gets them comfortable talking. Provides context for their skill level.

---

### Task 1: [Primary task]

> [Read task exactly as written. Don't add hints.]

**Observe silently. Note:**
- Where they click first
- Hesitations
- Misclicks
- Facial expressions
- Verbal confusion

**Only prompt if stuck for 30+ seconds:**
- "What are you looking for?"
- "What would you expect to happen?"
- "What's going through your mind?"

**After task complete:**
- "On a scale of 1-5, how easy was that? 1 = very hard, 5 = very easy."
- "Anything confusing about that?"

---

### Task 2: [Secondary task]

> [Read task exactly as written.]

**Same observation protocol.**

---

### Task 3: [Tertiary task]

> [Read task exactly as written.]

**Same observation protocol.**

---

### Follow-up questions (5 min)

> Now that you've used it, what's your overall impression?

> What was the easiest part?

> What was the most confusing part?

> What would you change?

> Would you use this? Why or why not?

---

### Wrap-up (1 min)

> That's all I have. Any final thoughts?

> Thanks for your time. [Explain incentive if applicable.]

---

## During the test: Don't do these

| Don't | Do instead |
|-------|------------|
| "Click that button there" | Let them struggle |
| "That's correct!" | Neutral: "What made you choose that?" |
| "What you should do is..." | "What would you try next?" |
| Answer questions about how to use it | "What do you think?" |
| Fill silences | Wait. They're processing. |

### Severity definitions

**High:** User cannot complete task. Ship-blocking.
**Medium:** User completes task with significant confusion. Fix before launch.
**Low:** Minor friction. Nice to fix, not urgent.

### Priority matrix

| | Low effort | High effort |
|--|------------|-------------|
| **High impact** | Fix immediately | Schedule soon |
| **Low impact** | Quick win | Backlog |

## Task library by app type

### Faith/Devotional app
- Log a devotional entry

- Skip today's planned workout
- Adjust a goal
- Find exercise instructions

- Create a new task
- Mark task as complete
- Reschedule a task
- Find overdue items
- Set up a recurring task
