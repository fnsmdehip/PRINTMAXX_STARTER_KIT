## Thread 1: Building a prompt library

**Post 1 (Hook):**
I have 200+ ChatGPT prompts saved.

Here's how to build your own prompt library from scratch:

**Post 2:**
Step 1: Start tracking what you use ChatGPT for.

For one week, write down every task.

Mine: writing emails, generating ideas, code review, research summaries, content outlines.

Yours might be different.

**Post 3:**
Step 2: For each task, create a "starter prompt."

Not perfect. Just functional.

Example email prompt:
"Write a professional email to [RECIPIENT] about [TOPIC]. Tone: [TONE]. Key points: [POINTS]"

**Post 4:**
Step 3: Iterate when you get good outputs.

When ChatGPT nails it, look at your prompt.

What made it work?

Save that version. Delete the old one.

**Post 5:**
Step 4: Add variables for things that change.

[RECIPIENT], [TOPIC], [TONE], [POINTS]

One template handles infinite variations.

Much faster than rewriting each time.

**Post 6:**
Step 5: Organize by use case.

My folders:
- Writing
- Coding
- Research
- Analysis
- Ideas
- Templates

Find what matches your workflow.

**Post 7:**
Step 6: Review and prune monthly.

Some prompts stop being useful.

Some get replaced by better versions.

Keep your library clean.

**Post 8:**
Step 7: Share and steal.

The best prompts often come from other people.

Share yours. Ask for theirs.

Twitter is full of prompt gold.

**Post 9:**
My library started with 5 prompts.

Now it's 200+.

The key: save every prompt that works.

Don't trust your memory.

## Thread 3: Why most people fail at ChatGPT

**Post 1 (Hook):**
Most people fail at ChatGPT.

Here's why (and how to fix it):

**Post 2:**
Problem 1: Vague prompts

"Write me something about marketing"

ChatGPT has no context. Output is generic.

Fix: Be specific. Include context, format, constraints, examples.

**Post 3:**
Problem 2: No iteration

First output isn't good. They give up.

Fix: Refine your prompt. Add context. Ask for revisions.

ChatGPT gets better with feedback.

**Post 4:**
Problem 3: Starting from scratch every time

Same tasks. Rewriting prompts each time.

Fix: Save prompts that work. Build a library.

Stop reinventing the wheel.

**Post 5:**
Problem 4: No variables

Rewriting entire prompts for small changes.

Fix: Use placeholders.
[TOPIC], [AUDIENCE], [FORMAT]

Fill in blanks instead of rewriting.

**Post 6:**
Problem 5: Wrong expectations

Expecting perfect output first try.

ChatGPT is a collaborator, not a magician.

Fix: Use it as a draft generator. Edit the output.

**Post 7:**
Problem 6: No system

Random usage. Random results.

Fix: Create a workflow.
1. Choose prompt from library
2. Customize variables
3. Generate output
4. Edit and use

**Post 8:**
The pattern:

Casual users: treat ChatGPT like a magic 8-ball
Power users: treat it like a trained assistant with a playbook

Same tool. Very different outcomes.

**Post 9:**
Start here:

1. Pick your 3 most common ChatGPT tasks
2. Create a solid prompt for each
3. Save them somewhere accessible
4. Use and improve over time

That's it. Complexity comes later.

## Thread 4: Prompt variables deep dive

**Post 1 (Hook):**
Variables made my prompts 10x more useful.

Here's how to use them:

**Post 2:**
What are prompt variables?

Placeholders you fill in before running the prompt.

Instead of rewriting: "Write a blog about dogs"
You have: "Write a [CONTENT TYPE] about [TOPIC]"

**Post 3:**
Why they matter:

One prompt becomes infinite prompts.

"Write a [TYPE] about [TOPIC] for [AUDIENCE]"

Blog about AI for marketers
Tweet about fitness for beginners
Email about sales for executives

Same template. Endless uses.

**Post 4:**
Common variables I use:

[TOPIC] - Subject matter
[AUDIENCE] - Who it's for
[TONE] - Voice/style
[FORMAT] - Structure/length
[CONSTRAINTS] - Rules to follow
[CONTEXT] - Background info

**Post 5:**
Example writing prompt:

"Write a [FORMAT] about [TOPIC] for [AUDIENCE]. Tone: [TONE]. Include: [REQUIREMENTS]. Constraints: [CONSTRAINTS]"

Fill in blanks. Get tailored output.

**Post 6:**
Example coding prompt:

"Write [LANGUAGE] code to [FUNCTION]. It should accept [INPUTS] and return [OUTPUTS]. Handle these edge cases: [EDGE_CASES]. Include comments explaining the logic."

**Post 7:**
Example research prompt:

"Research [TOPIC] and provide a [FORMAT] summary. Focus on: [FOCUS_AREAS]. Include: [REQUIRED_ELEMENTS]. Cite sources where possible."

**Post 8:**
Pro tip: Name variables clearly

Bad: [X], [Y], [Z]
Good: [TOPIC], [AUDIENCE], [TONE]

You should know what to fill in at a glance.

**Post 9:**
Pro tip: Include defaults in brackets

"[TONE - default: professional]"

When you don't have a preference, use the default.

Saves decision fatigue.

**Post 10:**
Try it today:

Take your most-used prompt.

Identify what changes each time.

Replace those parts with [VARIABLES].

Save it.

My team shares a prompt library.

**Post 2:**
The problem with team ChatGPT usage:

Everyone prompts differently.

Some get great results. Some get garbage.

Quality is inconsistent.

**Post 3:**
The solution: Shared prompt library

Standard prompts everyone uses.

When one person improves a prompt, everyone benefits.

**Post 4:**

Ask everyone: What do you use ChatGPT for?

Find the common tasks.

These become your priority prompts.

**Post 5:**
Step 2: Create standard prompts

For each common task, create one "official" prompt.

Include variables for customization.

Document how to use it.

**Post 6:**
Step 3: Organize by team/function
