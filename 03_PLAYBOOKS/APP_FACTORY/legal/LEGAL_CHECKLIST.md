# Legal Checklist for App Launch

**DISCLAIMER: This is a template requiring legal review before use. Consult with a qualified attorney to ensure compliance with applicable laws.**

---

## Overview

This checklist covers legal requirements for launching a mobile app. Use it as a guide, but always consult with a qualified attorney for your specific situation.

---

## 1. Pre-Launch Legal Requirements

### Essential Documents

- [ ] **Privacy Policy**
  - [ ] Describes all data collection
  - [ ] Lists third-party services and their privacy policies
  - [ ] Includes user rights (access, deletion, portability)
  - [ ] GDPR compliant (if serving EU users)
  - [ ] CCPA compliant (if serving California users)
  - [ ] COPPA compliant (if children might use app)
  - [ ] Hosted at publicly accessible URL
  - [ ] Link in app and app store listing

- [ ] **Terms of Service**
  - [ ] Account terms defined
  - [ ] Subscription terms (if applicable)
  - [ ] Acceptable use policy
  - [ ] Disclaimers appropriate for app type
  - [ ] Limitation of liability
  - [ ] Dispute resolution clause
  - [ ] Termination provisions
  - [ ] Hosted at publicly accessible URL

- [ ] **End User License Agreement (EULA)**
  - [ ] License grant defined
  - [ ] Restrictions listed
  - [ ] Intellectual property provisions
  - [ ] Platform-specific acknowledgments (Apple, Google)

### Additional Documents (As Needed)

- [ ] **Refund Policy** (if handling payments)
- [ ] **DMCA Policy** (if hosting user-generated content)
- [ ] **Cookie Policy** (if app has web component)
- [ ] **Affiliate Agreement** (if running affiliate program)
- [ ] **FTC Disclosures** (if marketing involves endorsements)

## 2. Apple App Store Requirements

### Required Before Submission

- [ ] **Privacy Policy URL**
  - Must be publicly accessible
  - Must accurately describe data practices
  - Required for all apps

- [ ] **App Privacy Details (Nutrition Labels)**
  - [ ] Identify data types collected
  - [ ] Specify if data is linked to user
  - [ ] Specify if data is used for tracking
  - [ ] Update if practices change

- [ ] **Age Rating Questionnaire**
  - [ ] Complete accurately
  - [ ] Consider all content (including user-generated)

- [ ] **Terms of Use URL** (optional but recommended)

### Subscription Apps

- [ ] **Subscription Terms** clearly explained
- [ ] **Free trial terms** disclosed
- [ ] **Auto-renewal disclosed** before purchase
- [ ] **Price clearly displayed**
- [ ] **How to cancel** documented

### In-App Purchases

- [ ] All purchases go through Apple IAP (required)
- [ ] No external payment links (prohibited)
- [ ] "Restore Purchases" button included

### Account Creation

- [ ] "Sign in with Apple" offered if third-party logins exist
- [ ] Account deletion functionality provided
- [ ] Account deletion accessible and easy to complete

### Content Guidelines Compliance

- [ ] No misleading content or functionality
- [ ] No hidden features
- [ ] No app store manipulation
- [ ] User-generated content moderation in place

### Special Categories

- [ ] **Health/Fitness apps:** Include appropriate disclaimers
- [ ] **Finance apps:** Required licenses obtained
- [ ] **Kids apps:** COPPA compliance, no ads or tracking
- [ ] **VPN/Security apps:** Accurate claims, required permissions only

---

## 3. Google Play Store Requirements

### Required Before Submission

- [ ] **Privacy Policy**
  - [ ] Required for all apps
  - [ ] Must be on active URL
  - [ ] Link in Play Console and app

- [ ] **Data Safety Section**
  - [ ] Complete all required fields
  - [ ] Accurate data collection disclosure
  - [ ] Security practices disclosed
  - [ ] Data sharing disclosed

- [ ] **Content Rating Questionnaire**
  - [ ] Complete IARC questionnaire
  - [ ] Rate accurately for all regions

- [ ] **Target Audience Declaration**
  - [ ] Select appropriate age groups
  - [ ] If under 13 included, comply with Families Policy

### Subscription Apps

- [ ] **Google Play Billing** used for subscriptions
- [ ] **Subscription terms** clearly disclosed
- [ ] **Free trial terms** prominently displayed
- [ ] **Grace period** handling defined
- [ ] **Account hold** behavior defined

### Ads and Monetization

- [ ] **Ads disclosure** if app contains ads
- [ ] **Interstitial ads** don't violate user experience policies
- [ ] **Rewarded ads** clearly disclosed

### Account and Data

- [ ] **Account deletion** functionality provided
- [ ] **Data deletion** request process defined
- [ ] **Data export** available (if applicable)

### Sensitive Permissions

- [ ] Request only necessary permissions
- [ ] Explain permission usage to users
- [ ] Handle permission denials gracefully
- [ ] Declare permissions in manifest accurately

### Special Categories

- [ ] **Health apps:** Medical claims substantiated
- [ ] **Financial apps:** Licenses obtained
- [ ] **Kids/Families apps:** Families Policy compliance
- [ ] **Government/Official apps:** Verification obtained

## 4. GDPR Compliance (EU Users)

### Required If Serving EU Users

- [ ] **Lawful basis** for processing identified
  - [ ] Consent
  - [ ] Contract
  - [ ] Legitimate interest
  - [ ] Legal obligation

- [ ] **Privacy Policy** includes:
  - [ ] Controller identity and contact
  - [ ] DPO contact (if required)
  - [ ] Purposes of processing
  - [ ] Lawful basis for each purpose
  - [ ] Data recipients/categories
  - [ ] International transfers and safeguards
  - [ ] Retention periods
  - [ ] User rights explanation
  - [ ] Right to lodge complaint
  - [ ] Automated decision-making disclosure

- [ ] **User Rights** mechanisms:
  - [ ] Right to access
  - [ ] Right to rectification
  - [ ] Right to erasure ("right to be forgotten")
  - [ ] Right to restrict processing
  - [ ] Right to data portability
  - [ ] Right to object
  - [ ] Rights related to automated decisions

- [ ] **Consent Mechanisms**:
  - [ ] Freely given (no dark patterns)
  - [ ] Specific and informed
  - [ ] Unambiguous (clear affirmative action)
  - [ ] Easy to withdraw
  - [ ] Consent records maintained

- [ ] **Data Processing Records**
  - [ ] Processing activities documented
  - [ ] Record of processing operations maintained

- [ ] **Data Protection by Design**
  - [ ] Privacy built into development
  - [ ] Data minimization practiced
  - [ ] Purpose limitation enforced

- [ ] **International Transfers**
  - [ ] Adequate safeguards in place (SCCs, etc.)
  - [ ] Transfer mechanisms documented

- [ ] **Data Breach Procedures**
  - [ ] 72-hour notification process defined
  - [ ] Breach documentation process in place

## 5. CCPA Compliance (California Users)

### Required If Serving California Users AND Meet Thresholds

CCPA applies if you:
- Have gross revenue over $25 million
- Buy/sell/share data of 100,000+ consumers
- Derive 50%+ revenue from selling personal information

### Compliance Requirements

- [ ] **Privacy Policy** includes:
  - [ ] Categories of personal information collected
  - [ ] Sources of personal information
  - [ ] Purposes for collecting/using
  - [ ] Categories of third parties shared with
  - [ ] Description of consumer rights

- [ ] **Consumer Rights** mechanisms:
  - [ ] Right to know (access)
  - [ ] Right to delete
  - [ ] Right to opt-out of sale
  - [ ] Right to non-discrimination

- [ ] **"Do Not Sell" Link**
  - [ ] Prominent on website/app
  - [ ] Functional opt-out process

- [ ] **Verification Process**
  - [ ] Identity verification for requests
  - [ ] Response within 45 days

- [ ] **Training**
  - [ ] Staff trained on CCPA compliance

## 6. COPPA Compliance (Children Under 13)

### Required If App May Be Used by Children Under 13

- [ ] **Parental Consent**
  - [ ] Verifiable consent before collecting data
  - [ ] Consent mechanisms appropriate for age

- [ ] **Privacy Policy**
  - [ ] Describes data collected from children
  - [ ] Explains parental rights
  - [ ] Contact information for questions

- [ ] **Data Minimization**
  - [ ] Collect only necessary data
  - [ ] Delete data when no longer needed

- [ ] **No Behavioral Advertising**
  - [ ] No targeted ads based on child's data

- [ ] **Third-Party Restrictions**
  - [ ] Third parties cannot collect child data
  - [ ] Or have verifiable consent

- [ ] **Parental Access**
  - [ ] Parents can review data
  - [ ] Parents can request deletion
  - [ ] Parents can revoke consent

## 7. Industry-Specific Requirements

### Health/Fitness Apps

- [ ] **Medical Device Regulations**
  - [ ] FDA classification assessed (if applicable)
  - [ ] CE marking (for EU, if medical device)

- [ ] **HIPAA Compliance** (if handling PHI)
  - [ ] Business Associate Agreements
  - [ ] Security safeguards
  - [ ] Breach notification procedures

- [ ] **Health Disclaimers**
  - [ ] Not a substitute for professional medical advice
  - [ ] Consult healthcare provider disclaimers

### Finance/Banking Apps

- [ ] **Financial Regulations**
  - [ ] Required licenses obtained
  - [ ] Regulatory compliance verified

- [ ] **PCI DSS** (if handling card data)
  - [ ] Cardholder data protection
  - [ ] Network security requirements

- [ ] **Investment Disclaimers**
  - [ ] Not investment advice statements
  - [ ] Risk disclosures

### E-Commerce Apps

- [ ] **Consumer Protection Laws**
  - [ ] Return/refund policies
  - [ ] Clear pricing
  - [ ] Delivery terms

- [ ] **Payment Security**
  - [ ] PCI compliance
  - [ ] Secure payment processing

### Education/EdTech Apps

- [ ] **FERPA** (if handling student records)
- [ ] **COPPA** (if children under 13)
- [ ] **State Student Privacy Laws**

## 8. Marketing and Advertising Compliance

### FTC Compliance

- [ ] **Endorsement Guidelines**
  - [ ] Affiliate disclosures in place
  - [ ] Sponsored content disclosed
  - [ ] Material connections disclosed

- [ ] **Advertising Claims**
  - [ ] Claims substantiated
  - [ ] Testimonials representative
  - [ ] Results disclaimers included

- [ ] **Email Marketing (CAN-SPAM)**
  - [ ] Unsubscribe mechanism
  - [ ] Accurate sender information
  - [ ] Clear identification as ad

### Social Media Marketing

- [ ] **Disclosure templates** prepared
- [ ] **Influencer guidelines** documented
- [ ] **Contest/giveaway rules** compliant

## 9. Intellectual Property

### Before Launch

- [ ] **Trademark Search**
  - [ ] App name doesn't infringe existing marks
  - [ ] Consider trademark registration

- [ ] **Copyright**
  - [ ] Original content ownership confirmed
  - [ ] Licensed content properly licensed
  - [ ] User content rights defined in ToS

- [ ] **Third-Party Components**
  - [ ] Open source licenses reviewed
  - [ ] Attribution requirements met
  - [ ] License compatibility verified

- [ ] **DMCA Compliance** (if user content)
  - [ ] Designated agent registered
  - [ ] Takedown procedure documented

## 10. Security and Data Protection

### Technical Requirements

- [ ] **Encryption**
  - [ ] Data encrypted in transit (TLS/SSL)
  - [ ] Sensitive data encrypted at rest

- [ ] **Authentication**
  - [ ] Secure password requirements
  - [ ] Session management
  - [ ] MFA option (if appropriate)

- [ ] **Access Controls**
  - [ ] Principle of least privilege
  - [ ] Admin access secured

- [ ] **Vulnerability Management**
  - [ ] Security testing performed
  - [ ] Dependency scanning
  - [ ] Regular updates process

### Incident Response

- [ ] **Breach Notification Plan**
  - [ ] Detection procedures
  - [ ] Notification timelines (GDPR: 72 hours)
  - [ ] User communication templates

- [ ] **Data Backup**
  - [ ] Regular backups
  - [ ] Recovery testing

## 13. Quick Reference: Launch Blockers

**DO NOT LAUNCH WITHOUT:**

1. Privacy Policy accessible via URL
2. Terms of Service accessible via URL
3. Privacy Policy link in app store listing
4. Data collection accurately described
5. Age rating completed
6. Required permissions explained
7. Subscription terms clearly disclosed (if applicable)
8. Account deletion available (required by Apple and Google)
9. Sign in with Apple (if other third-party logins exist)
