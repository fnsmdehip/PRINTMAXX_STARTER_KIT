================================================================================
                       ARCHITECTURE OVERVIEW
================================================================================

DATA FLOW:
  User Input (UI) → Zustand Store → AsyncStorage (persistent)
                 ↓
              Screens/Components (re-render on state change)

NAVIGATION:
  app/ (file-based routes)
    ├── index.tsx → checks onboarding → redirects
    ├── onboarding.tsx → first-time setup
    └── (tabs)/ → 4 main tabs
         ├── home → library of prompts
         ├── favorites → saved prompts
         ├── improve → enhancement tool
         └── settings → preferences

STATE MANAGEMENT (Zustand + AsyncStorage):
  - promptStore: Prompt data, search, filtering
  - favoriteStore: Saved prompt IDs
  - subscriptionStore: Pro/Free status, trial
  - onboardingStore: Setup preferences, completion

================================================================================
                          TROUBLESHOOTING
================================================================================

Common issues and solutions:

"Module not found: expo-router"
  → Run: npm install

"Cannot find ./assets/icon.png"
  → Run: cp ../promptvault/assets/* ./assets/

"App crashes on startup"
  → Clear cache: expo start -c
  → Check assets exist

"Onboarding appears every time"
  → Check AsyncStorage permissions
  → Verify useOnboardingStore.completeOnboarding() called
  → Check app.json has expo-router plugin
