# Text testimonial template

Formats for displaying written testimonials on websites, landing pages, and marketing materials.

---

## Basic testimonial format

```
"[Quote - 1-3 sentences, specific and authentic]"

- [Full Name], [Role/Title], [Company or Context]
```

**Example:**
```
"I switched from [competitor] after 3 years. [App Name] does the same thing in half the time. Setup took 20 minutes."

- Marcus Chen, Freelance Designer
```

---

## Testimonial with result

```
"[Quote focusing on outcome]"

- [Full Name], [Role]
[Specific result: metric or outcome]
```

**Example:**
```
"Finally an app that doesn't make me think. I just open it and start working."

- Jamie Rodriguez, Project Manager
Saves 4+ hours per week on task management
```

---

## Before/after testimonial

```
BEFORE: [Problem state in their words]
AFTER: [Improved state in their words]

- [Name], [Role]
```

**Example:**
```
BEFORE: "Spent Sunday nights dreading Monday's inbox"
AFTER: "Inbox zero by 10am, every day"

- Taylor Brooks, Startup Founder
```

## Testimonial with photo

```
[Headshot photo - square, clear face, professional or casual depending on brand]

"[Quote]"

[Full Name]
[Role], [Company]
[Optional: Location or context]
```

**Photo requirements:**
- Minimum 200x200px
- Clear face visible
- Consistent style across testimonials
- Get explicit photo permission

## Long-form testimonial

Use for landing pages and case study previews.

```
## What [Name] says about [App Name]

"[Opening hook - the transformation or main benefit]"

[Paragraph 1: The problem they had]

[Paragraph 2: How they found/started using the app]

[Paragraph 3: Specific results and benefits]

"[Closing quote - would they recommend it]"

**[Full Name]**
[Role], [Company]
[Optional: Link to full case study]
```

## Collection guidelines

### What makes a good testimonial

**Strong testimonials include:**
- Specific problem they had
- Specific outcome or result
- Comparison to alternatives (if applicable)
- Timeframe (how long to see results)
- Quantifiable metrics when possible

**Weak testimonials to avoid:**
- "Great app!" (too vague)
- "I love it!" (no substance)
- "5 stars!" (just a rating)
- Anything you wrote for them

### Editing rules

**You can:**
- Fix obvious typos
- Remove filler words ("um", "like", "you know")
- Trim for length (indicate with ...)
- Add context in brackets [like this]

**You cannot:**
- Change meaning
- Add claims they didn't make
- Make results sound better than stated
- Remove caveats or qualifications
- Combine quotes from different conversations

**Always:** Send edited version for approval before publishing.

## Attribution requirements

### Minimum attribution
- First name + Last initial: "Sarah M."
- Context: "Small business owner"

### Standard attribution
- Full name: "Sarah Miller"
- Role: "Marketing Manager"
- Company type (if not specific company): "at a SaaS startup"

### Full attribution
- Full name: "Sarah Miller"
- Specific role: "Head of Marketing"
- Company name: "TechCorp Inc."
- Photo
- Link to profile (optional)

### Anonymous (last resort)
- Role only: "Marketing Manager at a Fortune 500 company"
- Must have written permission to use anonymously
- Less credible than named testimonials

## Display locations

### Landing page hero
- 1-2 short, punchy testimonials
- Photo adds credibility
- Place near CTA

### Testimonial section
- 3-6 testimonials in grid or carousel
- Mix of roles/industries if applicable
- Include ratings if available

### Pricing page
- Testimonials about value/ROI
- "Worth every penny" type quotes
- Near pricing table

### Feature pages
- Testimonials specific to that feature
- Match quote to feature being described

### Checkout/signup page
- Trust-building testimonials
- Reduce purchase anxiety
- Social proof for conversion

---

## FTC compliance

- All testimonials must be from real users
- Must reflect genuine opinions
- Disclose material connections (free product, payment, etc.)
- Results must be typical OR include disclaimer
- Keep records of permission granted
