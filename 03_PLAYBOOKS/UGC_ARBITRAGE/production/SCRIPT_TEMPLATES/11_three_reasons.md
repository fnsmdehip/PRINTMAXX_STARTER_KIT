# Script template: Three reasons

List format content. Easy to follow, highly shareable.

---

## Structure

```
[HOOK - 2-3 seconds]
Promise the list

[REASON 1 - 8-12 seconds]
First and often strongest point

[REASON 2 - 8-12 seconds]
Supporting point

[REASON 3 - 8-12 seconds]
Final compelling point

[SUMMARY/CTA - 5-8 seconds]
Tie together and direct
```

---

## Example script: SaaS product (45 seconds)

```
HOOK:
"3 reasons I switched to [Product] and won't go back."

REASON 1:
"First - it's actually simple. Most project management tools have a million features you'll never use. [Product] does the core stuff really well. I was set up in 10 minutes."

REASON 2:
"Second - the price. I was paying $15 per user with [Competitor]. [Product] is $8. For my team of 5, that's $420 saved per year."

REASON 3:
"Third - it integrates with everything I already use. Slack, Google Calendar, email. No more tab switching."

SUMMARY/CTA:
"Simple, cheap, works with my stack. That's why I switched. Link to try it free below."
```

## List hook formulas

### Number + Benefit
- "3 reasons I love [Product]"
- "5 things [Product] does better"
- "4 ways [Product] changed my [routine/workflow]"

### Number + Challenge
- "3 reasons you need to try [Product]"
- "5 things I wish I knew before [action]"
- "4 mistakes [audience] makes with [category]"

### Number + Outcome
- "3 things that happened when I started using [Product]"
- "5 results I got from [Product]"
- "4 changes I noticed in [timeframe]"

---

## Reason ordering strategies

### Strongest first
- Lead with best point
- Captures attention
- Works for shorter content

### Build to strongest
- Save best for last
- Creates anticipation
- Works for engaged viewers

### Sandwich
- Strong - Medium - Strong
- Keeps attention throughout
- Most versatile

---

## Reason frameworks

### Feature-benefit
"[Feature] - which means [benefit]."
"It has [feature], so you can [benefit]."

### Problem-solution
"[Problem]? [Product] solves this with [solution]."
"Most products [problem]. This one [solution]."

### Comparison
"Unlike [alternatives], this one [difference]."
"[Competitors] do X. [Product] does Y instead."

### Personal result
"Since using this, [result]."
"This helped me [achieve outcome]."

---

## Transition phrases

### Between reasons
- "Second..."
- "Next..."
- "Also..."
- "Another thing..."
- "And finally..."

### Adding emphasis
- "But the best part..."
- "What really sold me..."
- "This one's huge..."
- "Here's the game changer..."

---

## Number variations

### 3 reasons (optimal)
- Most common format
- Easy to remember
- Good for 30-45s content

### 5 reasons
- More comprehensive
- Works for longer content
- Good for complex products

### 1 reason (focused)
- "The #1 reason I use [Product]"
- Intensely focused
- Great for short form

### 7-10 reasons
- Listicle style
- Works with fast cuts
- Good for rapid-fire content

---

## Visual enhancement

### On-screen text
- Number each reason: "1.", "2.", "3."
- Key word highlights
- Makes content skimmable

### Visual proof
- Show feature while discussing
- Screenshots/demos
- Before/after for results

### Pacing
- Pause between reasons
- Let each point land
- Don't rush transitions

## Alternative list formats

### Things I love/hate
- "3 things I love, 1 thing I'd change"
- Adds credibility through honesty

### Before/after list
- "3 things that changed when I started using [Product]"
- Results-focused

### Mistake list
- "3 mistakes I made before finding [Product]"
- Problem-focused

### Comparison list
- "3 ways [Product] beats [competitor type]"
- Competitive positioning

## Creator brief notes

For list content:
- Provide exact 3 points to cover
- Note order of importance
- Include talking points for each
- Specify any must-say phrases
- Allow natural delivery within structure
