# Script template: Secret/hack reveal

Insider knowledge format. High curiosity, high engagement.

---

## Structure

```
[HOOK - 2-3 seconds]
Tease the secret

[SETUP - 5-8 seconds]
Why this matters

[THE REVEAL - 15-20 seconds]
The actual secret/hack

[PROOF/APPLICATION - 10-12 seconds]
Show it working

[CTA - 3-5 seconds]
How to get/do it
```

---

## Example script: Productivity (30 seconds)

```
HOOK:
"The productivity hack nobody talks about."

SETUP:
"Everyone focuses on time management. But the real bottleneck isn't time - it's energy."

THE REVEAL:
"I stopped scheduling by hours and started scheduling by energy. High-focus work goes in my first 3 hours. Meetings and emails go after lunch when my brain is slower anyway."

PROOF/APPLICATION:
"Since doing this, I get twice as much done in half the time. [Product] makes this easy because [feature that enables it]."

CTA:
"Try it. Link to the tool I use below."
```

## Hook formulas

### The secret
- "The secret to [outcome] that nobody tells you"
- "What [experts] don't want you to know about [topic]"
- "The hidden truth about [category]"

### The hack
- "This [category] hack changed everything"
- "The hack that took me from [before] to [after]"
- "A simple hack for [problem]"

### The insider knowledge
- "A [professional] told me this and..."
- "I learned this from [credible source]"
- "This is what [successful people] actually do"

### The thing nobody talks about
- "Why nobody talks about this"
- "The [topic] tip everyone ignores"
- "What I wish someone told me about [topic]"

---

## Secret/hack categories

### Technique secrets
- Different way to use a product
- Order of operations that matters
- Timing that makes difference

### Access secrets
- Where to find better deals
- Who to follow for info
- What to ask for

### Mindset secrets
- Reframe that changes everything
- Question to ask yourself
- Perspective shift

### System secrets
- Automation that saves time
- Setup that compounds
- Process that prevents problems

---

## Building credibility for secrets

### Source attribution
- "A [professional] told me..."
- "I learned this from [credible person]"
- "Research shows..."
- "After [X years] of doing this..."

### Personal proof
- "This changed [specific result] for me"
- "I went from [before] to [after]"
- "I've been doing this for [time]"

### Logical explanation
- "It works because [mechanism]"
- "The reason this matters..."
- "Most people don't realize that..."

---

## Reveal pacing

### Build anticipation
- Don't give away immediately
- Let curiosity build
- Use setup to create desire

### Deliver clearly
- State the secret/hack plainly
- Don't overcomplicate
- Make it actionable

### Prove it works
- Show result or application
- Connect to product naturally
- Make viewer believe

---

## Timing breakdown

| Section | Duration | Emotional state |
|---------|----------|-----------------|
| Hook | 2-3s | Curiosity |
| Setup | 6-8s | Interest |
| Reveal | 15-18s | "Aha" moment |
| Proof | 10-12s | Belief |
| CTA | 3-5s | Action |
| **Total** | ~40s | |

---

## What makes a good "secret"

### It should be
- Genuinely useful
- Not widely known (or presented fresh)
- Easy to implement
- Connected logically to product

### It should NOT be
- Misleading or false
- Overhyped for minor benefit
- Requiring expensive investment
- Dangerous or unethical

---

## Connecting secret to product

### Direct connection
"The secret is [technique], and [Product] makes this easy because [feature]."

### Enabler connection
"Once I learned this, I needed a tool to implement it. [Product] is perfect for this."

### Discovery connection
"I discovered this while using [Product]. [Feature] led me to realize..."

## Creator brief notes

For secret/hack content:
- Provide the actual secret/hack
- Explain why it works (for authentic delivery)
- Note how it connects to product
- Include any credibility sources
- Specify if can/should name the source
