# UGC Quality Checklist

**Updated:** 2026-01-20
**Purpose:** How to evaluate UGC content quality before delivery to clients

---

## Quality Framework Overview

### The Three Pillars

```
QUALITY = Technical Quality × Performance Quality × Brand Fit

Where:
- Technical: Video/audio specs, lighting, framing
- Performance: Delivery, authenticity, engagement
- Brand Fit: Message alignment, tone, audience match
```

### Quality Grading Scale

| Grade | Score | Description | Action |
|-------|-------|-------------|--------|
| A | 90-100 | Exceptional, ready to ship | Deliver |
| B | 75-89 | Good, minor issues | Deliver with notes |
| C | 60-74 | Acceptable, some issues | Consider revisions |
| D | 40-59 | Below standard | Request revision |
| F | <40 | Unacceptable | Reject, reshoot |

## Revision Request Guidelines

### How to Request Revisions

**Be specific:**
- Timestamp the issue (e.g., "At 0:15...")
- Describe exactly what's wrong
- Explain what you need instead
- Provide examples if helpful

**Be constructive:**
- Start with what works
- Then address what needs fixing
- End with encouragement

**Template:**
```
Hi [Creator],

Thanks for the first version! Here are some notes:

WHAT WORKS:
- [Positive point]
- [Another positive]

REVISION REQUESTS:
1. [Timestamp] - [Issue] - Please [specific fix]
2. [Timestamp] - [Issue] - Please [specific fix]

Everything else looks great! Let me know if you
have questions.

Thanks!
```

### Common Revision Categories

| Category | Example Request |
|----------|-----------------|
| Audio | "Please re-record audio in quieter space" |
| Lighting | "Can you reshoot with light facing you?" |
| Script | "Please include the product name in hook" |
| Delivery | "A bit more energy in the hook" |
| Length | "Please trim to under 45 seconds" |
| Framing | "Keep product in frame during demo" |

### Revision Limits

**Standard policy:**
- 1-2 revisions included
- Additional revisions = extra fee
- Major reshoots = renegotiate

**When to reject vs revise:**
- Fixable in editing = accept
- Needs minor reshoot = revision request
- Fundamentally wrong = reject/reshoot

## Quality Control Workflow

### Step 1: Initial Review (2 min)

**Quick scan for dealbreakers:**
- Is it the right product?
- Is length acceptable?
- Is quality acceptable at a glance?
- Is the script followed?

If major issue → immediate revision request

### Step 2: Detailed Review (5 min)

**Full quality checklist:**
- Technical elements
- Performance elements
- Brand fit elements
- Score each section

### Step 3: Decision (1 min)

Based on score:
- 90+: Approve immediately
- 75-89: Approve with notes
- 60-74: Consider revision
- <60: Revision required

### Step 4: Documentation (1 min)

**Record:**
- Score
- Decision
- Notes for creator
- Notes for client (if applicable)
- File organization

## Creator Feedback System

### Feedback Categories

| Category | When to Use |
|----------|-------------|
| Quick approval | 90+ score, no notes needed |
| Approval with praise | Good work, want to encourage |
| Approval with tips | Good but could improve |
| Revision request | Below standard, fixable |
| Rejection with explanation | Fundamentally wrong |

### Building Creator Relationships

**Good feedback habits:**
- Be prompt (within 24-48 hours)
- Be specific (not just "looks good")
- Be constructive (help them improve)
- Be consistent (same standards always)
- Be appreciative (they're working for you)

**Feedback that improves creators:**
- "The hook was great - that energy is perfect"
- "Next time, try facing the window for better light"
- "Your delivery is natural - keep that authentic feel"
- "The product demo was clear - that's exactly what we need"

## Red Flags Checklist

### Immediate Rejection

| Red Flag | Why |
|----------|-----|
| Wrong product shown | Misleading to viewers |
| Competitor visible | Brand safety issue |
| False claims made | Legal liability |
| Copyright music/content | Legal liability |
| Inappropriate content | Brand safety |
| Clearly AI-generated | Authenticity issue |

### Revision Required

| Red Flag | Why |
|----------|-----|
| Script not followed | Missing key messages |
| Audio unusable | Can't understand |
| Lighting too dark | Unprofessional |
| Too long/short | Doesn't meet specs |
| No CTA | Missing conversion point |
| Visible script reading | Not authentic |

### Minor Issues (Accept with Notes)

| Issue | Note to Include |
|-------|-----------------|
| Slightly off-brand | "For next time, try X tone" |
| Small audio issue | "We'll fix in edit" |
| Minor framing issue | "Cropped slightly in edit" |
| Subtle delivery issue | "Great, even better with X energy" |

## Platform-Specific Quality Notes

### TikTok

**Must have:**
- Strong hook (first 1 second)
- Vertical 9:16
- Native feel (not over-produced)
- Trending sound potential

**Watch for:**
- Over-produced = doesn't perform
- Too polished = feels like an ad

### Instagram Reels

**Must have:**
- 9:16 vertical
- High visual quality
- Clear audio
- Professional but authentic

**Watch for:**
- Can be slightly more polished than TikTok
- Carousel-style edits work

### YouTube Shorts

**Must have:**
- 9:16 vertical
- Good audio (YouTube prioritizes)
- Under 60 seconds
- Strong hook

**Watch for:**
- YouTube more forgiving on production quality
- Audio matters more than video

### Facebook Ads

**Must have:**
- Multiple aspect ratios often needed
- Can be more direct/sales-focused
- Caption-friendly
- Works with sound off

**Watch for:**
- May need 1:1 and 4:5 versions
- Test different hooks
