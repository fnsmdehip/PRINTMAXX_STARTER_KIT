# Quality control process

Systematic approach to ensuring consistent quality.

---

## QC philosophy

**Quality is your competitive advantage.**

You can compete on price (race to bottom) or quality (sustainable margin).

Every piece of content reflects on your reputation. One bad delivery can lose a client. Consistent quality builds trust and referrals.

---

## Three-stage QC process

### Stage 1: Pre-production QC

Before content is created:

**Brief QC checklist:**
- [ ] All client requirements captured
- [ ] Talking points clear and complete
- [ ] Visual requirements specified
- [ ] Product details accurate
- [ ] Compliance requirements noted
- [ ] References provided
- [ ] Timeline realistic

**Creator match QC:**
- [ ] Creator style fits brief
- [ ] Creator has capacity
- [ ] Creator understands requirements
- [ ] Product shipped/accessible
- [ ] Timeline confirmed

---

### Stage 2: Production QC

While content is being created:

**Midpoint check:**
- Is creator on track?
- Any questions or issues?
- Does sample footage look good? (if available)
- Timeline still achievable?

**First draft review:**
- Initial watch without notes
- Detailed technical check
- Content alignment check
- Performance quality check

---

### Stage 3: Final QC

Before client delivery:

**Technical checklist:**
- [ ] Resolution: 1080p minimum
- [ ] Aspect ratio: Correct for platform
- [ ] Audio: Clear, no noise
- [ ] Lighting: Face visible, even
- [ ] Focus: Sharp throughout
- [ ] Stability: No unwanted shake
- [ ] Length: Within spec (+/- 3 seconds)

**Content checklist:**
- [ ] All talking points covered
- [ ] Product name mentioned correctly
- [ ] Key benefits stated
- [ ] CTA included
- [ ] No compliance issues
- [ ] Brand-safe

**Performance checklist:**
- [ ] Hook grabs attention
- [ ] Delivery natural, not scripted
- [ ] Energy level appropriate
- [ ] Believable/authentic
- [ ] Would you keep watching?

## QC scoring system

### Technical score (1-5)

| Score | Definition |
|-------|------------|
| 5 | Perfect technical quality |
| 4 | Minor issues, client-ready |
| 3 | Noticeable issues, may need revision |
| 2 | Significant issues, revision required |
| 1 | Unacceptable, reshoot required |

### Content score (1-5)

| Score | Definition |
|-------|------------|
| 5 | All requirements exceeded |
| 4 | All requirements met |
| 3 | Most requirements met, minor gaps |
| 2 | Missing key requirements |
| 1 | Does not meet brief |

### Performance score (1-5)

| Score | Definition |
|-------|------------|
| 5 | Exceptional, would perform great |
| 4 | Strong, confident in performance |
| 3 | Acceptable, should work |
| 2 | Weak, concerned about performance |
| 1 | Would not convert |

### Overall score calculation

```
Overall = (Technical x 0.3) + (Content x 0.3) + (Performance x 0.4)

4.5-5.0: Excellent - Deliver as-is
4.0-4.4: Good - Deliver with minor notes
3.5-3.9: Acceptable - Consider revision
3.0-3.4: Below standard - Revision required
<3.0: Unacceptable - Reshoot or replace
```

## Common QC issues

### Technical issues

| Issue | How to spot | Action |
|-------|-------------|--------|
| Low resolution | Pixelated, blurry | Request original file |
| Bad audio | Noise, echo, quiet | Request re-record or VO |
| Poor lighting | Dark, shadows, uneven | Request reshoot with guidance |
| Out of focus | Blurry subject | Request reshoot |
| Wrong format | Horizontal when vertical | Request re-export |

### Content issues

| Issue | How to spot | Action |
|-------|-------------|--------|
| Missing talking point | Review against brief | Request pickup shot |
| Wrong product name | Listen carefully | Request re-record section |
| No CTA | End without action | Request pickup |
| Compliance problem | Review claims | Request edit/reshoot |
| Off-brand tone | Compare to guidelines | Provide feedback, revise |

### Performance issues

| Issue | How to spot | Action |
|-------|-------------|--------|
| Reading script | Eyes moving, stilted | Request memorized delivery |
| Low energy | Flat, boring | Request more enthusiasm |
| Not believable | Feels fake | Provide coaching, consider new creator |
| Wrong pacing | Too fast/slow | Request pace adjustment |
| Poor eye contact | Looking away | Request camera focus |

## QC workflow

### Daily QC routine

```
1. Check for new deliveries
2. Download/access files
3. Watch each piece once (first impression)
4. Detailed review with checklist
5. Score each piece
6. Document issues
7. Prepare feedback/approval
8. Communicate with creators
9. Prepare client delivery
```

### QC time allocation

| Content type | QC time |
|--------------|---------|
| 15s video | 5-10 min |
| 30s video | 10-15 min |
| 60s video | 15-20 min |
| Photo set | 10-15 min |
| Package (5 vids) | 45-60 min |

## Handling QC failures

### When revision is needed

1. Document specific issues
2. Provide actionable feedback
3. Include timestamps
4. Offer references/examples
5. Set clear deadline
6. Track revision

### When reshoot is needed

1. Assess what went wrong
   - Brief unclear?
   - Creator capability?
   - Circumstances?

2. Decide next steps
   - Same creator with clearer direction
   - Different creator
   - Scope/rate discussion

3. Communicate clearly
   - What needs to change
   - Timeline
   - Any additional compensation

### When to reject entirely

Reject if:
- Multiple reshoots don't fix issues
- Creator unable to meet basic requirements
- Content fundamentally unsuitable
- Relationship not working

Process:
- Pay for work done (partial or full)
- Provide honest feedback
- Remove from roster if necessary
- Find replacement quickly

## Quality improvement

### Learning from QC

Track patterns:
- What issues recur?
- Which creators have issues?
- Are briefs the problem?
- Are expectations misaligned?

### Brief improvement

Common brief gaps:
- Not enough visual guidance
- Vague on energy/tone
- Missing reference examples
- Incomplete product details

Fix: Update brief template after each learning.

### Creator development

Share feedback that helps:
- "This worked really well because..."
- "Next time, try..."
- "Here's an example of what we need..."

Invest in creators improving quality.

### Process improvement

Regular review:
- Is QC catching issues?
- Are clients getting revisions?
- Is QC taking too long?
- What can be systematized?

## Client-ready standards

### Minimum acceptable quality

Content should meet ALL:
- [ ] Technical specs correct
- [ ] All talking points covered
- [ ] Natural, believable delivery
- [ ] You'd be confident sharing
- [ ] No obvious issues to explain

### When to discuss issues with client

**Don't mention:**
- Minor technical variations within spec
- Small deviations if quality good
- Issues you've already addressed

**Do mention:**
- Anything that might affect performance
- Deviations from specific requests
- Limitations of what was possible
- Recommendations for future
