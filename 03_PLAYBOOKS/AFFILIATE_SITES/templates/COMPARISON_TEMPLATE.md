# Comparison Article Template (X vs Y)

Use this template for head-to-head product comparisons (e.g., "Claude vs ChatGPT", "Bowflex vs PowerBlock").

---

## Meta Information

```
Title Tag: [Product A] vs [Product B]: [Hook] ([Year])
Meta Description: [Product A] or [Product B]? We tested both. Here's which wins for [use case 1], [use case 2], and which is best overall.
Target Keyword: [product a] vs [product b]
Secondary Keywords: [product a] or [product b], [product a] compared to [product b]
```

**Example:**
- Title: Claude vs ChatGPT: I Used Both for 30 Days (2024 Comparison)
- Meta: Claude or ChatGPT? After 30 days testing both on real work, here's which wins for writing, coding, and daily use.

---

## Article Structure

### 1. Hook (100-150 words)

```markdown
[The decision readers are facing]

[Your credibility - how you tested]

[Promise of clear recommendation]
```

**Example:**
> If you're choosing between Claude and ChatGPT, you're not alone. Both cost $20/month for premium, and both are genuinely useful. I spent 30 days using both for every task to find out which is actually better. Here's my honest comparison.

---

Brief explanation of what the specs mean for real-world use.

---

### 4. Category Comparisons (1,500-2,000 words)

For each major comparison category:

```markdown
### [Category]: [Winner] Wins

**[Product A]:**
- [Specific strength/weakness]
- [Example or evidence]
- [Screenshot if applicable]

**[Product B]:**
- [Specific strength/weakness]
- [Example or evidence]
- [Screenshot if applicable]

**Verdict:** [Clear statement of which wins this category and why]
```

**Example categories:**
- Writing quality
- Ease of use
- Price/value
- Features
- Performance
- Durability
- Support/community

Include 4-6 comparison categories depending on product type.

---

### 5. Side-by-Side Test Results (300-400 words)

If applicable, show identical tests on both products:

```markdown

**Setup:** [What we tested]

[Outcome, time, quality]

**[Product B] Result:**
[Outcome, time, quality]

**Winner:** [Product] - [Why]
```

Include 2-3 specific tests with evidence (screenshots, photos, metrics).

- [Specific need/trait]

---

### 8. Can You Use Both? (100-150 words)

Address whether using both makes sense:
- Cost consideration
- When it's worth it
- How to use them together (if applicable)

---

### 9. FAQ (200-400 words)

Answer 4-6 common questions:
- Is [A] better than [B]?
- Is [A] worth it over [B]?
- Can I switch from [A] to [B]?
- [Specific use case] - which is better?
- Which has better [specific feature]?

Use FAQ schema markup.

---

### 10. Final Verdict (200-250 words)

```markdown
### The Winner: [Product]

[Summary of why it wins overall]

[Acknowledge when the other product is better]

[Clear recommendation]

**[Product A] - [Try/Buy CTA]**
**[Product B] - [Try/Buy CTA]**
```

## Affiliate Link Placement

1. **Quick verdict box** - Links to both products
2. **Specs comparison table** - Links in product names
3. **After each category winner** - Link to that product
4. **Pricing section** - Links to pricing pages
5. **Final verdict** - Strong CTAs for both

**CTA Examples:**
- "Try [Product A]" / "Try [Product B]"
- "Check [Product A] Price" / "Check [Product B] Price"
- "Get [Product A]" / "Get [Product B]"

---

## Comparison Fairness Rules

1. **Test both equally** - Same conditions, same tasks
2. **Update when products change** - Features evolve
3. **Acknowledge subjectivity** - Preferences vary
4. **Don't force a winner** - Tie is okay if true
5. **Disclose affiliates** - Which you earn from
