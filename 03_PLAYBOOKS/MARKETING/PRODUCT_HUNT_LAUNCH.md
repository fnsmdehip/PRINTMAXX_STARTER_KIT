# Product Hunt Launch Playbook

**Last Updated:** 2026-01-23
**Status:** Ready for app launches

## Quick Start Checklist

- [ ] Product Hunt Maker profile created
- [ ] Ship page created (pre-launch)
- [ ] Hunter identified (or self-hunt)
- [ ] 200+ supporters lined up
- [ ] Launch assets ready
- [ ] Launch day scheduled (Tue-Thu)

## Why Product Hunt

**Average results for Product of the Day:**
- 2,000-5,000 website visits
- 500-2,000 signups
- Press/blog coverage
- Backlinks for SEO
- Social proof badge

**Best for:**
- B2B SaaS
- Developer tools
- Productivity apps
- AI products
- Mobile apps (increasingly)

## Pre-Launch (2-4 Weeks Before)

### 1. Create Maker Profile

```
1. Go to producthunt.com
2. Sign up / Connect Twitter
3. Complete profile (photo, bio, links)
4. Follow other makers
5. Engage with products (comment, upvote)
```

### 2. Build Community First

**Ship Page:**
- Create a "Ship" page for your product
- Collect email subscribers
- Build anticipation
- Notify subscribers on launch day

**Target:** 200+ Ship subscribers before launch

### 3. Find a Hunter (Optional)

**Benefits of having a hunter:**
- Their followers get notified
- Adds credibility
- Experienced hunters know the game

**How to find hunters:**
1. Top hunters list: producthunt.com/hunters/leaderboard
2. DM on Twitter with personalized pitch
3. Offer early access / feature request

**Self-hunting is fine** if you have your own community.

### 4. Line Up Supporters

**You need 200+ upvotes in first 4 hours:**

- Personal network (friends, family, colleagues)
- Ship page subscribers
- Email list
- Twitter/X followers
- Discord/Slack communities
- Previous customers

**Create a DM/email template:**

```
Hey [Name]!

I'm launching [Product] on Product Hunt tomorrow at 12:01 AM PT.

Would mean a lot if you could upvote and leave a comment when it goes live.

I'll send you the link right when it launches.

Thanks!
```

## Launch Assets

### Required Assets

| Asset | Spec | Notes |
|-------|------|-------|
| Name | 60 chars max | Clear, memorable |
| Tagline | 60 chars max | Value prop in one line |
| Description | 260 chars | Expand on tagline |
| Thumbnail | 240x240 PNG | App icon works |
| Gallery | 1270x760 | 3-5 images/GIFs |
| Maker comment | No limit | First comment strategy |

### Tagline Formula

```
[Action verb] [what it does] [unique angle/benefit]

Examples:
- "Lock your phone until you pray"
- "The AI that writes your cold emails"
- "Turn screen time into exercise"
```

### Gallery Images

1. **Hero shot** - Main value prop visualization
2. **Feature 1** - Key differentiator
3. **Feature 2** - Second key feature
4. **Social proof** - Stats, testimonials (if any)
5. **How it works** - Simple 3-step visual

**GIFs outperform static images.** Show the product in action.

### First Comment (Critical)

The maker's first comment sets the tone. Write it in advance.

**Structure:**

```
Hey Product Hunt! 👋

I'm [Name], maker of [Product].

**Why I built this:**
[1-2 sentences about the problem you experienced]

**What it does:**
[3 bullet points of key features]

**What's next:**
[Roadmap teaser]

I'd love your feedback - what features would make this more useful for you?

[Signature]
```

## Launch Day Timeline

### Night Before

- [ ] Finalize all assets
- [ ] Draft first comment
- [ ] Alert supporters (send reminder)
- [ ] Set alarm for 12:01 AM PT

### 12:01 AM PT - LAUNCH

```
1. Product goes live
2. Post first comment immediately
3. Send launch link to all supporters
4. Start responding to comments
```

### First 4 Hours (Critical)

**The algorithm weighs early momentum heavily.**

- Respond to EVERY comment within 30 minutes
- Like and reply to positive comments
- Address questions/concerns directly
- Share on Twitter/X with launch link
- Update Ship page with launch link

### Throughout Day

- Continue responding to all comments
- Share milestone updates (if trending)
- Engage with other launches (builds goodwill)
- Monitor ranking

### End of Day

- Thank supporters
- Share final ranking
- Follow up with interested users

## Comment Response Strategy

### Types of Comments

**Positive feedback:**
```
"Love this concept!"

Reply: "Thanks [Name]! Really appreciate the support.
Anything specific you'd like to see added?"
```

**Feature requests:**
```
"Would be cool if it had [feature]"

Reply: "Great idea! Adding this to the roadmap.
Would you want [variation A] or [variation B]?"
```

**Criticism:**
```
"This seems like [existing product]"

Reply: "Good question! The main difference is [differentiation].
We're focused specifically on [niche] which [competitor] doesn't serve well."
```

**Questions:**
```
"Does this work on Android?"

Reply: "Yes! Available on both iOS and Android.
Here's the direct link: [link]"
```

## Ranking Factors

### What PH Algorithm Considers

1. **Upvote velocity** - Early upvotes matter most
2. **Upvote quality** - Active PH users > new accounts
3. **Comments** - Engagement signals interest
4. **Saves** - People saving for later
5. **Click-through** - Traffic to your site

### What Doesn't Help

- Fake accounts (will get caught)
- Buying upvotes (against TOS)
- Mass DMs to strangers
- Gaming the system (algorithmic detection)

### Launch Time

**12:01 AM PT** - Maximum time in ranking

Products are ranked within a 24-hour window starting at 12:01 AM Pacific Time.

## Common Mistakes

1. **Not building supporters first** - Launch to crickets
2. **Poor first comment** - Missed engagement opportunity
3. **Not responding to comments** - Looks abandoned
4. **Launching on Friday** - Wasted momentum
5. **Asking for upvotes explicitly** - Against spirit
6. **Only upvotes, no comments** - Suspicious pattern

## App-Specific Considerations

### Mobile App Launches

**Challenges:**
- Friction to try (need to download)
- Can't demo in browser

**Solutions:**
- GIF showing app in action
- Clear App Store links
- Web landing page with more info
- Video demo in gallery
