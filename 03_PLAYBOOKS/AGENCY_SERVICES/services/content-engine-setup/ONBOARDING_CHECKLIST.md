# Client Onboarding Checklist - Content Engine Setup

## Pre-kickoff

### Client tasks
- [ ] Sign proposal/SOW
- [ ] Pay 50% deposit
- [ ] Complete client intake form
- [ ] Gather 3-5 sample pieces of content
- [ ] Identify content you admire (voice reference)

### Your tasks
- [ ] Send welcome email
- [ ] Create client project folder
- [ ] Review their existing content
- [ ] Note voice/tone characteristics
- [ ] Schedule kickoff call

---

## Kickoff call (30-45 min)

### Agenda
1. Confirm content types and platforms
2. Discuss brand voice and tone preferences
3. Review sample content together
4. Identify "golden" examples (content that performed well)
5. Clarify posting frequency goals
6. Walk through timeline
7. Discuss access requirements

### Voice calibration questions
- How would you describe your brand voice in 3 words?
- What content creators do you admire?
- What should your content never sound like?
- Any phrases or words to always use/avoid?

### After kickoff
- [ ] Send meeting notes
- [ ] Send access request list
- [ ] Document voice guidelines
- [ ] Request content samples if not received

---

## Access collection

### Content source access

**Notion:**
- Invite to workspace or specific database
- Share template if using Notion to CMS

**Google Docs:**
- Shared folder access
- Or specific document links

**WordPress/CMS:**
- API access or RSS feed URL
- Application password if needed

**Podcast host (Transistor, Buzzsprout, etc.):**
- RSS feed URL
- API key if available

**Video (YouTube, Loom):**
- Channel API access
- Or manual transcript workflow

### Publishing access

**Twitter/X:**
- Connect via OAuth (Buffer/Typefully/direct)
- Or manual copy-paste workflow

**LinkedIn:**
- Connect via scheduling tool
- Native posting requires manual step

**Newsletter (ConvertKit, Beehiiv, etc.):**
- API key
- Or draft delivery workflow

### Automation platform
- n8n cloud invite or self-hosted access
- Make.com workspace invite

---

## Build phase checklist

### Voice calibration
- [ ] Analyze 5+ content samples
- [ ] Document tone characteristics
- [ ] Note common phrases and vocabulary
- [ ] Identify formatting preferences
- [ ] Create voice reference document

### Prompt engineering
For each output format:
- [ ] Write initial prompt
- [ ] Test with 3+ content pieces
- [ ] Review outputs against voice guide
- [ ] Refine prompt based on results
- [ ] Document final prompt with examples

### Workflow construction
- [ ] Build trigger mechanism
- [ ] Create transformation steps
- [ ] Add error handling
- [ ] Test end-to-end with real content
- [ ] Document workflow logic

### Output quality check
- [ ] Platform-appropriate length
- [ ] Consistent voice across formats
- [ ] Proper formatting (line breaks, emojis if used)
- [ ] Hashtag handling (if applicable)
- [ ] Link handling (if applicable)

---

## Review phase

### Send to client
- [ ] 3-5 sample outputs from real content
- [ ] Side-by-side: original vs. transformed
- [ ] Questions about preferences
- [ ] Request feedback within 48 hours

### Revision areas to expect
- Tone adjustments
- Length preferences
- Formatting changes
- Specific phrase additions/removals
- Platform-specific tweaks

---

## Training session

### Preparation
- [ ] All workflows tested and working
- [ ] Sample outputs ready to show
- [ ] Documentation complete
- [ ] Screen share ready
- [ ] Recording ready

### Training agenda
1. Overview: what the system does
2. Demo: run a piece of content through
3. Show each output and explain prompts
4. How to trigger workflows
5. How to edit outputs before posting
6. How to handle errors
7. Q&A

### After training
- [ ] Send recording within 24 hours
- [ ] Send documentation package
- [ ] Send final invoice

---

## Project close

### Final delivery
- [ ] All workflows live
- [ ] Prompts documented
- [ ] Training complete
- [ ] Final payment received

### Support handoff
- [ ] Confirm support period
- [ ] Set end date reminder
- [ ] Explain support scope

### Post-project
- [ ] Request testimonial (after 1-2 weeks of use)
- [ ] Offer prompt optimization (after 1 week)
- [ ] Note additional platform opportunities
- [ ] Add to case study pipeline
