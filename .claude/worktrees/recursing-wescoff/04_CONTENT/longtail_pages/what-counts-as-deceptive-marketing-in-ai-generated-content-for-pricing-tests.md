## What counts as deceptive marketing in AI-generated content for pricing tests

You're running a pricing test. One landing page says "Only 3 left in stock." Another says "50 available." Both are AI-generated copy.

Neither is true. You're testing which converts better.

Is that deceptive? To the FTC, it depends.

## The FTC standard for tests

The FTC doesn't ban A/B testing. They ban false claims that deceive consumers into buying.

The distinction: a false scarcity claim ("only 3 left") when the intention is to trick someone into impulse buying. That's deceptive.

Transparent testing ("we're testing different messaging") is fine. Hidden manipulation is not.

Here's where the line is.

## What's clearly deceptive

Fake scarcity meant to manipulate purchase decisions. "Only 2 left" when you have 10,000 in stock. The FTC has sued companies for this.

Fake social proof in tests. "Join 50,000 happy customers" (you have 100) to test if social proof increases conversions. That's false.

Fake price comparisons. "Was $99, now $49" when the original price never existed. Even if it's just for testing.

Fake authority claims. "Doctors recommend" or "Proven by Harvard" in AI copy to test if authority converts better. False claims are false.

The pattern: the false statement is the variable you're testing. You're using deception as the experimental lever.

## What's in the gray area

Some tests are risky but not clearly illegal:

**Emotional scarcity.** "Register before December 25" when registration closes then, but not every December 25. You're testing if deadline copy converts better. The deadline is real but somewhat artificial. Risky but not clearly deceptive.

**Aspirational framing.** "Become a productivity master" when no one becomes a master from your product. You're testing inspirational language. Borderline. The FTC is less aggressive here but it's not bulletproof.

**Exaggerated benefit.** "Save 5 hours per week" when the average is 2.5 hours. You're testing if higher benefit claims drive conversions. This is risky. If even one customer sues, you might lose.

**Composite data.** "Customers report saving $500/month" when that's the average of five testimonials but most save $50-100. You're testing if higher numbers convert better. The FTC views this as misleading.

## What's safe

Test different truthful messaging. "Save time" vs "Work smarter" vs "Get your weekends back." All true. Different framings. Safe.

Test real scarcity. You actually have limited inventory. You test "4 left" vs "9 left" vs "Last one." All true. Safe.

Test different pricing tiers. "$29/month" vs "$79/month." Both real price points. Test which converts better. Safe.

Test real social proof. Use actual customer counts. "Trusted by 500 businesses" if that's real. "Loved by 2000 users" if that's real.

Test genuine urgency. "Sale ends Friday" if Friday is real. "Early-bird pricing" if pricing actually changes.

Test transparent disclaimers. "We're testing different messaging to see what works. This page is experimental." Then any reasonable claim is safer because you're being transparent.

## The tricky part

The FTC cares about the consumer's perspective. Would a reasonable person be deceived?

If you're testing "The #1 productivity app" (you're #47) vs "The fastest productivity app" (#3), the first is deceptive. A reasonable person thinks you're #1.

If you're testing "Join thousands" (you have 2,000) vs "Join millions" (you have 2,000), the second is false. A reasonable person thinks you have millions.

The fact that you're testing doesn't make it okay.

## How to run tests legally

Use only truthful claims. "We save people time" vs "We save people money" vs "We help you focus." All true about your product. Test which resonates.

Use real data. If you have customer testimonials, use real ones. Different testimonials convey different value. Test message, not truth.

Use transparent disclaimers if testing edge cases. "Based on feedback from beta users" or "Estimated time savings" or "Test results from our research."

Document your testing. If the FTC ever audits you, having a file that says "we tested different pricing language" proves intent was research, not deception.

Set a time limit. Don't test indefinitely. "This test runs January 1-15" shows you're not permanently deceiving.

## Real scenario

You're building a SaaS. You have two landing pages:

Page A: "Save 3 hours per week based on user testing."
Page B: "Most people save 2-4 hours per week."

Both true. Different framing. Test away. Legal.

Page A: "Save 10 hours per week." (You have no data for this.)
Page B: "Save 2 hours per week." (This is real.)

Page A is deceptive. Don't use it even for testing.

## The FTC's position on AI

The FTC doesn't have special rules for AI content. The same standard applies: don't deceive.

But using AI to generate fake testimonials or false claims is worse optics-wise. If audited, having "we used AI to generate scarcity text" is harder to defend than "our copywriter wrote aspirational language."
