## OpenCode vs Make for sales follow-ups

You have 100 leads in your inbox. You need to send follow-ups to the ones who didn't reply in 3 days. Manually? That's 2 hours of work. Automation? That's where OpenCode and Make come in.

But which one should you pick?

## The quick answer

Use Make if you want a visual workflow builder and have $10-$15 per month to spend. Use OpenCode if you want to write code and prefer free.

Here's the full breakdown.

## Make: the visual approach

Make is Zapier's less strict cousin. It has a drag-and-drop interface where you build workflows by connecting blocks.

For sales follow-ups, it looks like this: Gmail trigger (new email) → delay 3 days → check if reply exists → send follow-up email.

What works:

- Visual builder is fast to understand. No coding required.
- Pre-built connectors for everything. Gmail, Airtable, Stripe, HubSpot.
- Logging is clear. You see what each step did.
- Pricing is reasonable. $9.99-$16 per month for most solopreneurs.

What doesn't:

- Workflow logic has limits. Complex conditional branching gets messy fast.
- Rate limiting happens. If you hit 1000s of operations per month, it slows down.
- Customer support is slow. Response times are 24-48 hours.
- Testing requires full runs. No dry-run mode.

Cost for a typical solo sales automation: $10-$15/month.

## OpenCode: the code-first approach

OpenCode is a newer player. It's basically a backend for code-first automation. You write JavaScript, it runs on a schedule.

For sales follow-ups: fetch all leads from database → filter ones without replies in 3 days → send email via Resend or SendGrid → log result.

What works:

- Free tier is generous. 1000 executions per month, no credit card needed.
- No UI overhead. You're thinking in code, not dragging blocks.
- Flexibility is huge. You can do complex logic easily.
- Fast support. Small team, quick responses.

What doesn't:

- You need to code. If you can't write basic JavaScript, it's a barrier.
- Fewer pre-built integrations. You wire things manually.
- Logs require reading code. Less visual than Make.
- Scaling costs money. After 10,000 runs, it gets pricey.

Cost for typical solo sales automation: $0 (free tier works).

## Real scenario

You run a SaaS with 50 leads per day. After 3 days with no reply, you want to send a follow-up.

**With Make:**
- Build the workflow in the UI (30 minutes).
- Connect Gmail and SendGrid.
- Set up conditional logic (if no reply after 3 days, send email).
- Cost: $12/month.
- Time to first auto-email: ~1 hour.

**With OpenCode:**
- Write 20 lines of JavaScript (15 minutes).
- Set it to run daily.
- Deploy.
- Cost: $0/month.
- Time to first auto-email: ~30 minutes.

If you can code, OpenCode wins. If you can't, Make wins.

## The hidden costs

Make: you'll spend $120-$180 per year, plus time debugging visual workflows when they break. And they break.

OpenCode: free for months 1-6, then $5-$20 per month as you scale. But when something breaks, you fix it in code, which is usually faster.

## Which one actually works better for sales follow-ups

Both work. Neither is secretly broken.

Make is reliable because it's been around longer. OpenCode is newer but faster to iterate on.

The real question: can you code or do you prefer not to?

If you can code, you're leaving money on the table using Make. Use OpenCode. If you can't, Make is worth the $12/month to avoid learning JavaScript.
