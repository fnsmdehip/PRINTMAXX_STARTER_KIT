## 1. Above-the-Fold Anatomy

The first viewport (above-the-fold) determines 80% of conversions. Every element must earn its place.

### The 5-Second Test

Users decide in 5 seconds:
1. What is this? (Headline)
2. Is it for me? (Subheadline)
3. What's the benefit? (Value prop)
4. What do I do? (CTA)
5. Can I trust this? (Social proof)

### Above-the-Fold Checklist

**Must have:**
- [ ] Clear headline (8 words or less)
- [ ] Benefit-focused subheadline
- [ ] Product visual (app screenshots or video)
- [ ] Primary CTA button
- [ ] Basic social proof (user count, rating)

**Optional but powerful:**
- [ ] App Store badges
- [ ] Trust badges
- [ ] Video thumbnail with play button

### Hero Section Layouts

**Layout A: Left Text, Right Visual (Classic)**
```
|---------------------------|
| Headline          | App   |
| Subheadline       | Mock  |
| Social proof      | up    |
| [CTA Button]      | Image |
|---------------------------|
```
Best for: Feature-focused apps, B2B

**Layout B: Centered with Visual Below**
```
|---------------------------|
|        Headline           |
|       Subheadline         |
|      [CTA Button]         |
|     [App Screenshots]     |
|---------------------------|
```
Best for: Consumer apps, simple value props

**Layout C: Full-Width Video Background**
```
|---------------------------|
|  [Video/Animation BG]     |
|     Headline over         |
|     [CTA Button]          |
|---------------------------|
```
Best for: Lifestyle apps, emotional appeal

**Layout D: Split Screen**
```
|---------------------------|
| App Video  |  Headline    |
| or Demo    |  Subhead     |
| Playing    |  [CTA]       |
|---------------------------|
```
Best for: Apps with impressive UX, complex features

## 2. Headline Formulas for Apps

### Formula 1: Outcome + Mechanism
"[Desired Outcome] with [App Mechanism]"

**Examples:**
- "Build a daily prayer habit with gentle reminders"
- "Hit your step goal by unlocking your phone"
- "Focus on studying by blocking distractions"

### Formula 2: Who + What + Benefit
"[App] helps [audience] [achieve benefit]"

**Examples:**
- "PrayerLock helps Christians pray consistently"
- "WalkToUnlock helps busy professionals move more"
- "StudyLock helps students focus without willpower"

### Formula 3: The [Adjective] [Category] App
"The [unique adjective] [app category] app"

**Examples:**
- "The faith-based focus app"
- "The movement-first phone locker"
- "The distraction-free study companion"

### Formula 4: Specific Result
"[Specific Number] [Result] [Timeframe]"

**Examples:**
- "10 minutes of prayer, every day, on autopilot"
- "2,000 more steps daily without thinking about it"
- "3 hours of focused study time, guaranteed"

### Formula 5: Problem-Solution
"Stop [pain]. Start [benefit]."

**Examples:**
- "Stop missing prayer. Start building a habit."
- "Stop sitting all day. Start moving to unlock."
- "Stop getting distracted. Start focusing deeply."

### Headline Testing Priority

Test these elements in order:
1. **Benefit framing** (outcome vs feature)
2. **Specificity** (numbers vs vague)
3. **Audience call-out** (explicit vs implicit)
4. **Emotional vs rational** (feel vs think)

## 3. Social Proof Strategy

### Social Proof Hierarchy (by impact)

| Type | Conversion Impact | Where to Place |
|------|-------------------|----------------|
| Specific results | +25-30% | Above fold, near CTA |
| User count (specific) | +20-25% | Above fold |
| Star rating (4.5+) | +15-20% | Near app store badges |
| Video testimonials | +20-30% | Below fold |
| Written testimonials | +15-25% | Middle of page |
| Press/brand logos | +5-15% | Footer or logo bar |
| App Store badges | +10-15% | Near primary CTA |

### Social Proof Formulas

**User Count:**
- "Join 47,392 Christians praying daily" (specific = credible)
- "Trusted by 50,000+ users" (rounded = milestone)

**Results-Based:**
- "Users report 3x more consistent prayer habits"
- "Average user walks 2,400 more steps per day"
- "Students gain 2+ hours of focused study time"

**Rating-Based:**
- "4.8 stars from 12,000+ reviews"
- "Rated #1 prayer app on the App Store"

### Testimonial Structure

**Strong testimonial formula:**
```
"[Specific result/transformation] [How the app helped] [Emotional impact]"
- [Full Name], [Relevant Context]
```

**Example:**
```
"I went from praying once a week to every single day. The gentle
reminders made it feel natural, not forced. My faith life has
completely transformed."
- Sarah M., Worship Leader, Texas
```

**Testimonial must-haves:**
- [ ] Specific result (not vague praise)
- [ ] Real name (first + last initial minimum)
- [ ] Photo (increases trust 30%)
- [ ] Relevant context (job, location, use case)
- [ ] Transformation arc (before/after implied)

### Where NOT to Use Social Proof

- Generic quotes without specifics
- Fake or AI-generated testimonials (FTC violation)
- Testimonials that sound too perfect
- Overwhelming quantity over quality
- Testimonials that don't match your audience

## 4. CTA Optimization

### CTA Copy Framework

**Formula: Action Verb + Benefit/Object**

| Weak | Strong |
|------|--------|
| Submit | Start my free trial |
| Download | Get the app free |
| Sign up | Join 10,000+ users |
| Learn more | See how it works |
| Click here | Try PrayerLock free |

### CTA Psychology

**First-person increases clicks 24%:**
- "Start my free trial" (not "Start your free trial")
- "Get my prayer guide" (not "Get the guide")

**Specificity increases trust:**
- "Download the iOS app" (not just "Download")
- "Start 7-day free trial" (not "Start trial")

### CTA Button Design

| Element | Best Practice |
|---------|---------------|
| Color | High contrast with page (green/orange work well) |
| Size | Minimum 44px height (tap target) |
| Padding | Generous (feels clickable) |
| Shape | Rounded corners (3-8px radius) |
| Text | 16px+ bold, high contrast |
| Icon | App Store badges where relevant |

### CTA Placement Strategy

**Above fold (required):**
- Primary CTA visible without scrolling
- Place in right-side of hero or centered below headline

**Sticky footer (mobile):**
- Always-visible CTA on scroll
- Appears after first scroll or 3 seconds
- Include brief value prop: "Try free for 7 days"

**After social proof:**
- Repeat CTA after testimonials
- Add urgency element if applicable

**Bottom of page:**
- Final CTA before footer
- Summary of offer + CTA

### App Store Badge Guidelines

**iOS App Store badge:**
- Official Apple badge only
- "Download on the App Store" or "Get"
- Link directly to App Store page

**Google Play badge:**
- Official Google badge only
- "Get it on Google Play"
- Link directly to Play Store page

**Placement:**
- Near primary CTA
- On hero section for app landing pages
- Always include both if cross-platform

## 5. App Preview Strategies

### Screenshot Gallery vs Video

| Format | Pros | Cons | Best For |
|--------|------|------|----------|
| Screenshots | Fast to load, scannable | Static, less engaging | Simple apps |
| Video | Engaging, shows UX flow | Slower load, requires click | Complex apps |
| GIF/Animation | Eye-catching, auto-plays | File size, accessibility | Key features |
| Interactive demo | Highest engagement | Complex to build | SaaS, web apps |

### Screenshot Gallery Best Practices

**Sequence:**
1. Core value proposition (main benefit)
2. Key feature in action
3. Secondary feature
4. Social proof / results screen
5. Settings / customization

**Design:**
- Device frame (iPhone, Android) for context
- Text overlay with benefit headline per screen
- Consistent visual style across all screenshots
- High contrast, readable on mobile

**Avoid:**
- Splash screens or login screens
- Dense settings pages
- Features without context
- Too many screenshots (5-6 max)

### Video Preview Best Practices

**Duration:** 15-30 seconds (shorter is better)

**Structure:**
```
0-3s:   Hook - Core benefit visual
3-10s:  Demo - Key feature in action
10-20s: Flow - User journey highlight
20-30s: CTA - Download/try prompt
```

**Technical:**
- Portrait orientation for mobile viewing
- No audio dependency (80% watch muted)
- Text overlays for key points
- High frame rate (30fps minimum)
- Autoplay (muted) on desktop, click-to-play on mobile

**What to show:**
- Real app UI (not marketing renders)
- Actual user flows
- Speed up long interactions
- Highlight micro-interactions

**What to avoid:**
- Talking heads (save for testimonials)
- Marketing fluff without substance
- Complex features before simple ones
- Poor quality screen recordings

## 6. Feature-Benefit Mapping

### The Transformation Framework

Every feature should connect to a transformation:

```
Feature -> Benefit -> Transformation
```

**Example mapping:**

| Feature | Benefit | Transformation |
|---------|---------|----------------|
| Prayer reminders | Never forget to pray | Consistent faith life |
| Step-locked screen | More movement throughout day | Healthier lifestyle |
| App blocker | No social media distractions | Better grades, more free time |
| Progress tracking | See your streaks grow | Feel accomplished |

### Feature Display Patterns

**Pattern 1: Icon + Feature + Benefit (Grid)**
```
[Icon] Feature Name
Brief benefit description

[Icon] Feature Name
Brief benefit description
```

**Pattern 2: Left Icon, Right Content (List)**
```
[Icon]  Feature Name
        Benefit description in 1-2 lines
        that explains the value.
```

**Pattern 3: Screenshot + Feature Highlight**
```
|-------------------------|
| [App Screenshot]        |
|  ^ Feature callout      |
|-------------------------|
| Feature Name            |
| Benefit description     |
|-------------------------|
```

### Priority Order for Features

1. **Core differentiator** - What makes you unique
2. **Primary benefit** - Main problem you solve
3. **Secondary benefits** - Supporting value
4. **Nice-to-haves** - Additional features
5. **Technical specs** - Only if relevant (device support, etc.)

## 7. Pricing Display

### When to Show Pricing

| Scenario | Show Pricing? | Why |
|----------|---------------|-----|
| Free app | Yes (free badge) | Removes friction |
| Freemium | Show free + premium options | Sets expectations |
| Subscription | Show tiers clearly | Reduces price shock |
| One-time purchase | Yes, prominently | Builds trust |
| Free trial | Emphasize trial, show post-trial price | Conversion focus |

### Price Anchoring Techniques

**1. Monthly vs Annual comparison:**
```
Monthly: $9.99/month
Annual: $59.99/year (save 50%) <- RECOMMENDED
```

**2. Daily breakdown:**
```
$59.99/year = just $0.16/day
"Less than a coffee per week"
```

**3. Tier anchoring:**
```
Basic: Free       <- Anchor
Pro: $7.99/month  <- Target
Team: $24.99/month <- Premium anchor
```

### Free Trial Display

**Above fold:**
- Emphasize "Free" prominently
- "7-day free trial" or "Try free for 7 days"
- De-emphasize price (show after trial ends)

**Near CTA:**
- "No credit card required" (if true)
- "Cancel anytime"
- Clear trial length

**Pricing section:**
- Show what happens after trial
- Clear cancellation policy
- No surprise charges messaging

## 8. Mobile-First Design

### Mobile Optimization Checklist

**Speed:**
- [ ] Page loads in under 3 seconds
- [ ] Images optimized (WebP, proper sizing)
- [ ] Above-fold content loads first
- [ ] No render-blocking resources

**Touch targets:**
- [ ] CTAs minimum 44x44px
- [ ] Adequate spacing between tappable elements
- [ ] No hover-dependent interactions

**Content:**
- [ ] Text readable without zooming (16px minimum)
- [ ] Single column layout
- [ ] No horizontal scrolling
- [ ] Concise copy (40% shorter than desktop)

**Navigation:**
- [ ] Sticky header with CTA
- [ ] Hamburger menu for secondary nav
- [ ] Clear back/home navigation

**Forms:**
- [ ] Large input fields
- [ ] Appropriate keyboard types (email, phone)
- [ ] Single field per row
- [ ] Autocomplete enabled

### Mobile-Specific Patterns

**Sticky footer CTA:**

```
+ Is it really free?
```

## 9. A/B Testing Ideas for Landing Pages

### High-Impact Tests (Test First)

| Test | Variants | Expected Impact |
|------|----------|-----------------|
| Headline | Outcome vs feature-focused | +15-30% |
| Hero visual | Video vs screenshots | +10-25% |
| CTA copy | "Start free trial" vs "Download free" | +10-20% |
| Social proof | User count vs testimonial | +10-20% |
| Pricing display | Show vs hide above fold | +/-10-15% |

### Medium-Impact Tests

| Test | Variants | Expected Impact |
|------|----------|-----------------|
| Page length | Long vs short | +5-15% |
| Feature order | Core first vs social proof first | +5-15% |
| Testimonial format | Text vs video | +5-15% |
| CTA color | Green vs orange vs blue | +2-10% |
| Button size | Large vs standard | +2-8% |

### Test Tracking Template

```
Test ID: LP-[APP]-[NUMBER]
Page: [URL]
Hypothesis: [Statement]
Variants:
  A (Control): [Description]
  B: [Description]
Primary Metric: conversion_rate
Sample Size: [Number]
Duration: [Days]
Result: [Winner/No winner]
Lift: [%]
Notes: [Learnings]
```

## 12. Landing Page Templates

### Template 1: App Launch Page

```
SECTION 1: Hero
- Headline: [Benefit Statement]
- Subheadline: [Who it's for + what it does]
- App mockup/video
- CTA: [Download/Try Free]
- Social proof: [User count + rating]
- App Store badges

SECTION 2: Problem/Solution
- "The problem" statement
- "The solution" introduction
- 3 bullet benefits

SECTION 3: Features
- 3-4 key features
- Icon + headline + description each
- Screenshot for each feature

SECTION 4: Social Proof
- 3 testimonials
- Results/stats callout
- Press mentions (if available)

SECTION 5: How It Works
- 3 simple steps
- Visual for each step

SECTION 6: Pricing (if applicable)
- Clear tier comparison
- Free trial emphasis
- Risk reversal (guarantee)

SECTION 7: FAQ
- 5-7 common questions
- Address objections

SECTION 8: Final CTA
- Headline recap
- CTA button
- Last social proof element

FOOTER:
- Links (Privacy, Terms, Contact)
- Copyright
```

```
SECTION 1: Hero
- Headline: [What they get]

SECTION 2: What's Inside
- 3-5 bullet points of value
- Preview/sample

SECTION 3: Author/Creator Credibility
- Brief bio
- Social proof
- Photo

SECTION 4: Final CTA
- Repeat form
- "Free" emphasis
- Privacy reassurance

FOOTER: Minimal
```

## 14. Post-Launch Optimization Cadence

### Week 1
- [ ] Monitor conversion rate daily
- [ ] Check heatmaps for unexpected patterns
- [ ] Watch session recordings (10+ per day)
- [ ] Note any technical issues

### Week 2-4
- [ ] Launch first A/B test (headline or CTA)
- [ ] Continue heatmap analysis
- [ ] Document learnings
- [ ] Plan next test

### Monthly
- [ ] Full conversion audit
- [ ] Competitive page review
- [ ] Update testimonials/social proof
- [ ] Review and optimize underperforming sections

### Quarterly
- [ ] Major redesign consideration
- [ ] New feature/offer integration
- [ ] Complete analytics review
- [ ] Update screenshots/videos if app changed

## Quick Reference: Conversion Killers to Avoid

| Killer | Why It Hurts | Fix |
|--------|--------------|-----|
| Slow load time | 53% leave if > 3s | Optimize images, reduce scripts |
| Confusing headline | Users don't understand value | Simplify, test variants |
| No CTA above fold | Users don't know what to do | Add clear button |
| Too many choices | Decision paralysis | Single primary CTA |
| No social proof | No trust, no conversion | Add testimonials/numbers |
| Hidden pricing | Feels like a trick | Be upfront about cost |
| Too many form fields | Friction kills conversion | Email only to start |
| No mobile optimization | 60%+ traffic is mobile | Responsive design |
| Generic stock photos | Feels inauthentic | Use real product/people |
| Wall of text | Nobody reads it | Bullet points, white space |
