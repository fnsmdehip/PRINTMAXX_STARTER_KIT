## Fraud Prevention

### Red Flags to Watch

**Immediate investigation (auto-flag):**

| Signal | Threshold | Risk |
|--------|-----------|------|
| Same device ID | Referrer = referee | Self-referral |
| Same IP within 1 hour | Referrer = referee | Self-referral |
| Sequential emails | john1@, john2@, john3@ | Fake accounts |
| Disposable email domains | mailinator, guerrillamail, etc. | Fake signups |
| Bulk signups | >10 from same source in 1 hour | Bot farm |

**Monitor closely (hold rewards):**

| Signal | Threshold | Risk |
|--------|-----------|------|
| High velocity | >10 referrals/day | Gaming system |
| Low engagement | Referee <3 sessions | Fake users |
| Geographic mismatch | Referrer US, referees all overseas | Purchased signups |
| Referral link on coupon sites | Any | Policy violation |
| Paid ads to referral link | Any | Policy violation |

### Verification Steps Before Reward

**Automated checks:**

```python
def verify_referral(referrer, referee):
    # Check 1: Different devices
    if referrer.device_id == referee.device_id:
        return {"valid": False, "reason": "same_device"}

    # Check 2: Different IPs (within timeframe)
    if referrer.ip == referee.ip:
        time_diff = referee.signup_time - referrer.last_active
        if time_diff < timedelta(hours=1):
            return {"valid": False, "reason": "same_ip_suspicious"}

    # Check 3: Email pattern detection
    if detect_sequential_email(referrer.email, referee.email):
        return {"valid": False, "reason": "email_pattern"}

    # Check 4: Disposable email
    if is_disposable_email(referee.email):
        return {"valid": False, "reason": "disposable_email"}

    # Check 5: Referee engagement (delayed check)
    if referee.session_count < 3 and days_since_signup > 7:
        return {"valid": False, "reason": "low_engagement"}

    # Check 6: Referrer velocity
    recent_referrals = get_referral_count(referrer.id, hours=24)
    if recent_referrals > 10:
        return {"hold": True, "reason": "velocity_review"}

    return {"valid": True}
```

**Manual review triggers:**
- User hits 15+ referrals in one month (first time)
- Conversion rate >50% (suspiciously high)
- Multiple referees cancel within 48 hours
- Referral link found on deal/coupon sites

### What Happens When Fraud is Detected

**Tier 1: Warning (first offense, minor):**
- Email user explaining the issue
- Void specific fraudulent referrals
- Keep account active
- Flag for monitoring

**Tier 2: Reward freeze (repeated or moderate):**
- Freeze all pending rewards
- Require manual review for future referrals
- Email warning with specific policy violated
- 30-day probation period

**Tier 3: Ban (confirmed fraud or severe):**
- Void all pending and historical rewards
- Terminate affiliate/referral access
- Ban email from future programs
- Optional: Terminate main account

### Anti-Fraud Measures

**Fraud Patterns to Detect:**

1. **Self-referrals**
   - Same device fingerprint
   - Same IP address within 24 hours
   - Email pattern matching (john1@, john2@, john3@)

2. **Bot farms**
   - High volume from single IP range
   - Signup velocity > 10/hour from single source
   - No app interaction post-signup

3. **Incentivized spam**
   - Mass sharing to low-engagement users
   - High share rate, low conversion rate

**Prevention Code:**

```javascript
function validateReferral(referrer, referred) {
  // Check 1: Different devices
  if (referrer.deviceId === referred.deviceId) {
    return { valid: false, reason: 'same_device' };
  }

  // Check 2: Different IPs (with grace period)
  if (referrer.lastIP === referred.signupIP) {
    const timeDiff = referred.signupTime - referrer.lastActiveTime;
    if (timeDiff < 3600000) { // 1 hour
      return { valid: false, reason: 'same_ip_suspicious' };
    }
  }

  // Check 3: Email pattern
  const emailSimilarity = calculateEmailSimilarity(referrer.email, referred.email);
  if (emailSimilarity > 0.8) {
    return { valid: false, reason: 'email_pattern' };
  }

  // Check 4: Referrer velocity
  const recentReferrals = getReferralCount(referrer.id, lastHours: 24);
  if (recentReferrals > 10) {
    return { valid: false, reason: 'velocity_limit' };
  }

  return { valid: true };
}
```

- Max 10 successful referrals per 24 hours
- Max 50 referrals per month per user

## Viral Coefficient Optimization

### K-Factor Formula

**Definition:** K-factor = (number of invites sent per user) × (conversion rate of invites)

**Example:**
- Each user invites 5 friends
- 20% of invites convert to signups
- K-factor = 5 × 0.20 = 1.0

**K-factor targets:**

| K-Factor | Result | Status |
|----------|--------|--------|
| < 0.5 | Declining growth | Bad |
| 0.5-0.9 | Slow growth | Okay |
| 1.0 | Exponential growth | Good |
| > 1.0 | Viral growth | Excellent |

**How to increase K-factor:**

1. **Increase invites sent per user:**
   - Make sharing easier (one-tap share vs multi-step)
   - Incentivize sharing (double-sided rewards)
   - Trigger prompts at high-intent moments
   - Show social proof ("1,234 users shared today")

2. **Increase invite conversion rate:**
   - Improve landing page (clear value prop)
   - Offer referred user reward
   - Use social proof on signup page
   - Reduce friction (email-only signup vs full form)

### Viral Cycle Time

**Definition:** Time from when user signs up to when they refer someone.

**Target:** < 3 days for consumer apps, < 7 days for B2B.

**How to reduce:**

- Prompt to share immediately after first "aha moment"
- Send reminder email 24 hours after signup
- Show referral CTA on success screens
- Gamify with "Share to unlock [feature]"

### Viral Coefficient Calculation

**Tracking formula:**

```javascript
function calculateViralCoefficient(startDate, endDate) {
  const initialUsers = getUserCount(startDate);
  const newUsers = getNewUsers(startDate, endDate);
  const invitesSent = getInvitesSent(startDate, endDate);
  const invitesConverted = getInvitesConverted(startDate, endDate);

  const avgInvitesPerUser = invitesSent / initialUsers;
  const inviteConversionRate = invitesConverted / invitesSent;

  const kFactor = avgInvitesPerUser * inviteConversionRate;

  return {
    kFactor,
    avgInvitesPerUser,
    inviteConversionRate,
    cycleTime: getAvgCycleTime(startDate, endDate)
  };
}
```

### Fraud Detection Checklist

**Pre-Launch:**
- [ ] Device fingerprinting implemented
- [ ] IP address logging enabled
- [ ] Email verification required before reward
- [ ] Velocity limits set (max referrals per hour/day)
- [ ] Manual review for high-value rewards (>$100)

**Post-Launch Monitoring:**
- [ ] Daily review of referral velocity by user
- [ ] Weekly review of conversion rates by source
- [ ] Monthly audit of top referrers (check for patterns)
- [ ] Quarterly fraud rate calculation (flagged / total)

### Red Flags

| Signal | Threshold | Action |
|--------|-----------|--------|
| Same IP for referrer + referred | < 1 hour apart | Auto-flag |
| Device ID match | Any | Auto-reject |
| Email similarity (john1@, john2@) | > 80% match | Auto-flag |
| Referral velocity | > 10/day | Manual review |
| Low engagement post-signup | < 1 session | Delay reward 7 days |
| Mass signups from single source | > 50/day | Pause referrals, investigate |

### Response Protocols

**Auto-Reject (No Reward):**
- Same device ID
- Same IP within 10 minutes
- Previously banned user
- Disposable email domains (confirmed via API)

**Manual Review (Hold Reward):**
- High velocity (10+ referrals/day)
- Low post-signup engagement
- Pattern matching emails
- Unusual geographic clustering

**Ban User:**
- Confirmed fraud (after manual review)
- Multiple violations
- Using bots or scripts
- Selling referral services

### Fraud Rate Targets

**Acceptable fraud rates:**

| Program Type | Target Fraud Rate | Action Threshold |
|--------------|-------------------|------------------|
| Low-value rewards (<$10) | < 5% | Tighten at 8% |
| Medium rewards ($10-50) | < 3% | Tighten at 5% |
| High rewards (>$50) | < 1% | Tighten at 2% |

**Calculation:**

```
Fraud Rate = (Flagged Referrals / Total Referrals) × 100
```

**If above threshold:**
1. Increase verification requirements
2. Add manual review step
3. Reduce reward value
4. Extend reward delay period

- [ ] Unique referral code generation system
- [ ] Deep linking for mobile apps (Branch.io, Firebase)
- [ ] Attribution tracking (who referred whom)
- [ ] Reward delivery system (credits, access, emails)
- [ ] Fraud detection logic
- [ ] Analytics dashboard (referral funnel)

**Copy:**
- [ ] In-app referral prompt (headline + CTA)
- [ ] Share message template (SMS, email, social)
- [ ] Landing page for referred users
- [ ] Email templates (milestone, reminder, achievement)
- [ ] FAQ page (how it works, rules, payout)

**Legal:**
- [ ] Terms & conditions (referral program specific)
- [ ] Fraud policy (what's not allowed)
- [ ] Payout terms (thresholds, timing, methods)
- [ ] Tax implications (1099 for affiliates >$600/year)

### Launch

**Day 1:**
- [ ] Announce to existing users (email + in-app)
- [ ] Post on social media (X, LinkedIn)
- [ ] Add referral page to website nav
- [ ] Monitor first 100 referrals for issues

**Week 1:**
- [ ] Send reminder email to users who haven't shared
- [ ] Post success stories (first users to hit milestones)
- [ ] Check fraud rate (should be < 2%)
- [ ] Fix any bugs in attribution or reward delivery

**Month 1:**
- [ ] Calculate K-factor
- [ ] Review top referrers (interview for case study)
- [ ] A/B test reward amounts or copy
- [ ] Optimize based on data

### Ongoing Optimization

**Monthly:**
- [ ] Calculate K-factor and viral coefficient
- [ ] Review fraud rate
- [ ] Analyze conversion funnel (prompt → share → signup → activation)
- [ ] Run one A/B test

**Quarterly:**
- [ ] Deep dive on top referrers (what's working)
- [ ] Survey non-sharers (why didn't you refer?)
- [ ] Adjust reward structure based on ROI
- [ ] Update copy and creative

## Example Reward Structures (Specific Numbers)

### App: Fitness Tracker

**Referrer:** 1 month Premium free (value: $10)
**Referred:** 1 week Premium free (value: $2.50)

**Milestones:**
- 5 referrals: Custom workout plan (value: $50)
- 10 referrals: 1-on-1 coaching session (value: $100)
- 25 referrals: Lifetime Pro (value: $500)

**K-Factor Target:** 0.6
**Expected:** 30% of users share → avg 3 invites → 20% convert = 0.18 K-factor (need to optimize)

### Newsletter: Solopreneur Growth

**Milestones:**
- 1 referral: Content repurposing checklist (value: $10)
- 5 referrals: ASO keyword research sheet (value: $50)
- 10 referrals: App marketing stack (value: $200)
- 25 referrals: 30-min strategy call (value: $500)
- 50 referrals: Lifetime Pro access (value: $1000)

**K-Factor Target:** 1.2 (newsletter referrals typically higher than apps)
**Expected:** 15% of subscribers share → avg 8 invites → 20% convert = 0.24 K-factor

### SaaS Product: Email Tool

**Affiliate Structure:**
- 30% recurring commission for 12 months
- Monthly plan: $50/mo → $15/mo to affiliate
- Annual plan: $500/yr → $150 to affiliate (one-time)

**Tiers:**
- 0-10 customers: 30% commission
- 11-50 customers: 35% commission
- 51+ customers: 40% commission + $500 quarterly bonus

**Payout:** $100 minimum, monthly via PayPal or bank transfer

## Copy Templates for Referral Prompts

### High-Performing Prompts (Tested)

**Prompt 1: Mutual Benefit**
```
Enjoying [APP]? Share it.

You and your friend both get [REWARD].

[Share Now]
```
**Why it works:** Clear, reciprocal, no fluff.

**Prompt 2: Social Proof**
```
1,234 users shared [APP] today.

Invite a friend. You both get [REWARD].

[Get My Link]
```
**Why it works:** FOMO + bandwagon effect.

**Prompt 3: Milestone-Based**
```
You just hit [MILESTONE]!

Celebrate by sharing [APP] with friends.
You both unlock [REWARD].

[Share]
```
**Why it works:** Ties to positive moment (dopamine high).

**Prompt 4: Scarcity (Limited Time)**
```
Double rewards this week only.

Refer a friend → You both get [2X_REWARD].

Ends Sunday.

[Share Now]
```
**Why it works:** Urgency + increased perceived value.

## Key Metrics Dashboard

Track these weekly:

| Metric | Formula | Target |
|--------|---------|--------|
| K-Factor | (Invites/User) × (Conversion Rate) | > 1.0 |
| Invites per User | Total Invites / Total Users | > 3 |
| Invite Conversion Rate | Signups from Invites / Total Invites | > 20% |
| Viral Cycle Time | Avg days from signup to first referral | < 3 days |
| Fraud Rate | Flagged Referrals / Total Referrals | < 3% |
| Referral Attribution | % of new users from referrals | > 30% |
| Reward Cost per Acquisition | Total Reward Cost / New Referred Users | < 50% of LTV |
