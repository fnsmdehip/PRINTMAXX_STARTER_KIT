1. Execution Hierarchy (L0-L6)

Not a flat swarm of agents — a 7-layer hierarchy where each layer feeds the next. L4 scrapes 133 accounts and 18 subreddits overnight. L2 scores and routes by 5:30 AM. L0 makes promote/kill decisions every 2 hours. L5 blocks anything that fails quality gates. All via launchd/cron, zero human prompting.

2. Intelligence Pipeline

This is the system's nervous system. 7 scrapers run between 5:00-7:00 AM, dump raw intel into ALPHA_STAGING (44K+ entries so far), which gets auto-scored on 4 dimensions, routed through 10-step integration, ranked by Capital Genesis, and served to every agent. Agent outputs feed back into the scrapers. Closed loop, daily, no human in the chain.

Alpha Scoring Formula (alpha_auto_processor.py)

4 dimensions, 25 points each (0-100 total):

Engagement (25): "viral", "trending", "million", "followers" keywords
Specificity (25): $-amounts, percentages, "step-by-step", "framework", "template"
Actionability (25): "sign up", "create", "build", "automate", "deploy"
Revenue (25): "revenue", "profit", "MRR", "commission", "affiliate"

Bonuses: +10 HIGHEST ROI, +5 HIGH ROI, +5 synergy_score > 500, +2 synergy > 100
Penalties: -5 per low-value signal ("engagement bait", "vague", "generic")

Deduplication: URL hash → content hash (first 150 chars) → OPS filename keyword overlap (3+ word match)

Phase-Aware Weight Shifting

As revenue grows, the system automatically rebalances priorities. Phase 0 sprints for speed. Phase 4 protects what's been built.

Hidden Twitter Signals

TweepCred Score (0-100): Hidden reputation score. New accounts start at -128. X Premium gives +100 boost. Below 65, only 3 tweets considered for distribution.

30-Minute Velocity Window: First 30 minutes of a tweet's life determine its distribution trajectory. Miss the window = buried.

Cold Start Suppression: <0.5% engagement on first 100 tweets = distribution cut to 10% of normal. Warmup protocol directly addresses this.

Penalties: 3+ hashtags = 40% reach penalty. External links in tweet body = 30-50% reach penalty. Solution: links in reply.

Link Penalty: External links in post body = -60% reach (up from -30-40%). Links must go in first comment.

Brave Cookie AES Decryption (Custom Engineering)

The system bypasses the X/Twitter API entirely by extracting and decrypting cookies directly from Brave browser's SQLite database on macOS. No API keys, no rate limits from the API tier, no OAuth.

Exact decryption process:
1. Locate Brave cookie DB: ~/Library/Application Support/BraveSoftware/Brave-Browser/Default/Cookies
2. Get encryption key from macOS Keychain: security find-generic-password -s "Brave Safe Storage" -w
3. Cache key at AUTOMATIONS/.brave_cookie_key
4. Derive AES key via PBKDF2-HMAC-SHA1 (salt: saltysalt, iterations: 1003, 16-byte output)
5. Copy cookie DB to temp file (Brave stays open, no interruption)
6. Query SQLite for cookies matching .x.com and .twitter.com
7. Decrypt v10-prefixed values using AES-CBC with 16-byte null IV
8. Strip PKCS7 padding, skip first 32 bytes
9. Inject into Playwright browser context

Result: Full authenticated X/Twitter access as logged-in user. Used by 4 scrapers (825-line twitter_alpha_scraper covering 133 accounts, 859-line viral_content_scanner for 10K+ like detection, 564-line twitter_content_scraper, 437-line background_twitter_scraper).
