What Is OpenCode?

The new open-source AI coding agent that hit #1 on Hacker News today

OpenCode at a Glance

OpenCode is an open-source AI coding agent that runs in your terminal. It launched today and immediately hit #1 on Hacker News with 685 upvotes and 301 comments — that's a strong signal from the developer community.


Key facts:

  • Fully open-source (MIT license)
  • Terminal-based, similar UX to Claude Code
  • Works with any LLM provider (Claude API, OpenAI, local models via Ollama)
  • Self-hostable — you own your data and configuration
  • No subscription required — you pay only for API calls you make
  • Actively developed, community-maintained

Important note on pricing: OpenCode is free software, but you still need an LLM API key. If you use Claude API to power OpenCode, you pay Anthropic per token. At moderate usage, this often costs more than the $20/month Claude Code subscription. The "free" only applies if you use cheaper providers or local models.

When to Use Each

Real use cases, not marketing copy

Individual feature dev OpenCode

Building a specific feature, fixing a bug, refactoring a module. OpenCode handles focused coding tasks well at zero software cost.

Autonomous pipelines Claude Code

Running multiple agents in parallel, long-running tasks, systems that make decisions and act autonomously over hours. Claude Code's subagent architecture handles this natively.

Privacy-first development OpenCode

Working with sensitive codebases where you don't want data leaving your machine. Self-host OpenCode with a local model and nothing leaves your environment.

MCP tools + integrations Claude Code

Using MCP servers for Stripe, Playwright, Pinecone, Firebase, etc. Claude Code's MCP support is mature. OpenCode's is still developing.

Solo dev on a budget OpenCode

Light to moderate usage with a cheaper LLM provider. For devs who don't need agentic features, OpenCode + GPT-4o-mini or local models is a strong cost play.

Revenue system building Claude Code

Building apps, automations, content pipelines, or any system where Claude orchestrates multiple tools. The full Claude Code ecosystem (subagents, MCP, CLAUDE.md, hooks) has no equivalent in OpenCode yet.

The Architecture Difference

Why they don't actually compete for the same users

Claude Code is a platform. OpenCode is a tool.

This is the most important distinction. Claude Code has evolved into a full agent platform:

  • Spawn subagents that run in parallel and report back
  • CLAUDE.md project memory that persists across sessions
  • Hooks that run shell commands on tool use events
  • Skills system for reusable agent behaviors
  • Full MCP ecosystem with 19,000+ available servers
  • Permission system with fine-grained control

OpenCode is a great coding assistant. It's not (yet) a platform for building agentic systems. If you need Claude Code as a coding assistant, OpenCode is a compelling free alternative. If you need Claude Code as an agent platform, OpenCode isn't a replacement.

The power user perspective: The developers who get the most from Claude Code aren't using it as a faster way to write code. They're using it to build systems that run autonomously — scrapers, content pipelines, deployment orchestrators. For that use case, Claude Code's subagent architecture and MCP support is the product. OpenCode doesn't offer this today.

Frequently Asked Questions

Is OpenCode safe to use?

OpenCode is open-source (MIT license) — you can read every line of code. It doesn't collect telemetry by default and doesn't phone home unless you configure it to use cloud APIs. For self-hosted local model usage, it's as private as the model you use.

Will OpenCode get the features Claude Code has?

Likely some of them, over time. The open-source community moves fast when a tool gets 685 HN upvotes on day one. But Claude Code's tightest integration points (Claude model access, Anthropic's permission system architecture, native MCP support) are harder to replicate in open source.

Can I switch from Claude Code to OpenCode?

For basic usage, yes. Your CLAUDE.md files won't transfer directly, but the concept of project instruction files exists in OpenCode too. Your shell scripts, MCP server configs, and actual code obviously transfer fine. The biggest loss would be skills, hooks, and the subagent system.

What about Cursor, Windsurf, and GitHub Copilot?

Cursor and Windsurf are IDE-based (you work inside a GUI editor). Claude Code and OpenCode are terminal-based (you type prompts, they execute). They solve similar problems with different UX philosophies. See our Cursor vs Claude Code comparison for that breakdown.
