Protocol sensitivity and the "testing paradox" (non-magic constraints)

The framework allows protocol sensitivity, but only as extra variables, not as an exemption. Therefore every claimed effect must specify:

  • State variables measured (HRV/GSR/EMG etc)
  • Regime variables tracked (geomagnetic proxy, social context)
  • Channel constraints (mechanical contact, EM shielding, acoustic environment)
  • Pre-registered analysis to prevent narrative drift

Instrument limitation principle (what "nulls" can and cannot mean)

A null result is weak evidence against existence when:

  • The signal is below sensitivity
  • The wrong coupling/bandwidth is measured
  • Effects are thresholded/conditional
  • The protocol collapses the relevant state variables

But a null becomes strong evidence against when:

  • Sensitivity and coupling are adequate
  • Regime variables are matched
  • Analysis is locked
  • The predicted structure fails to appear

This is not a blanket excuse for nulls. It is a classification system that makes explicit which nulls are informative and which are not.
